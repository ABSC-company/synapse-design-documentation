# 2.2. Редактор документов — Техническое задание для дизайнера

## Что такое редактор документов

Редактор документов — это WYSIWYG-компонент для создания переиспользуемых шаблонов медицинских и иных документов. Пользователь строит структуру документа визуально: добавляет текст, таблицы, изображения, вставляет переменные (данные из системы), поля ввода (данные, которые будут заполнены пользователем при создании документа по шаблону), блоки подписей, формулы (вычисленные значения) и результаты по условию (значения, зависящие от выполнения логических условий).

Итоговый документ хранится в HTML-формате. Редактор отображает его визуально, не показывая пользователю код.

## Общий layout компонента

```
┌─────────────────────────────────────────────────────────────────┐
│  TOOLBAR (фиксированная, прилипает к верху редактора)           │
│  [Документ ▾] [B] [I] [S] [U] [A▾] [14▾] [← ↔ → ↔↔]          │
│  [Изображение ▾] [Таблица ▾] [Список ▾]                         │
│  [{{ }} Переменная ▾] [Поле ввода ▾] [✏ Подпись ▾]             │
│  [Σ Формула ▾] [? Условие ▾]                                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  РАБОЧАЯ ОБЛАСТЬ (белый лист документа, скроллится вертикально) │
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                                                         │   │
│  │  Текстовый контент, таблицы, изображения,               │   │
│  │  переменные, поля ввода, блоки подписей,                │   │
│  │  формулы и результаты по условию                        │   │
│  │                                                         │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

- Toolbar: фиксированная панель у верхнего края редактора. При скролле документа прилипает к верху.
- Рабочая область: белый прямоугольник, имитирующий лист документа (A4), центрированный горизонтально. Скроллится вертикально при длинном контенте.
- Общий фон вокруг "листа": нейтральный серый — чтобы граница листа была читаема.

## Панель инструментов (Toolbar) — детализация

### Группировка кнопок

Кнопки toolbar сгруппированы по смысловым блокам. Между группами — визуальный разделитель (вертикальная линия).

```
[Документ ▾] | [B] [I] [S] [U] [A▾] [14▾] [← ↔ → ↔↔] | [🖼 ▾] [⊞ ▾] [≡ ▾] | [{{ }} ▾] [Поле ▾] [✏ Подпись ▾] [Σ Формула ▾] [? Условие ▾]
```

**Блок 1: Документ**
- Кнопка-дропдаун "Документ" — управление макетом документа.
- Открывает панель с вариантами:
  - Одна колонка (по умолчанию)
  - Две колонки

**Блок 2: Форматирование текста**
- `B` — жирный (toggle)
- `I` — курсив (toggle)
- `S` — зачёркнутый (toggle)
- `U` — подчёркнутый (toggle)
- `A▾` — цвет текста (открывает color picker)
- `14▾` — размер шрифта (открывает дропдаун или inline-input со значением)
- Кнопки выравнивания: по левому краю, по центру, по правому краю, по ширине (toggle-группа, одна активна)

**Блок 3: Медиа и структура**
- `Изображение ▾` — дропдаун с вариантами: загрузить файл, drag & drop зона, вставить по URL; также удалить выбранное изображение
- `Таблица ▾` — дропдаун с действиями: добавить таблицу, добавить строку, убрать строку, добавить столбец, убрать столбец, объединить ячейки, закрасить ячейки
- `Список ▾` — дропдаун: добавить маркированный список, добавить нумерованный список, удалить список

**Блок 4: Вставки**
- `{{ }} Переменная ▾` — дропдаун-поиск по списку переменных (см. ниже)
- `Поле ввода ▾` — дропдаун: использовать существующее поле / создать новое поле ввода
- `✏ Подпись ▾` — дропдаун: список подписей, используемых в шаблоне + кнопка добавить новую подпись
- `Σ Формула ▾` — дропдаун: список формул + кнопка добавить новую формулу
- `? Условие ▾` — дропдаун: список результатов по условию + кнопка добавить новый

## Рабочая область — элементы документа

### 1. Текст

- Стандартный inline-текст. Курсор появляется при клике.
- Выделенный текст подсвечивает toolbar-кнопки активного форматирования.
- Пустая строка показывает плейсхолдер: "Начните вводить текст..." (серый, исчезает при вводе).

### 2. Изображение

- Изображение отображается по фактическому размеру или масштабированным под ширину колонки.
- При выборе изображения (клик) появляются ручки изменения размера (resize handles) по углам и по бокам.
- Выбранное изображение обведено рамкой с акцентным цветом.
- Курсор при hover над ручкой — resize cursor.
- Вверху выбранного изображения — плавающая мини-панель: [Изменить] [Удалить].

```
┌─────────────────────────────────┐
│         [Изменить] [Удалить]    │  ← мини-панель при выборе
├─────────────────────────────────┤
│                                 │
│         [изображение]           │  ← resize handle по углам
│                                ○│  ← ○ = ручка изменения размера
└─────────────────────────────────┘
```

**Drop area (для загрузки изображения):**
- Зона drag & drop — пунктирная рамка, иконка загрузки, текст "Перетащите изображение или нажмите для выбора файла". Совмещена с дополнительным инпутом для указания ссылки на файл — пользователь может либо загрузить файл, либо вставить URL.
- При наведении файла — рамка меняет цвет, текст меняется на "Отпустите для загрузки".
- Пока изображение загружается — прогресс-бар или spinner внутри зоны.

### 3. Таблица

- Таблица отображается с видимыми границами ячеек.
- При hover на строку/столбец — появляются управляющие кнопки добавления (+) рядом с крайней строкой/столбцом.
- Выбранная ячейка подсвечивается.
- Объединённые ячейки визуально сливаются (без внутренней границы).
- Закрашенная ячейка имеет фоновый цвет выбранного цвета.
- При выделении нескольких ячеек (drag) toolbar таблицы активируется для группового действия.

```
┌──────────┬──────────┬──────────┐  ←  [+] справа для добавления столбца
│  Ячейка  │  Ячейка  │  Ячейка  │
├──────────┼──────────┼──────────┤
│  Ячейка  │  Ячейка  │  Ячейка  │  ← [+] снизу для добавления строки
└──────────┴──────────┴──────────┘
```

### 4. Список

- Маркированный: точки/тире.
- Нумерованный: 1. 2. 3.
- Стиль визуально совпадает с системным стилем типографики.

### 5. Переменная

Переменная отображается как **inline-чип** (pill/highlighted) внутри текста.

```
Пациент: [ФИО пациента]  дата рождения: [Дата рождения пациента]
```

- Чип имеет заливку фоновым акцентным цветом (например, светло-синий или лавандовый).
- Внутри чипа — человекочитаемое название переменной (не технический синтаксис).
- Чип — неразрывный инлайн-элемент: его нельзя редактировать текстом, только удалить целиком.
- При hover — tooltip с техническим идентификатором переменной (например, `var__22`).
- При клике на чип — он выбирается (обводится), клавиша Delete/Backspace удаляет его.

**Визуальные состояния чипа переменной:**

| Состояние | Описание |
|-----------|----------|
| default | Светло-синий фон, тёмный текст |
| hover | Чуть темнее фон, появляется tooltip |
| selected | Обводка акцентного цвета, слегка темнее фон |

### 6. Поле ввода

Поле ввода отображается как **inline-чип** (pill/highlighted) внутри текста, но визуально отличается от переменных.

```
Диагноз: [📝 Диагноз пациента]  Назначение: [📋 Назначение]
```

- Чип имеет другой цвет заливки от переменных (например, светло-зелёный или янтарный).
- Иконка типа поля (карандаш для input, список для select и т.д.) перед названием.
- Название поля — свойство `name` из схемы поля ввода.
- При hover — tooltip: тип поля, обязательное или нет.
- При клике — выбирается, Delete/Backspace удаляет из документа.
- Удаление чипа из документа не удаляет само поле ввода из системы.

**Визуальные состояния чипа поля ввода:**

| Состояние | Описание |
|-----------|----------|
| default | Светло-зелёный/янтарный фон, тёмный текст |
| hover | Чуть темнее фон, tooltip с типом и обязательностью |
| selected | Обводка акцентного цвета |

**Иконки типов полей ввода:**

| Тип | Иконка |
|-----|--------|
| `input` | Текстовое поле (T или карандаш) |
| `number_input` | Цифра или # |
| `file_input` | Скрепка или файл |
| `select` | Один выбор из списка (шеврон вниз) |
| `multi_select` | Множественный выбор (галочки) |

### 7. Переменные подписи

Переменные подписи — это **inline-чипы**, работающие точно так же, как и обычные переменные, и вставляемые в любое место документа по отдельности. Каждая подпись в шаблоне имеет локальный порядковый номер (queue). Для каждой подписи доступно пять переменных:

| Переменная | Пример отображения в чипе |
|------------|--------------------------|
| Отдел подписанта | `[Отдел подписанта #1]` |
| Должность подписанта | `[Должность подписанта #1]` |
| ФИО подписанта | `[ФИО подписанта #1]` |
| Подпись | `[Подпись #1]` |
| Дата подписания | `[Дата подписания #1]` |

Пример использования в документе (пользователь верстает структуру подписи вручную):

```
Лечащий врач:

Отдел:    [Отдел подписанта #1]
Должность:[Должность подписанта #1]
          [ФИО подписанта #1]

Подпись:  [Подпись #1]     Дата: [Дата подписания #1]
```

- Чипы переменных подписи по визуальному стилю идентичны обычным переменным, но имеют отличительный оттенок или иконку для различения.
- При hover — tooltip с техническим именем переменной (например, `signs__signer_fullname__1`).
- При клике — выбирается, Delete/Backspace удаляет из документа.
- Удаление чипа из документа не удаляет саму подпись из шаблона.

**Визуальные состояния чипа переменной подписи:**

| Состояние | Описание |
|-----------|----------|
| default | Светло-фиолетовый или сиреневый фон, тёмный текст |
| hover | Чуть темнее фон, tooltip с техническим именем |
| selected | Обводка акцентного цвета |

### 8. Формула

Формула отображается как **inline-чип** внутри текста — аналогично переменной, но с визуально отличимым цветом и иконкой.

```
Итого: [Σ Сумма по назначениям]  Средний показатель: [Σ Среднее ИМТ]
```

- Чип имеет цвет, отличный от переменных, полей ввода и подписей (например, светло-оранжевый или бирюзовый).
- Иконка `Σ` или аналогичная перед названием.
- Внутри чипа — название формулы (`Formula.name`).
- При hover — tooltip с техническим идентификатором (например, `formula__3`).
- При клике — выбирается, Delete/Backspace удаляет из документа.
- Удаление чипа не удаляет саму формулу из системы.

**Визуальные состояния чипа формулы:**

| Состояние | Описание |
|-----------|----------|
| default | Светло-оранжевый/бирюзовый фон, тёмный текст |
| hover | Чуть темнее фон, tooltip с техническим именем |
| selected | Обводка акцентного цвета |

### 9. Результат по условию

Результат по условию отображается как **inline-чип** — аналогично формуле, но с другим обозначением.

```
Статус: [? Определение группы риска]  Рекомендация: [? Тип лечения]
```

- Чип имеет собственный цвет (например, светло-жёлтый или персиковый).
- Иконка `?` или `if` перед названием.
- Внутри чипа — название результата по условию (`Condition.name`).
- При hover — tooltip с техническим идентификатором (например, `condition__1`).
- При клике — выбирается, Delete/Backspace удаляет из документа.
- Удаление чипа не удаляет сам результат по условию из системы.

**Визуальные состояния чипа результата по условию:**

| Состояние | Описание |
|-----------|----------|
| default | Светло-жёлтый/персиковый фон, тёмный текст |
| hover | Чуть темнее фон, tooltip с техническим именем |
| selected | Обводка акцентного цвета |

## Дропдаун "{{ }} Переменная ▾" — список переменных

При нажатии открывается выпадающая панель с поиском и сгруппированным списком.

```
┌─────────────────────────────────┐
│  🔍 Поиск переменной...         │
├─────────────────────────────────┤
│  📅 Дата и время                │
│    ○ Текущая дата               │
│    ○ Текущее время              │
├─────────────────────────────────┤
│  🏥 Клиника и филиал            │
│    ○ Название клиники           │
│    ○ Логотип клиники            │
│    ○ Название филиала           │
│    ○ ...                        │
├─────────────────────────────────┤
│  👨‍⚕️ Заполняющий                  │
│    ○ Фамилия заполняющего       │
│    ○ Имя заполняющего           │
│    ○ Отчество заполняющего      │
│    ○ Специализация              │
│    ○ Категория                  │
│    ○ Отдел                      │
│    ○ Должность                  │
├─────────────────────────────────┤
│  🧑 Пациент                     │
│    ○ Фамилия пациента           │
│    ○ Имя пациента               │
│    ○ Отчество пациента          │
│    ○ Дата рождения пациента     │
│    ○ Пол пациента               │
│    ○ Адрес пациента             │
│    ○ Группа крови (ABO)         │
│    ○ Резус-фактор (Rh)          │
│    ○ Рост пациента              │
│    ○ Вес пациента               │
│    ○ ИМТ пациента               │
│    ○ Аллергии пациента          │
│    ○ Хронические заболевания    │
└─────────────────────────────────┘
```

- Список разбит на группы, структура которых приходит с бэкенда. Вычисляемые переменные (дата/время, клиника, заполняющий, пациент) входят в группу «Основные» и загружаются при инициализации. Статичные переменные (данные из таблиц БД) группируются по названию таблицы и подгружаются при раскрытии группы (аккордион с lazy-loading).
- Группа "Дата и время" содержит два пункта: "Текущая дата" и "Текущее время" (отдельно).
- Группа "Заполняющий" не содержит "Подпись" — подписи управляются отдельным инструментом.
- Внутри каждой раскрытой статичной группы — поиск по данным.
- Поиск фильтрует по названию переменной (не по техническому имени).
- Клик на элемент списка вставляет чип переменной в позицию курсора.
- Панель закрывается после вставки.

## Дропдаун "Поле ввода ▾"

```
┌─────────────────────────────────┐
│  [+ Создать новое поле ввода]   │
├─────────────────────────────────┤
│  🔍 Поиск существующего поля... │
├─────────────────────────────────┤
│  📝 Диагноз пациента  (input)   │
│  📋 Тип приёма (select)         │
│  # Количество сеансов           │
│  ...                            │
└─────────────────────────────────┘
```

- Первая опция в дропдауне — кнопка "Создать новое поле ввода", открывающая модальное окно.
- Ниже — список уже созданных полей ввода с поиском по названию.
- Клик на существующее поле вставляет его чип в позицию курсора.

## Дропдаун "✏ Подпись ▾"

При нажатии открывается панель со списком подписей, добавленных в текущий шаблон. Каждая подпись раскрывается и показывает свои пять переменных — каждую можно вставить в документ по отдельности, как обычную переменную.

```
┌──────────────────────────────────────────┐
│  Подписи в шаблоне                       │
├──────────────────────────────────────────┤
│  ▼ #1  Иванов И.И.  (Хирург)             │  ← раскрытая подпись
│    ○ Отдел подписанта #1                 │  ← клик вставляет чип
│    ○ Должность подписанта #1             │
│    ○ ФИО подписанта #1                   │
│    ○ Подпись #1                          │
│    ○ Дата подписания #1                  │
├──────────────────────────────────────────┤
│  ▶ #2  —  (Главный врач)                 │  ← свёрнутая подпись
├──────────────────────────────────────────┤
│  [+ Добавить новую подпись]              │
└──────────────────────────────────────────┘
```

- Каждый пункт списка показывает: порядковый номер, имя сотрудника (если указан) или прочерк, должность, отдел.
- Подпись можно раскрыть (▼) и свернуть (▶).
- Клик на любую переменную из раскрытого списка вставляет её inline-чип в позицию курсора.
- Одну и ту же переменную подписи можно вставить в несколько мест документа.
- Кнопка "Добавить новую подпись" открывает модальное окно создания подписи.

## Дропдаун "Σ Формула ▾"

При нажатии открывается панель со списком формул, доступных в системе.

```
┌──────────────────────────────────────────┐
│  [+ Добавить новую формулу]              │
├──────────────────────────────────────────┤
│  🔍 Поиск формулы...                     │
```

## Состояния редактора

### empty (пустой шаблон)

- Рабочая область показывает плейсхолдер: иконка документа + текст "Шаблон пустой. Начните добавлять контент."
- Toolbar активен — пользователь может сразу начать работу.

### active (в процессе редактирования)

- Стандартное состояние редактора с курсором в тексте.
- Активные кнопки toolbar отражают форматирование в позиции курсора.

### loading (загрузка изображения)

- В месте вставки изображения — скелетон или spinner.
- Toolbar временно недоступен для действий с изображением, пока оно загружается.

### error (ошибка загрузки изображения)

- На месте изображения — иконка ошибки + текст "Не удалось загрузить изображение. Попробуйте снова."
- Кнопка повтора загрузки.

### read-only (режим просмотра)

- Если шаблон открыт в режиме просмотра (не редактирования) — toolbar скрыт.
- Переменные, поля ввода, подписи, формулы и условия отображаются как чипы, но не кликабельны.
- Курсор — default, не текстовый.

## Взаимодействия и анимации

- Открытие дропдаунов toolbar: появление с лёгким fade + scale (100–150 мс).
- Вставка чипа любого типа: чип появляется в точке курсора без прыжка текста.
- Закрытие модального окна: fade-out (150 мс).
- Hover-состояние кнопок toolbar: смена фона (100 мс).
- Активное состояние toggle-кнопок: залитый/иной фон.
- Появление resize handles у изображения: fade-in (100 мс).
- Hover на + добавить строку/столбец в таблице: плавное появление кнопки.
- Hover на блок подписи: плавная подсветка рамки (100 мс).

## Визуальные правила

1. Все пять типов чипов (переменная, поле ввода, подпись, формула, условие) — разные цвета, чтобы пользователь сразу различал их тип.
2. Переменные: акцентный холодный цвет (голубой, лавандовый).
3. Поля ввода: тёплый или нейтральный акцентный цвет (зелёный, янтарный).
4. Переменные подписи: сиреневый/фиолетовый.
5. Формулы: бирюзовый или светло-оранжевый.
6. Результаты по условию: светло-жёлтый или персиковый.
7. Чипы не должны выглядеть кликабельными ссылками — это встроенные элементы текста.
8. Toolbar: компактный, без лишних подписей. Иконки + tooltip при hover.
9. "Лист" документа: белый фон, тень, отступы имитируют страницу A4.
10. Общий фон редактора: нейтральный серый (#F5F5F5 или аналог).
11. Все состояния hover и focus — плавные, без резких изменений.
12. Таблицы в редакторе — с видимой рамкой (для работы с ними). В итоговом документе рамка управляется шаблоном.
13. Шрифт по умолчанию в рабочей области — системный шрифт документов клиники или стандартный sans-serif.

## Acceptance criteria для дизайнера

- Toolbar сгруппирован по смысловым блокам с разделителями.
- Все инструменты toolbar имеют иконки и tooltip-подписи.
- Чипы всех пяти типов визуально различимы по цвету (переменная, поле ввода, подпись, формула, условие).
- Hover-состояние чипа показывает tooltip с техническим именем/типом.
- Дропдаун переменных разбит на группы с поиском; содержит "Текущая дата" и "Текущее время" как отдельные пункты; группа "Заполняющий" не содержит "Подпись" — подписи управляются отдельным инструментом; группа "Пациент" содержит "Рост пациента", "Вес пациента" и "ИМТ пациента" и не содержит "Родственники пациента"; статичные группы раскрываются по аккордиону с подгрузкой данных.
- Дропдаун "Поле ввода ▾" объединяет создание нового и выбор существующего поля.
- Дропдаун "Подписи ▾" отображает список подписей шаблона с раскрытием 5 переменных каждой и кнопку добавления новой.
- Дропдаун "Σ Формула ▾" отображает список формул с поиском и кнопку добавления новой.
- Дропдаун "? Условие ▾" отображает список результатов по условию с поиском и кнопку добавления нового.
- Клик на переменную подписи в дропдауне вставляет её как inline-чип в позицию курсора.
- Модальное окно создания подписи содержит поля: Отдел (обязательно), Должность (обязательно), Сотрудник (необязательно).
- Модальное окно создания формулы содержит: поле названия и inline-редактор выражения с поддержкой чипов переменных/полей/формул.
- Модальное окно создания результата по условию содержит блоки ЕСЛИ/ИНАЧЕ ЕСЛИ/ИНАЧЕ с возможностью добавлять/удалять ветки.

- Рабочая область визуально имитирует лист документа.
- Режим двух колонок отображает корректный разделённый макет.
- Пустое состояние редактора содержит понятный плейсхолдер.
- Состояние загрузки изображения имеет индикацию.
- Состояние ошибки загрузки изображения содержит понятное сообщение и действие.
- Режим read-only скрывает toolbar и делает все элементы некликабельными.
- Кнопки "Сохранить как шаблон" и "Отправить" появляются только при наличии контента в редакторе.
