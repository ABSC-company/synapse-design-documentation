# Page Header

## Описание компонента

**Page Header** — заголовочный блок, расположенный в верхней части контентной области страницы на всю ширину. Компонент разделён на две части:

- **Левая часть**: заголовок страницы с названием текущей секции или модуля.
- **Правая часть**: две основные кнопки-триггеры для управления пациентами и их визитами.

---

## Кнопка «Зарегистрировать пациента»

Предназначена для создания новой записи пациента в системе.

### Task Flow

#### Этап 1. Инициация регистрации

1. Пользователь нажимает кнопку **Зарегистрировать пациента** в Header.
2. Открывается модальное окно регистрации.
3. Пользователь выбирает тип гражданства:
   - Гражданин Республики Узбекистан
   - Иностранный гражданин

#### Этап 2. Поиск существующего пациента

Над компонентом Tab располагается **поле для поиска** пациента с дебаунсингом:

- **Для граждан РУз**: поиск по PINFL или FaceID
- **Для иностранных граждан**: поиск по MRZ (Machine Readable Zone)
  - Опционально: сканер документов для автоматического извлечения данных из MRZ
  - При сканировании автоматически заполняются связанные поля (страна, пол, фамилия, имя)

#### Этап 3. Заполнение данных пациента

Форма содержит три раздела, доступные через компонент **Tab**:

##### **Раздел 1: Персональные данные** (обязательно)

- **MRZ паспорта** (опционально) — автозаполняется при сканировании
- **Идентификационный номер/Серия паспорта** (опционально)
  - Поддерживаемые типы: номер/серия паспорта, загран. паспорта, метрика
- **Ф.И.О** (обязательно) — три отдельных инпута: фамилия, имя, отчество
  - Отчество опционально
- **Дата рождения** (обязательно)
- **Пол** (обязательно)
- **Страна** (обязательно)
- **Город/Провинция** (обязательно)
- **Адрес** (обязательно для граждан РУз, опционально для иностранцев)
- **Контактная информация** (минимум 1)
  - Тип: номер телефона, почта, социальная сеть
  - Значение: номер, адрес почты, ссылка на профиль
- **Фото пациента** (опционально)
- **Категория пациента** (опционально) — выбор из предопределённого списка

##### **Раздел 2: Физиологические данные** (полностью опционально)

- **Рост**
- **Вес**
- **Индекс массы тела** — рассчитывается автоматически на основе роста и веса
- **Группа крови**

##### **Раздел 3: Особые данные** (полностью опционально)

- **Хронические заболевания**
- **Инвалидность**
- **Аллергии**

#### Этап 4. Проверка дубликатов

5. После заполнения формы система **автоматически выполняет поиск дубликатов**.
   - Поиск выполняется перед финальным подтверждением регистрации.

#### Этап 5. Подпись соглашения

6. Если дубликаты не найдены:
   - Система генерирует документ: **«Соглашение о обработке персональных данных»**
   - Пациент подписывает документ **графической подписью** (на планшете, смартфоне или ПК)

#### Этап 6. Завершение регистрации

7. После подписания соглашения:
   - Система подтверждает регистрацию
   - Создаётся ЭМК (электронная медицинская карта) пациента

#### Сценарии блокировки

8. Подтверждение регистрации блокируется в следующих случаях:
   - Обнаружен дубликат пациента в системе
   - Пациент не подписал соглашение в течение **10 минут**
   - В обоих случаях показывается **Toast-уведомление** об ошибке, регистрация не выполняется

## Кнопка «Запись пациента к врачу»

Предназначена для создания новой записи (визита) пациента к врачу.

### Task Flow

#### Этап 1. Инициация записи

1. Пользователь нажимает кнопку **Запись пациента к врачу** в Header.
2. Открывается модальное окно создания визита пациента.

#### Этап 2. Поиск пациента

3. Пользователь ищет пациента по:
   - Ф.И.О
   - ID пациента
   - PINFL
   - FaceID
   - QR-код
   - MRZ (если пациент — иностранный гражданин)

#### Этап 3. Обработка активных визитов

4. Система показывает **активные незавершённые визиты** пациента в клинику (если они существуют).

5. **Если выбран активный визит**:
   - Разблокируются формы визита к врачу
   - Разблокируется загрузка файлов

6. **Если активного визита нет или нужен новый**:
   - Визит в клинику формируется **автоматически** на основе данных из первого визита к врачу
   - Разблокируются формы визита к врачу
   - Разблокируется загрузка файлов

#### Этап 4. Заполнение формы визита

Пользователь заполняет **форму визита к врачу** с интеллектуальной фильтрацией:

- **Степень срочности (Триаж)** (обязательно)
  - Выбор из предопределённого списка приоритетов

- **Направление** (опционально)
  - При выборе направления — список врачей автоматически фильтруется под это направление
  - Влияет на доступный список врачей

- **Врач** (обязательно)
  - При выборе врача — направление подставляется автоматически на основе специальности врача
  - Влияет на доступный список услуг

- **Услуга** (обязательно)
  - После выбора врача список услуг фильтруется
  - Содержит только те услуги, которые может предоставить выбранный врач

- **Дата и время** (необязательно)
  - Выбор даты и времени визита к врачу (букинг)
  - Если не указывается система создаёт запись в ближайшее свободное время в графике приёмов

#### Этап 5. Сохранение записи

8. Пользователь сохраняет запись — визит к врачу создаётся в системе
