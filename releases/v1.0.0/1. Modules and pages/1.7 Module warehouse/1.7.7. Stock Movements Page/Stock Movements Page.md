#### «Перемещения»

- **Назначение**: единственный способ изменить остаток (`WMS-FR-04`); реализует также блокировку превышения остатка (`WMS-FR-06`).
- **Список документов**: *№, Тип (внутреннее / выдача в расход / возврат), Откуда, Куда, Статус, Дата создания, Инициатор*.
  - Фильтры: тип, статус, период, место (откуда/куда).
- **Статусы документа**: «Черновик» → «Отправлено» → «Принято» (или «Отклонено») → возможна «Отмена» из статусов «Черновик»/«Отправлено» (до приёмки получателем).
- **Карточка документа**: шапка (тип, откуда, куда, инициатор, дата, статус) + таблица строк (партия/товар, количество; при выдаче без явного указания партии — авто-подбор по FEFO, `WMS-FR-05`).
- **Кнопки по статусу и роли**:
  - «Отправить» — доступно инициатору из «Черновика», проверяет остаток в месте-источнике (блокирует превышение, `WMS-FR-06`);
  - «Принять» / «Отклонить» — доступно ответственному/МОЛ места-получателя из статуса «Отправлено»; «Принять» списывает остаток источника и зачисляет в место-получатель, публикует `WMS.ПеремещениеПринято` и `WMS.ОстатокИзменился`; «Отклонить» возвращает документ инициатору с комментарием причины;
  - «Отменить» — доступно инициатору, только пока документ не принят.
- **Валидация**: сумма количества по строке не превышает остаток партии в месте-источнике на момент отправки (повторная проверка при приёме — на случай параллельных операций).
- **Edge cases**: остаток в источнике изменился между «Черновиком» и «Отправить» — система пересчитывает доступное количество и предупреждает при недостаче.

**Примечания для дизайнера:**
- Кнопки «Принять»/«Отклонить»/«Отправить»/«Отменить» показываются по правилу «статус × роль» — на макете должно быть видно, что набор кнопок разный для инициатора и для получателя (два состояния карточки, не одно).
- Строка со статусом «Черновик» видна только инициатору (или показана как «в разработке»/приглушённая для остальных, если видимость по всей организации).
- Строки с авто-подбором по FEFO помечаются мелкой подписью «подобрано автоматически» рядом с номером партии — пользователь должен понимать, что партию выбрала система, а не он сам.
- «Отменить» — вторичная (текстовая/ghost) кнопка, «Отклонить» — предупреждающего цвета, «Принять»/«Отправить» — основная (primary) кнопка.
