#### Экран «Инвентаризация»

- **Назначение**: реализует `WMS-FR-22`, `WMS-FR-23`.
- **Основной вид**: карточки или таблица текущих и запланированных сессий; фильтры — место хранения, ответственный, статус.
- **Создание сессии**: кнопка «Создать сессию» → выбор места хранения/помещения, назначение ответственного (`HR`), период инвентаризации (даты начала/окончания), комментарий.
- **Статусы сессии**: «Запланирована» → «В процессе» → «Ожидает завершения» → «Завершена».
- **Внутри сессии**:
  - поле ввода/фокус для сканера;
  - список найденных объектов по мере сканирования: объект, ожидаемое место, результат сверки — «совпадает» / «расхождение» / «отсутствует»;
  - для расхождений — подсказка с типом: «найдено не в своём месте», «ожидаемое место пустое» (объект учтён, но не найден при сканировании), «объект не найден в системе» (отсканирован штрихкод, которого нет в учёте).
- **Завершение сессии**: действие «Завершить сессию» формирует отчёт по расхождениям — объекты не на своём месте, не найденные объекты, учтённые несоответствия; действия «Сохранить результат», «Распечатать отчёт», «Перейти к объекту» (открывает карточку партии/ТМЦ).
- **Уведомления о необходимости инвентаризации**: складской руководитель задаёт сроки и периодичность (см. «Настройки склада»); система присылает напоминания в интерфейсе и/или десктоп/почтовые уведомления (`NOTIF`) за заданный срок до начала. Отображаются на главном экране и в панели уведомлений: «Инвентаризация места X должна быть проведена до DD.MM.YYYY» и, отдельно, «Инвентаризация просрочена».
- **Открытый вопрос** (см. «Принятые решения»): не определён источник истины при расхождении (данные учёта vs. фактическое расположение) и процесс разбора/корректировки расхождений после завершения сессии — до уточнения отчёт носит информационный характер и не меняет остатки/местоположения автоматически.

┌────────────────────────────────────────────────────────────┐

│  [ 🔦 Поле сканера: ________________ ]                       │
├──────────────────────┬──────────────┬───────────────────────┤
│ Объект               │ Ожид. место  │ Результат             │
├──────────────────────┼──────────────┼───────────────────────┤
│ A-1042 Бинт стер.    │ Стеллаж А1   │ 🟢 Совпадает          │

**Примечания для дизайнера:**
- Поле сканера — всегда в фокусе на экране сессии (автофокус при открытии/после каждого скана), крупное, визуально выделено как «активный ввод».
- Результаты сверки добавляются в таблицу в реальном времени сверху вниз (последний скан — первая строка или подсвечен), чтобы кладовщик видел немедленную обратную связь по каждому скану.
- Отчёт по завершении сессии — отдельный экран/PDF-preview с тремя явными блоками: «Не на своём месте», «Не найдены», «Учтённые несоответствия» — не единый список, три визуально разделённых секции.
- Напоминания на главном экране/в панели уведомлений — текст-шаблон фиксирован («Инвентаризация места X должна быть проведена до DD.MM.YYYY» / «Инвентаризация просрочена») — не перефразировать в макетах.
