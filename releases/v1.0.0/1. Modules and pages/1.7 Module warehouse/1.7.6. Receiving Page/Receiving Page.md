### «Приёмка»

- **Назначение**: фиксация поступления партии на склад; реализует `WMS-FR-19` (штрихкод при приёмке), `WMS-FR-07` (проверка условий хранения).
- **Форма**:
  - поставка (select из внешней части/модуля закупок, если интегрирован) — при выборе автоподставляются товар(ы) поставки;
  - товар (авто по поставке, либо ручной выбор из справочника «Товары»);
  - номер партии (текст; обязателен, если у товара включён «партионный учёт»);
  - количество (обязательно, > 0);
  - срок годности (обязателен, если у товара включён «контроль срока годности»);
  - место хранения (select) — при выборе система сверяет условия места (темп/влажность) с требованиями товара:
    - режим «блокирующий» (настройка склада) — сохранение запрещено при несоответствии, ошибка с указанием причины;
    - режим «предупреждающий» — сохранение возможно, но показывается предупреждение и запись помечается для последующего контроля.
  - Кнопка «Оприходовать».
- **После сохранения**: система создаёт партию, генерирует уникальный штрихкод/QR (`WMS-FR-19`), публикует `WMS.ПартияПринята`; на экране подтверждения доступно действие «Печать этикетки» (`WMS-FR-20`).
- **Валидация**: количество > 0; номер партии уникален в рамках товара (если партионный учёт включён); срок годности не может быть в прошлом.
- **Edge cases**: повторная приёмка по той же поставке — система предупреждает о возможном дубле; отмена приёмки до подтверждения — без последствий для остатков.

**Wireframe:**
```
┌──────────────────────────────────────────────┐
│  Приёмка товара                          [✕] │
├──────────────────────────────────────────────┤
│  Поставка *      [Накладная №4412 ▾]          │
│  Товар           Бинт стерильный  (авто)      │
│  Номер партии *  [A-1042________]             │
│  Количество *    [120___] шт                  │
│  Срок годности * [01.10.2026]                 │
│  Место хранения *[Склад №1 / Стеллаж А1 ▾]    │
│  ⚠ Условия места (2–8°C) не соответствуют      │
│     требованиям товара (15–25°C)               │
├──────────────────────────────────────────────┤
│                    [Отмена]     [Оприходовать]│
└──────────────────────────────────────────────┘

После сохранения:
┌──────────────────────────────────────────────┐
│  ✅ Партия A-1042 оприходована                 │
│  Штрихкод: [QR-код превью]      [🖨 Печать этикетки]│
└──────────────────────────────────────────────┘
```

**Примечания для дизайнера:**
- Предупреждение о несоответствии условий хранения — инлайн-баннер жёлтого (режим «предупреждающий») или красного (режим «блокирующий», кнопка «Оприходовать» задизейблена) цвета сразу под полем «Место хранения», не в toast — пользователь должен увидеть его до попытки сохранить.
- Экран подтверждения после сохранения показывает превью сгенерированного QR и кнопку печати этикетки как основной следующий шаг — не закрывать форму молча.
- Поля, зависящие от настроек товара (партионный учёт / контроль срока), которые неприменимы — скрываются целиком, а не показываются disabled.
