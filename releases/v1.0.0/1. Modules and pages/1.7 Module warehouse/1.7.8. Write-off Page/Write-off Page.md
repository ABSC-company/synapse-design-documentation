#### «Списание»

- **Назначение**: реализует `WMS-FR-18` (обязательная причина), `WMS-FR-05` (подбор по FEFO), а также сценарий согласованного списания по сроку годности (см. «Принятые решения»).
- **Форма**:
  - партия (select; при выборе товара без явного указания партии система подсказывает кандидатов по FEFO/ближайшему сроку истечения);
  - количество (обязательно, ≤ остаток партии);
  - причина (select: истёк срок годности / порча / недостача / брак / прочее) — обязательное поле (`WMS-FR-18`);
  - комментарий (текст, обязателен при причине «прочее»).
  - Кнопка «Списать».
- **Особый сценарий — истечение срока годности**: партия, помеченная системой как «истекла», не может быть списана в одностороннем порядке автоматически — система отправляет уведомление ответственному лицу/МОЛ и требует его подтверждения в интерфейсе (кнопка «Подтвердить списание») перед проведением документа.
- **После проведения**: остаток партии уменьшается, публикуется `WMS.ТоварСписан`; операция отражается в «Аналитике склада» (топ причин списаний, `WMS-FR-13`).
- **Валидация**: количество > 0 и ≤ текущий остаток партии; причина обязательна.
- **Edge cases**: попытка списать партию, уже полностью выданную другим документом перемещения между открытием формы и сохранением — ошибка «Остаток изменился, обновите форму».

│             A-1015 (12.08)                     │

┌──────────────────────────────────────────────┐

**Примечания для дизайнера:**
- Выпадающий список партии сразу показывает подсказку с кандидатами по FEFO под полем (не отдельным экраном) — ближайшая по сроку партия визуально выделена (например, жирным/первой в списке).
- Блок подтверждения по истёкшей партии — отдельный, визуально более «тяжёлый» шаг (акцентная рамка/иконка ⚠), кнопка «Списать» в этом случае недоступна, пока не нажато «Подтвердить списание».
- Поле «Комментарий» становится обязательным (звёздочка появляется динамически) только при выборе причины «Прочее» — показать это состояние на макете отдельно.
