### Функциональные требования

| ID | Требование | Приоритет |
|---|---|---|
| `WMS-FR-01` | Система хранит карточку товара с метриками: наименование, описание, упаковка, единица измерения, условия хранения, группа | must |
| `WMS-FR-02` | Система хранит партии товара отдельно от карточки товара, с количеством и сроком годности на партию | must |
| `WMS-FR-03` | Система ведёт остаток товара в разрезе «партия × место хранения» | must |
| `WMS-FR-04` | Изменение остатка возможно только через документ перемещения | must |
| `WMS-FR-05` | При выдаче/списании без явного указания партии система подбирает партию по FEFO | must |
| `WMS-FR-06` | Система блокирует перемещение/выдачу, превышающую остаток партии в месте-источнике | must |
| `WMS-FR-07` | Система проверяет соответствие условий хранения места требованиям товара при размещении партии | should |
| `WMS-FR-08` | Партия с истёкшим сроком годности недоступна к выдаче в расход | must |
| `WMS-FR-09` | Система переводит партию в статус «на исходе срока» за настраиваемое число дней до истечения и публикует событие | should |
| `WMS-FR-10` | Система хранит карточку поставщика и связанные с ним договоры поставки | must |
| `WMS-FR-11` | Договор поставки содержит перечень номенклатуры, поставляемой по нему | should |
| `WMS-FR-12` | Система формирует отчёты по расходу товара в разрезе подразделений и периодов | should |
| `WMS-FR-13` | Система формирует отчёты по списаниям (истёкший срок, порча, недостача) | should |
| `WMS-FR-14` | Система ведёт учёт ТМЦ отдельно от расходного товара, с закреплением за подразделением/сотрудником | could |
| `WMS-FR-15` | Система строит прогноз потребности на основе истории расхода | could |
| `WMS-FR-16` | Инвентарный номер ТМЦ уникален в системе | must |
| `WMS-FR-17` | Порог перевода партии в статус «на исходе срока» настраивается администратором склада | should |
| `WMS-FR-18` | Причина обязательна при создании документа списания | must |
| `WMS-FR-19` | Система генерирует уникальный штрихкод для партии при приёмке и для ТМЦ при постановке на учёт | must |
| `WMS-FR-20` | Система поддерживает печать этикетки со штрихкодом на принтер этикеток из карточки партии/ТМЦ и из формы приёмки | should |
| `WMS-FR-21` | Система находит объект (партию/ТМЦ) по отсканированному штрихкоду и отображает его учётное местоположение и ответственного | must |
| `WMS-FR-22` | Система поддерживает сессию инвентаризации по месту хранения: последовательное сканирование объектов со сверкой факта с учётом | should |
| `WMS-FR-23` | По завершении сессии инвентаризации система формирует отчёт по расхождениям (не на своём месте / не найден) | should |
