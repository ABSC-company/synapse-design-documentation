
 ### Общие UI-паттерны и обозначения

> Зафиксированы один раз, дальше разделы 7.1–7.14 ссылаются на них без повторного описания механики (по аналогии с паттерном `1.6.0. Overview` в HR-модуле).

**Типы экранов модуля:**
- **Реестр** — таблица + фильтры + действия строки (Товары, Поставщики, Места хранения, Партии и остатки, Перемещения, ТМЦ, Инвентаризация).
- **Карточка / форма** — просмотр и редактирование одной сущности, открывается как отдельная страница или slide-over панель справа (Товар, Поставщик, Место хранения, ТМЦ, документ Перемещения).
- **Одношаговая форма-действие** — короткая форма без вкладок, модальное окно или встроенный блок (Приёмка, Списание).
- **Дашборд** — KPI-плитки + виджеты (Обзор склада, Аналитика склада).
- **Сессия** — пошаговый процесс с явным началом/концом и живым логом событий (Инвентаризация, Сканирование).

**Цветовая семантика статус-бейджей** (единая для всего модуля, не изобретать новые цвета на экран):
| Цвет | Значения |
|---|---|
| Зелёный | «в наличии», «активен», «завершена», «принято», «совпадает» (сверка при инвентаризации) |
| Жёлтый | «на исходе срока», «черновик», «в процессе», «ожидает завершения», предупреждение по условиям хранения (не блокирует) |
| Красный | «истекла», «отклонено», «просрочено», «расхождение», нарушение условий хранения (блокирующий режим), «не найден» |
| Серый | «архивный», «неактивен», «отменено», «в ремонте» (нейтральный, не ошибка) |

**Иконки разделов сайдбара** (для единообразия в макетах): 🏠 Обзор склада · 📦 Товары · 🏭 Поставщики · 📍 Места хранения · 🧾 Партии и остатки · 📥 Приёмка · 🔁 Перемещения · 🗑️ Списание · 🔧 ТМЦ · 📊 Аналитика склада · 📋 Инвентаризация · ⚙️ Настройки склада.

**Компоненты и их назначение:**
- **Drawer (выезд справа)** — фильтры реестров; счётчик активных фильтров — бейдж на кнопке «Фильтр».
- **Slide-over панель / отдельная страница** — карточки сущностей с вкладками (Поставщик → «Договоры», ТМЦ → «История», Партия → «История движений»).
- **Модальное окно** — короткие точечные действия: «Назначить МОЛ», «Создать договор», подтверждения необратимых операций («Списать», «Подтвердить списание», «Архивировать»).
- **Toast** — результат фоновых операций (сохранение, ошибка валидации, печать этикетки поставлена в очередь).
- **Пустые состояния** — иконка раздела (см. выше) + текст + основной CTA-кнопкой того же смысла, что и в шапке раздела (например, «Создать товар»).

**Права доступа и видимость:**
- Элементы, недоступные по роли, **скрываются**, а не просто блокируются — так же, как в HR-модуле (пример: «Настройки склада» не отображается в сайдбаре пользователю без роли «администратор склада»).
- Исключение — необратимые действия, которые полезно показать в disabled-состоянии с подсказкой-тултипом о недостающем праве (например, «Подтвердить списание» видно МОЛ, но неактивно до наступления события).

**Общая палитра ролей, встречающихся в подсказках/тултипах:** кладовщик, администратор склада, складской руководитель, МОЛ (материально ответственное лицо) — раздел `AUTH`/`HR` определяет конкретные права, здесь фиксируется только видимость на уровне UI.
