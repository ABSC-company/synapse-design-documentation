#### «Аналитика склада»

- **Назначение**: реализует `WMS-FR-12` (отчёты по расходу), `WMS-FR-13` (отчёты по списаниям), `WMS-FR-15` (прогноз потребности).
- **Дашборд-виджеты**:
  - «Расход по подразделениям» — столбчатая диаграмма, подразделение × объём расхода за период;
  - «Расход по периодам» — линейный график динамики расхода;
  - «Оборачиваемость товара» — таблица/график (товар, средний остаток, скорость расхода, дней запаса);
  - «Топ причин списаний» — диаграмма/таблица по причинам (истёк срок, порча, недостача, брак, прочее) с долями;
  - «Прогноз потребности» (`WMS-FR-15`, could) — таблица «товар → рекомендуемое количество к заказу» на основе истории расхода.
- **Фильтры**: период (диапазон дат), подразделение, группа товара.
- **Экспорт**: кнопка «Экспортировать отчёт» — формирует файл (стык с `RPT`/`FILE`), доступные форматы — как в общем паттерне экспорта отчётов системы.
- **Состояния**: Loading — скелетоны виджетов; Empty — «Недостаточно данных за выбранный период» на конкретном виджете (не блокирует остальные).

**Wireframe:**
```
┌────────────────────────────────────────────────────────────────────┐
│  Аналитика склада      [Период ▾] [Подразделение ▾] [Группа ▾] [Экспорт]│
├───────────────────────────────┬──────────────────────────────────────┤
│ Расход по подразделениям       │ Расход по периодам                   │
│ ▇▇▇▇▇  Хирургия                │  ╱╲___╱╲                             │
│ ▇▇▇    Терапия                 │                                       │
├───────────────────────────────┼──────────────────────────────────────┤
│ Оборачиваемость товара          │ Топ причин списаний                  │
│ Товар │Ост.│Расх/дн│Дней запаса│ ◐ Истёк срок 45% ◑ Порча 30% ...     │
├───────────────────────────────┴──────────────────────────────────────┤
│ Прогноз потребности: Товар → рекомендуемое количество                │
└────────────────────────────────────────────────────────────────────┘
```

**Примечания для дизайнера:**
- Виджеты — независимая загрузка/ошибка на каждом (один сломанный виджет не блокирует остальные три).
- Палитра графиков — по общей палитре дизайн-системы (категориальные цвета для подразделений/причин), см. общий гайд по цветам для графиков, не изобретать локально для WMS.
- «Прогноз потребности» помечается как «could»-функциональность — на макете может идти отдельным, визуально второстепенным блоком/вкладкой, если аналитика перегружена.
