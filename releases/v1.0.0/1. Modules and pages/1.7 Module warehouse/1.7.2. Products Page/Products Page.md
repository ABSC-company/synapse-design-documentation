### Справочник «Товары»

- **Назначение**: номенклатура расходных товаров, отдельная от ТМЦ (см. «Принятые решения»); реализует `WMS-FR-01`.
- **Список**: таблица *Наименование, Группа, Упаковка, Ед. измерения, Партионный учёт (да/нет), Контроль срока годности (да/нет), Статус (активен/архивный)*.
  - Фильтры: группа товара (select), статус, переключатели «только с партионным учётом», «только с контролем срока годности», поиск по наименованию.
  - Действия строки: открыть карточку, архивировать/восстановить (архивный товар недоступен для выбора в новых документах приёмки, но остаётся видимым в истории).
- **Карточка товара**:
  - Наименование (обязательно), описание, группа (select, справочник групп товара);
  - упаковка: кратность (число) + тип упаковки (select — коробка, флакон, уп. и т.д.);
  - единица измерения (select);
  - условия хранения: температура мин/макс, влажность мин/макс;
  - переключатель «партионный учёт» — включает обязательность номера партии и срока годности при приёмке (`WMS-FR-02`);
  - переключатель «контроль срока годности» — включает проверку статуса «на исходе срока» / «истёк» (`WMS-FR-08`, `WMS-FR-09`);
  - статус (активен/архивный).
  - **Валидация**: наименование обязательно; темп_мин ≤ темп_макс; влажность_мин ≤ влажность_макс; при включённом «контроль срока годности» поле «срок годности» становится обязательным в форме приёмки для этого товара.
- **Действия**: «Создать товар», «Редактировать», «Архивировать». Удаление физическое не предусмотрено — только архивация (товар мог использоваться в исторических документах).
- **Состояния**: Empty («Справочник пуст» + «Создать товар»), Error при сохранении — toast с указанием невалидного поля.

**Примечания для дизайнера:**
- Переключатели «партионный учёт» и «контроль срока годности» — крупные, с коротким пояснением под ними (что именно включают), т.к. это влияет на обязательность полей в других формах (Приёмка) — пользователь должен понимать последствия до сохранения.
- Архивный товар — строка таблицы приглушена (серый текст), бейдж «Архивный» серый; не показывается в select при создании документа приёмки, но остаётся в фильтрах истории.
- Валидация темп_мин ≤ темп_макс — инлайн-ошибка под полем, а не только при сабмите.
