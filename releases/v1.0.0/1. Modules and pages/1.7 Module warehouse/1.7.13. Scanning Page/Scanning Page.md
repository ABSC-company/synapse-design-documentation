#### «Сканирование»

- **Назначение**: отдельная полноэкранная страница для сценария быстрого поиска объекта по штрихкоду (`WMS-FR-21`); механика самого сканирования и карточки результата идентична панели сканера в шапке (см. «1.8.0.7. Barcode Scanner Panel») — здесь та же логика разворачивается на всю страницу с журналом сканирований за сессию.
- **Открытие страницы**: по клику на иконку сканера в шапке (доступна глобально, с любого экрана модуля) либо по горячей клавише; страница не вынесена отдельным пунктом в сайдбар — это вспомогательный, а не основной раздел учёта.
- **Основной блок**: поле ввода/фокус для сканера (автофокус при открытии страницы и после каждого скана) + карточка последнего найденного объекта (наименование, учётное место, закреплён за, действие «Печать этикетки», `WMS-FR-20`) либо нейтральное состояние «Объект не найден».
- **Журнал сканирований страницы** (отличие от компактной панели в шапке, которая показывает только последний результат): список всех сканирований за текущую сессию страницы — время, штрихкод, результат («Найден» / «Не найден»), сверху вниз (последний скан — первая строка); клик по строке повторно открывает карточку результата.
- **Переход к инвентаризации**: действие «Начать сессию инвентаризации» ведёт в раздел «Инвентаризация» (см. «1.8.12. Stocktake Page») для создания/продолжения сессии по месту хранения; режим «Инвентаризация по месту» (Режим 2 панели сканера) выполняется внутри этого раздела, а не на странице «Сканирование» — здесь только точка входа.
- **Очистка журнала**: действие «Очистить журнал» сбрасывает список сканирований страницы; это чисто UI-журнал текущей сессии, на данные учёта не влияет.

**Wireframe:**
```
┌────────────────────────────────────────────────────────────────┐
│  Сканирование                      [Начать сессию инвентаризации]│
├────────────────────────────────────────────────────────────────┤
│  🔦 Отсканируйте объект: [_______________________]                │
├────────────────────────────────────────────────────────────────┤
│  ✅ Найдено:                                                      │
│  A-1042 · Бинт стерильный                                        │
│  Учётное место: Склад №1 / Стеллаж А1                            │
│  Закреплён за: —                              [🖨 Печать этикетки]│
├────────────────────────────────────────────────────────────────┤
│  Журнал сканирований                                [Очистить]   │
│  09:41  A-1042            ✅ Найден                                │
│  09:39  ТМЦ-00231         🔴 Не найден                            │
│  09:35  A-0991            ✅ Найден                                │
└────────────────────────────────────────────────────────────────┘
```

**Примечания для дизайнера:**
- Поле сканера — всегда в фокусе, крупное, визуально выделено как «активный ввод» (тот же паттерн, что и у панели в шапке, см. 1.8.0.7).
- Журнал специфичен для этой полноэкранной страницы: панель в шапке журнал не показывает, только последний результат; журнал хранится в рамках открытой сессии страницы и не персистентен между визитами.
- Действие «Начать сессию инвентаризации» видно только пользователям с правом на инвентаризацию (см. правила видимости по ролям, §7.0 «Общие UI-паттерны») — при отсутствии права элемент скрывается, а не блокируется.
- «Не найдено» — нейтральный стейт (не ошибка сервера): жёлтая/серая иконка предупреждения, не красный крест — совпадает с поведением панели сканера.
