### Справочник «Поставщики»

- **Назначение**: реализует `WMS-FR-10`.
- **Список**: *Наименование, ИНН/УНН, Контактное лицо, Телефон, Статус*; фильтры — статус (активен/неактивен), поиск по наименованию/ИНН.
- **Карточка поставщика**:
  - основные поля (наименование, ИНН/УНН, контактное лицо, телефон, email, адрес, статус);
  - вкладка **«Договоры»** — таблица: *Номер, Дата заключения, Дата окончания, Статус (действует/истёк/расторгнут), Ссылка на файл*.
    - Действия во вкладке: «Создать договор» (номер, даты, прикрепить файл через `FILE`), «Привязать номенклатуру» — мультиселект из справочника «Товары» (реализует `WMS-FR-11`: договор содержит перечень поставляемой номенклатуры), «Прикрепить файл».
    - Договор с истёкшей датой окончания визуально помечается (жёлтый/серый бейдж «Истёк»), но не блокирует просмотр истории.
- **Валидация**: наименование и ИНН/УНН обязательны; дата окончания договора ≥ даты заключения.
- **Состояния**: Empty на вкладке «Договоры» — «Договоров пока нет» + «Создать договор».

**Wireframe:**
```
Карточка поставщика:
┌───────────────────────────────────────────────────────────┐
│  ООО «МедСнаб»                                        [✕] │
├───────────────────────────────────────────────────────────┤
│  [ Основное ]  [ Договоры (3) ]                            │
├───────────────────────────────────────────────────────────┤
│  Вкладка «Договоры»:                    [+ Создать договор]│
│  ┌───────────┬───────────┬───────────┬──────────┬────────┐│
│  │ Номер     │ Заключён  │ Окончание │ Статус   │ Файл   ││
│  ├───────────┼───────────┼───────────┼──────────┼────────┤│
│  │ Д-2025-014│ 01.02.2025│ 01.02.2026│●Действует│ 📎     ││
│  │ Д-2024-091│ 01.01.2024│ 01.01.2025│●Истёк    │ 📎     ││
│  └───────────┴───────────┴───────────┴──────────┴────────┘│
│  [Привязать номенклатуру]                                  │
└───────────────────────────────────────────────────────────┘
```

**Примечания для дизайнера:**
- Вкладки «Основное» / «Договоры» — счётчик количества договоров в скобках у названия вкладки.
- «Истёк» — серо-жёлтый бейдж (не красный: это не ошибка, а естественный жизненный цикл договора), «Расторгнут» — красный.
- «Привязать номенклатуру» открывает мультиселект с поиском по справочнику «Товары»; выбранные позиции показываются чипами под полем.
