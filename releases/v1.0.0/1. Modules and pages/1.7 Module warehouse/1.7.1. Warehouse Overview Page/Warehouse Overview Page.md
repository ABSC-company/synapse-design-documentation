### Главный экран склада («Обзор склада»)

- **Назначение**: единая точка входа при заходе в модуль — сводка состояния склада и быстрый доступ к рабочим операциям.
- **Верхние KPI-плитки**:
  - суммарный остаток (в единицах и/или в денежном выражении, если доступна оценка стоимости);
  - число партий с истекающим сроком (на исходе срока / уже истёкшие — раздельно);
  - количество незавершённых документов перемещений (по статусам «Черновик», «Отправлено», «В пути»);
  - число поставок, ожидающих приёмку.
  - Каждая плитка кликабельна и открывает соответствующий раздел с применённым фильтром.
- **Быстрые действия** (кнопки): «Начать приёмку» → форма приёмки; «Создать перемещение» → новый документ перемещения; «Открыть инвентаризацию» → раздел «Инвентаризация»; «Проверить список ТМЦ» → раздел «ТМЦ».
- **Лента последних событий** (хронологический список, с иконкой типа события и временной меткой): новые оприходованные партии, изменения статусов перемещений, завершённые списания, завершённые инвентаризационные отчёты с расхождениями. Клик по событию открывает связанный документ/объект.
- **Состояния**:
  - Loading — скелетоны KPI-плиток и ленты событий;
  - Empty — при отсутствии событий за период: «Событий пока нет»;
  - Error — блок ошибки с «Повторить», KPI-плитки не блокируют доступ к разделам меню.

**Примечания для дизайнера:**
- KPI-плитки — кликабельны целиком (не только число), курсор pointer, hover-подсветка.
- Лента событий — иконка события цветом соответствует таблице статус-семантики из §7.0 (🟢 приёмка/принято, 🔁 перемещение, 🗑️ списание, 📋 инвентаризация с расхождениями — жёлтый акцент на числе расхождений).
- Быстрые действия — всегда 4 кнопки в одной строке, порядок фиксирован (приёмка → перемещение → инвентаризация → ТМЦ), не переставлять между релизами макета.
