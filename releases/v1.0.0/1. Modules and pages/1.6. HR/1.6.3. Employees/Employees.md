# 1.6.3 Employees

## Спецификация страницы интерфейса

* **Название страницы:** Сотрудники
* **Модуль:** HR (Human Resources)
* **Тип страницы:** Реестр (таблица) + точка входа в электронное дело
* **Основные роли:** Администратор / HR-менеджер, Руководитель отдела
* **Права доступа:** `HR_EMPLOYEE_GET` (просмотр), `HR_EMPLOYEE_CREATE` (приём), `HR_EMPLOYEE_DELETE` (удаление/bulk), `HR_ARCHIVE_MANAGE` (увольнение), `HR_TIME_TRACKING_GET` (учёт времени)
* **Маршрут:** `/hr/employees`
* **Платформа:** веб-приложение медицинской системы
* **Статус документа:** Draft

> Общие паттерны (реестр, форма, статус-бейдж, массовый выбор, контекстное меню, объективка) — см. `1.6.0. Overview`.

---

## Назначение страницы и сценарии использования

Табличный реестр всех сотрудников с базовой информацией и точкой входа в электронное дело (1.6.4). Отсюда выполняются приём, массовые операции и быстрые действия (объективка, учёт времени, увольнение). Уволенные (`dismissed`) физически в архиве и в основном списке не показываются — доступны через фильтр/раздел «Архив».

С помощью страницы пользователь может:

1. Просматривать сотрудников: ФИО, отдел, должность, статус, график работы.
2. Искать по ФИО, фильтровать (отдел, должность, пол, возраст, гражданство, семейное положение, системная роль, статус и т.д.).
3. Открывать электронное дело сотрудника.
4. Принимать нового сотрудника.
5. Массово выбирать строки (чекбокс) и удалять (bulk) через Action Bar.
6. Через контекстное меню (`⋮` / ПКМ): открыть эл. дело, уволить, скачать/распечатать объективку, скачать/распечатать учёт времени (check in-out).

## User Flows / Task Flows

### Flow 1. Просмотр, поиск, фильтрация

1. Пользователь открывает «Сотрудники».
2. Система показывает таблицу с пагинацией.
3. Пользователь вводит запрос в поиск → фильтрация по ФИО.
4. Пользователь открывает фильтры, задаёт параметры, применяет → таблица обновляется, на кнопке «Фильтр» счётчик активных.

### Flow 2. Открытие электронного дела

1. Пользователь кликает строку (вне чекбокса/`⋮`) → переход в эл. дело (1.6.4).

### Flow 3. Приём сотрудника

1. Пользователь нажимает «+ Принять сотрудника».
2. Открывается форма создания.
3. Заполняет обязательные поля (включая сканирование документа MRZ/паспорт, отдел/должность), сохраняет.
4. Система создаёт сотрудника (`at_work`) и переходит в его эл. дело.

### Flow 4. Массовое удаление (bulk)

1. Пользователь отмечает чекбоксы строк.
2. Снизу появляется Action Bar: «Выбрано: N» + «Удалить» + «Снять выбор».
3. «Удалить» → подтверждение «Удалить N сотрудников?» → удаление.

### Flow 5. Быстрые действия из контекстного меню

1. Пользователь нажимает `⋮` или кликает ПКМ по строке.
2. Открывается контекстное меню.
3. Выбор действия: открыть дело / уволить / объективка / учёт времени.

```
│▢ │ ФИО                   │ Отдел    │ Должность │ Статус  │ График  │⋮│
├──┼───────────────────────┼──────────┼───────────┼─────────┼─────────┼┤
```

### Header
* Заголовок «Сотрудники» — слева.
* «+ Принять сотрудника» — справа, видна при `HR_EMPLOYEE_CREATE`.

### Панель управления
* **Статусные вкладки:** Все · На работе (`at_work`) · Больничный · Отпуск (`on_leave`) · Командировка (`on_trip`) · **Архив** (уволенные, `dismissed`).
* **Поиск** — по ФИО, debounce.
* **Кнопка «Фильтр»** — открывает Drawer, счётчик активных фильтров.

### Колонки таблицы
| Колонка | Содержание |
|---------|-----------|
| Чекбокс | Массовый выбор |
| ФИО | Аватар + Фамилия Имя Отчество |
| Отдел | Отдел основной должности |
| Должность | Основная должность |
| Статус | Статус-бейдж |
| График работы | Кратко (например, `09–18`, или «Сменный») |
| `⋮` | Контекстное меню |

* Клик по строке (вне чекбокса/`⋮`) — переход в эл. дело.
* Сортировка по кликабельным заголовкам (ФИО, отдел, должность, статус).

```

│  📂  Открыть электронное дело          │
│  🖨️  Скачать / распечатать объективку  │
│  🕐  Скачать / распечатать учёт времени│
```

| Действие | Условие |
|----------|---------|
| Открыть электронное дело | `HR_EMPLOYEE_GET` |
| Объективка (скачать/печать) | `HR_EMPLOYEE_GET` |
| Учёт времени (скачать/печать) | `HR_TIME_TRACKING_GET` |
| Уволить | `HR_ARCHIVE_MANAGE`; визуально отделён как необратимое |

«Объективка» и «Учёт времени» открывают выбор формата (PDF / печать); генерация — бэкенд/File Module.

### Action Bar (массовые действия)
```
┌──────────────────────────────────────────────────────────────┐
│  Выбрано: 3 сотрудника         [🗑 Удалить]   [✕ Снять выбор] │
└──────────────────────────────────────────────────────────────┘
```
* «Удалить» — bulk-удаление после подтверждения; доступно при `HR_EMPLOYEE_DELETE`. Сотрудники без права удаления — пропускаются с предупреждением.

### Форма «Принять сотрудника»
```
┌──────────────────────────────────────────────┐
│  Принять сотрудника                       [✕] │
├──────────────────────────────────────────────┤
│  Фото                  [Загрузить файл]       │
│  Сканирование документа (MRZ / паспорт)       │
│  [Загрузить скан]  → авто-распознавание полей │
│                                               │
│  Имя *   Фамилия *   Отчество                 │
│  Дата рождения *     Пол *                    │
│  Семейное положение *   Гражданство *         │
│  Адрес проживания *                           │
│  Тип сотрудника *  ○ Сотрудник  ○ Врач        │
│                                               │
│  Контактные данные                            │
│   📞 Телефон   ✉️ Email   [+ ещё]             │
│                                               │
│  Отдел *      ▾      Должность *  ▾            │  ← Structure / штатное расписание
│  Рабочая ставка  ○ 1.0 ○ 0.5 ○ 0.25           │
│                                               │
│  Системный пользователь *  ▾  ← Auth Module   │
├──────────────────────────────────────────────┤
│              [Отмена]      [Создать]          │
└──────────────────────────────────────────────┘
```

**Примечания для дизайнера:**
* **Сканирование документа (MRZ/паспорт)** — загрузка скана с авто-распознаванием (MRZ-код, серия/номер, ФИО, дата рождения, гражданство); распознанные поля предзаполняются и редактируемы. Распознавание — бэкенд/File Module.
* **Пол** и допустимые значения «Семейное положение» зависят от глобального стандарта клиники (`gender_standard` и т.п.) — не хардкодить число опций.
* **Отдел → Должность** — каскад; «Должность» показывает доступный остаток ставки из штатного расписания.
* **Системный пользователь** — обязательная привязка к Auth Module; пустое состояние селекта подсказывает создать аккаунт.
* После создания «Системный пользователь», тип сотрудника и филиал — неизменяемы.

## Состояния и edge cases

### Empty
* Нет сотрудников: «Пока нет сотрудников» + «+ Принять сотрудника».
* Нет результатов по фильтрам/поиску: «Ничего не найдено» + «Сбросить фильтры».
* Вкладка «Архив» пуста: «Уволенных сотрудников нет».

### Loading
* Скелетон строк таблицы; спиннер на «Создать» при сохранении формы; спиннер при генерации объективки/учёта времени.

### Error
* Сбой загрузки списка — блок ошибки + «Повторить».
* Сбой приёма/удаления — toast; форма остаётся открытой.
* Сбой распознавания скана — поля остаются ручными, toast «Не удалось распознать документ».

### Offline
* Список из кэша; действия (приём, удаление, увольнение) заблокированы, toast «Нет соединения».

### Permissions
* Нет `HR_EMPLOYEE_GET` — страница недоступна.
* Нет `HR_EMPLOYEE_CREATE` — кнопка приёма скрыта.
* Нет `HR_EMPLOYEE_DELETE` — чекбоксы/bulk-удаление недоступны.
* Нет `HR_ARCHIVE_MANAGE` — «Уволить» скрыт.
* Нет `HR_TIME_TRACKING_GET` — пункт «Учёт времени» скрыт.

### Edge cases
* Bulk-удаление с частью защищённых записей — предупреждение со списком пропущенных.
* Сотрудник без назначенной должности — в колонках «Должность/Отдел» прочерк; виден в «не назначенных» (см. 1.6.2).
* Увольнение из меню — запускает флоу архивирования (модалка увольнения, освобождение ставок); после — строка исчезает из активного списка.

## Interaction & Animation

* **Статусные вкладки** — мгновенное переключение с fade-сменой контента; анимированный индикатор активной.
* **Поиск/фильтр** — debounce; область списка показывает мягкий скелетон при обновлении.
* **Drawer фильтра** — slide-in справа, затемнение фона, закрытие по Esc/клику вне.
* **Выбор строк** — чекбокс fade-in на hover; Action Bar выезжает снизу.
* **Контекстное меню** — fade/scale из точки клика; ПКМ открывает в позиции курсора.
* **Форма приёма** — fade+scale; индикатор авто-заполнения «подсвечивает» распознанные поля.
* **Toast** — выезд, авто-скрытие 3–5 с.

## Зависимости и ограничения

### API / интеграции
* `GET /hr/employees` — список (фильтры, поиск, пагинация, статус/архив).
* `POST /hr/employees` — приём; `DELETE /hr/employees` (bulk).
* `POST /hr/employees/:id/dismiss` — увольнение.
* `GET /hr/employees/:id/objective` — объективка (PDF/печать).
* `GET /hr/employees/:id/time-tracking/export` — выгрузка учёта времени.
* **Structure Module** — отделы, должности; **Auth Module** — пользователи, системные роли; **Management Module** — стандарты гендера/семейного положения; **File Module** — фото, скан документа, распознавание, генерация PDF.

### Роли / права
* См. раздел «Спецификация» и условия в контекстном меню.

### Технические ограничения
* Уволенные — отдельная архивная таблица; в активном списке только через вкладку «Архив».
* Тип сотрудника, системный пользователь, филиал — неизменяемы после создания.
* Остаток ставок ограничивает выбор должности при приёме.
* Распознавание MRZ/паспорта — best-effort; поля остаются редактируемыми.

### Feature flags
* `gender_standard` — число опций «Пол» (и связанных стандартов).

## Acceptance criteria

* Таблица показывает колонки ФИО, отдел, должность, статус, график работы; поддерживает пагинацию и сортировку.
* Поиск по ФИО и фильтры (отдел, должность, пол, возраст, гражданство, семейное положение, системная роль, статус) работают; счётчик активных фильтров отображается.
* Клик по строке открывает электронное дело.
* Кнопка «+ Принять сотрудника» видна при праве; форма включает ФИО, дату рождения, пол, семейное положение, гражданство, адрес, сканирование документа (MRZ/паспорт), тип сотрудника, контактные данные, отдел/должность.
* Массовый выбор чекбоксами показывает Action Bar с bulk-удалением и подтверждением.
* Контекстное меню открывается по `⋮` и по ПКМ и содержит: открыть эл. дело, уволить, объективку (скачать/печать), учёт времени (скачать/печать) — с учётом прав.
* Уволенные не показываются в активных вкладках; доступны во вкладке «Архив».
* Состояния empty / loading / error / offline / permissions обрабатываются корректно.
