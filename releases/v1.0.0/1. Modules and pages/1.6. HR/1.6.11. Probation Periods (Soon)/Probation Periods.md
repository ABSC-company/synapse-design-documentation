# 1.6.11 Probation Periods (Испытательные сроки) — Soon

> **Статус:** Soon (планируется). Раздел под feature flag `hr_probation`. До включения — пункт меню неактивен («Скоро»). Бизнес-логика испытательных сроков в бэкенде на момент написания не специфицирована — спецификация ниже является проектным предложением и подлежит подтверждению продуктом.

## Спецификация страницы интерфейса

* **Название страницы:** Испытательные сроки
* **Модуль:** HR (Human Resources)
* **Тип страницы:** Реестр + таймлайн
* **Основные роли:** Администратор / HR-менеджер, Руководитель отдела
* **Права доступа (предлагаемые):** `HR_EMPLOYEE_GET` (просмотр), `HR_PROBATION_MANAGE` (управление)
* **Маршрут:** `/hr/probation`
* **Feature flag:** `hr_probation`
* **Статус документа:** Draft / Soon

---

## Назначение страницы и сценарии использования

Контроль испытательных сроков новых сотрудников: даты начала/окончания, статус (идёт/завершён успешно/не пройден), приближение окончания. Раздел отвечает на вопрос «у кого скоро заканчивается испытательный срок и какое решение принять».

Пользователь сможет:
1. Видеть сотрудников на испытательном сроке с датами и статусом.
2. Отслеживать приближающиеся окончания (связано с виджетом дашборда 1.6.1).
3. Фиксировать результат (пройден / не пройден) с возможным переходом в ЭДО.

---

## User Flows / Task Flows

1. **Просмотр:** открыть раздел → список/таймлайн испытательных сроков → фильтр по отделу/статусу/сроку окончания.
2. **Завершение:** открыть запись → «Завершить испытательный срок» → результат (успешно/не пройден) + комментарий → при «не пройден» возможен переход к увольнению (1.6.4).
3. **Назначение:** при приёме (1.6.3) опционально задаётся испытательный срок (период) → запись появляется здесь.

```
┌─────────────────────────────────────────────────────────────────┐
│  Испытательные сроки                          [Фильтр ▾]         │
├──────────────┬──────────┬────────────┬──────────────┬───────────┤
│ Сотрудник    │ Отдел    │ Период     │ Осталось     │ Статус    │
```

* Колонки: сотрудник, отдел, период, осталось дней (с акцентом при ≤ N дней), статус-бейдж.
* Форма «Завершить»: результат (○ Успешно / ○ Не пройден) + комментарий + (опц.) переход в ЭДО.

## Состояния и edge cases

* **Empty:** «Нет сотрудников на испытательном сроке».
* **Loading/Error/Offline:** по общим паттернам (`1.6.0`).
* **Permissions:** без `HR_PROBATION_MANAGE` — только просмотр; раздел скрыт без флага `hr_probation`.
* **Edge cases:** авто-перевод в «истекает скоро» при приближении даты; «не пройден» как точка входа в увольнение; продление срока.

## Зависимости и ограничения

* **API (предлагаемые):** `GET /hr/probation`, `PATCH /hr/probation/:id` (завершение). Связь с приёмом (1.6.3), увольнением (1.6.4), дашбордом (1.6.1) и ЭДО.
* **Роли:** `HR_EMPLOYEE_GET` / `HR_PROBATION_MANAGE` (предлагаемые).
* **Ограничения:** раздел под `hr_probation`; бизнес-логика не финализирована в бэкенде — требует подтверждения.

---

## Acceptance criteria (предварительные)

* Раздел доступен при включённом `hr_probation`, иначе показан как «Скоро».
* Реестр показывает сотрудников на испытательном сроке с периодом, остатком дней и статусом.
* Приближающиеся окончания визуально выделены и согласованы с виджетом дашборда.
* Завершение фиксирует результат; «не пройден» ведёт к сценарию увольнения.
* Состояния empty / loading / error / offline / permissions обрабатываются корректно.
