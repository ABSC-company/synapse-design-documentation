# 1.6.5 Staffing Schedule

## Спецификация страницы интерфейса

* **Название страницы:** Штатное расписание
* **Модуль:** HR (Human Resources)
* **Тип страницы:** Реестр (таблица)
* **Основные роли:** Администратор, HR-менеджер, Руководитель
* **Права доступа:** `HR_STAFFING_GET` (просмотр), `HR_STAFFING_MANAGE` (управление)
* **Маршрут:** `/hr/staffing`
* **Платформа:** веб-приложение медицинской системы
* **Статус документа:** Draft

> Общие паттерны (реестр, форма, состояния) — см. `1.6.0. Overview`.

---

## Назначение страницы и сценарии использования

Просмотр и управление штатным расписанием — перечнем должностей, количеством ставок, окладами и уровнем заполненности. Занятость ставок формируется автоматически при назначении/снятии должностей сотрудников (1.6.4 / 1.6.2) и вручную не редактируется.

С помощью страницы пользователь может:

1. Просматривать должности расписания с количеством ставок и заполненностью.
2. Фильтровать по отделу и наличию вакансий.
3. Создавать, редактировать и удалять записи.
4. Переходить к назначению на вакантную должность.

---

## User Flows / Task Flows

### Flow 1. Просмотр и фильтрация
1. Пользователь открывает «Штатное расписание».
2. Система показывает список с прогресс-барами заполненности.
3. Пользователь фильтрует по отделу/вакансиям.

### Flow 2. Создание / редактирование
1. «+ Добавить позицию» (или `⋯` → «Редактировать»).
2. Заполняет/меняет поля формы, привязывает **обязательный первичный документ** (приказ о вводе/изменении штатной единицы — новый / из шаблона / загруженный / из входящих).
3. Сохраняет (кнопка активна только при привязанном документе).

### Flow 3. Удаление (упразднение)
1. `⋯` → «Удалить» → модалка подтверждения с **обязательным первичным документом** (приказ об упразднении штатной единицы) → удаление.

## Структура: Wireframes

│  Штатное расписание                          [+ Добавить позицию]│

### Header / Control Bar
* Заголовок + «+ Добавить позицию» (`HR_STAFFING_MANAGE`).
* Поиск по названию должности + кнопка «Фильтр» (Drawer).

### Drawer фильтрации
| Поле | Тип |
|------|-----|
| Отдел | Multi-select (Structure Module) |
| Наличие вакансий | Toggle (`tariff_rates > occupied_tariff_rates`) |

### Таблица
| Колонка | Описание |
|---------|----------|
| Должность | Из Structure Module |
| Отдел | Из Structure Module |
| Всего ставок | `tariff_rates` |
| Занято | `occupied_tariff_rates` (авто) |
| Вакантно | `tariff_rates − occupied_tariff_rates` |
| Заполненность | Прогресс-бар `occupied / tariff_rates` |
| Оклад (1 ставка) | `salary` |
| Базовая роль | Auth Module (если задана, иначе `—`) |
| Действия | `⋯` |

Цветовая индикация заполненности: полностью занято — зелёный; частично — жёлтый; пусто/вакантно — красный.

Контекстное меню `⋯`: Редактировать, Удалить (оба — `HR_STAFFING_MANAGE`); «Назначить сотрудника →» (переход в орг. структуру 1.6.2 для вакантных).

### Форма «Новая / Редактирование позиции»
| Поле | Тип | Обяз. |
|------|-----|:---:|
| Должность (`position_id`) | Select (Structure) | ✓ |
| Количество ставок (`tariff_rates`) | Number, шаг 0.25 | ✓ |
| Оклад за 1 ставку (`salary`) | Number | ✓ |
| Базовая роль (`base_role_id`) | Select (Auth) | — |
| Первичный документ | Блок «Первичный документ» (создать новый / из шаблона / загрузить файл / из входящих) | ✓ |

Поведение: `occupied_tariff_rates` не редактируется; при редактировании нельзя задать `tariff_rates` < `occupied_tariff_rates` (инлайн-ошибка). Создание/изменение/упразднение штатной единицы — структурное изменение и требует первичного документа (паттерн из 1.6.0); «Сохранить»/«Удалить» неактивны без привязанного документа.

## Состояния и edge cases

* **Empty:** «Штатное расписание не заполнено» + «+ Добавить позицию».
* **Loading:** скелетон строк; спиннер на сохранении.
* **Error:** баннер + «Повторить»; toast при сбое сохранения/удаления.
* **Offline:** таблица из кэша; управление заблокировано, toast.
* **Permissions:** нет `HR_STAFFING_GET` — 403/скрыто; нет `HR_STAFFING_MANAGE` — добавление и `⋯` скрыты.
* **Edge cases:** снижение `tariff_rates` ниже занятого — блок; удаление позиции с занятыми ставками — предупреждение/запрет; фильтр вакансий показывает только незанятые.

---

## Interaction & Animation

* Прогресс-бары плавно анимируются при загрузке/обновлении.
* Поиск — debounce 300–500 мс; Drawer — slide-in справа.
* Hover строки — подсветка, появление `⋯`; меню — fade/scale.
* Форма — fade+scale модалки; Toast — выезд, авто-скрытие 3–5 с.

---

## Зависимости и ограничения

* **API:** `GET/POST/PATCH/DELETE /hr/staffing`. **Structure Module** — должности/отделы; **Auth Module** — базовые роли.
* Связь с 1.6.4/1.6.2: назначение/снятие должности меняет `occupied_tariff_rates`.
* **Роли:** просмотр — `HR_STAFFING_GET`; управление — `HR_STAFFING_MANAGE`.
* **Ограничения:** `occupied_tariff_rates` — вычисляемое; `tariff_rates ≥ occupied_tariff_rates`; базовая роль авто-выдаётся при назначении.

---

## Acceptance criteria

* Таблица отображает все записи с колонками всего/занято/вакантно/заполненность/оклад/базовая роль.
* Прогресс-бар корректно отражает заполненность с цветовой индикацией.
* Фильтр вакансий показывает только должности с незанятыми ставками.
* Форма не позволяет задать `tariff_rates` меньше `occupied_tariff_rates`.
* Создание/изменение/упразднение позиции требует обязательного первичного документа; «Сохранить»/«Удалить» неактивны без привязанного документа.
* После назначения/снятия должности занятость обновляется автоматически.
* Добавление и контекстное меню видны только при `HR_STAFFING_MANAGE`.
* Состояния empty / loading / error / offline / permissions обрабатываются корректно.
