# 1.6.4 Employee E-File (Электронное дело сотрудника)

## Спецификация страницы интерфейса

* **Название страницы:** Электронное дело сотрудника
* **Модуль:** HR (Human Resources)
* **Тип страницы:** Карточка сущности с секциями/вкладками
* **Основные роли:** Администратор / HR-менеджер, Руководитель отдела, Сотрудник (своё дело — просмотр)
* **Права доступа:** `HR_EMPLOYEE_GET` (просмотр) + права `*_MANAGE` отдельных секций (см. «Зависимости»)
* **Маршрут:** `/hr/employees/:id`
* **Платформа:** веб-приложение медицинской системы
* **Статус документа:** Draft

> Общие паттерны (форма, статус-бейдж, загрузка файла, переход в ЭДО, объективка, состояния) — см. `1.6.0. Overview`.

---

## Назначение страницы и сценарии использования

Единое электронное дело сотрудника со всеми кадровыми данными и точкой кадровых действий. Дело разбито на секции (вкладки): персональные данные, рабочие данные, врачебные данные (для врачей), график, учёт времени, карьера, аттестации, документы. В шапке — быстрые кадровые действия (отпуск, больничный, командировка, увольнение).

С помощью страницы пользователь может:

1. Просматривать и редактировать персональные и рабочие данные.
2. Управлять специализациями/допусками врача (с удостоверяющими документами).
3. Настраивать рабочий график.
4. Смотреть учёт рабочего времени в виде календаря (приход/уход по дням, выбор месяца).
5. Видеть карьеру (таймлайн смены должностей).
6. Просматривать аттестации (таймлайн результатов).
7. Загружать/удалять документы сотрудника.
8. Выполнять кадровые действия: отпуск, больничный, командировка, увольнение.

## User Flows / Task Flows

### Flow 1. Открытие и навигация по секциям
1. Пользователь открывает дело из реестра (1.6.3), карты (1.6.2) или списка дашборда.
2. По умолчанию — секция «Персональные данные».
3. Для врача видны секции «Специализации», «Допуски», «Аттестации».

### Flow 2. Прямое редактирование данных
1. В секции «Персональные»/«Рабочие данные» пользователь нажимает «Редактировать» (или редактирует инлайн).
2. Вносит изменения, сохраняет → данные и шапка обновляются.

### Flow 3. Кадровое действие из шапки (отпуск/больничный/командировка)
1. Пользователь нажимает соответствующую кнопку в шапке.
2. Открывается модалка действия: период/даты, комментарий, блок «Переход в ЭДО».
3. Подтверждение → статус сотрудника меняется (`on_leave` / `away` / `on_trip`), запись отражается в графике/табеле.

### Flow 4. Увольнение
1. Пользователь нажимает «Уволить» в шапке.
2. Модалка увольнения с предупреждением об архивировании и освобождении ставок + переход в ЭДО.
3. Подтверждение → архивирование, сотрудник уходит в `dismissed`.

### Flow 5. Работа с врачебными данными / документами
1. В секции «Специализации»/«Допуски»/«Документы» — добавление через форму с загрузкой удостоверяющего файла; удаление с подтверждением.

## Структура: Wireframes

### Общий layout

```
┌───────────────────────────────────────────────────────────────────────────┐
│  ← Назад к списку                                                          │
├───────────────────────────────────────────────────────────────────────────┤
│  🖼  Иванов Иван Иванович            ● На рабочем месте ▾                   │
│      Хирург · Хирургия                                                      │
│      [🏖 В отпуск] [🤒 На больничный] [✈ В командировку] [🚪 Уволить] [⋮]   │
├───────────────────────────────────────────────────────────────────────────┤
│ [Персональные][Рабочие данные][Специализации]*[Допуски]*[График]            │
│ [Учёт времени][Карьера][Аттестации]*[Документы]                             │
├───────────────────────────────────────────────────────────────────────────┤
│                       содержимое активной секции                           │

```

\* «Специализации», «Допуски», «Аттестации» — для `employment_type = doctor`.

### Шапка дела
* Крупный аватар, ФИО, основная должность · отдел.
* Статус — кликабельный бейдж-дропдаун (допустимые переходы, см. `1.6.0`).
* Кнопки кадровых действий: **В отпуск**, **На больничный**, **В командировку**, **Уволить**.
* `⋮` — доп. действия: объективка (скачать/печать), учёт времени (экспорт), редактировать.

### Секция «Персональные данные»
Прямое редактирование (инлайн или кнопка «Редактировать»).
```
┌─────────────────────────────────────────────────────────┐
│  Персональные данные                  [Редактировать]    │
├─────────────────────────────────────────────────────────┤
│  ФИО · Дата рождения · Пол · Семейное положение          │
│  Гражданство · Адрес проживания                          │
│  Контакты: 📞 …  ✉️ …  💬 …                              │
│  Документы личности: MRZ · Серия/номер  (служебно)       │
└─────────────────────────────────────────────────────────┘
```
* Управление — `HR_EMPLOYEE_UPDATE`, контакты — `HR_CONTACTS_MANAGE`.

### Секция «Рабочие данные»
Прямое редактирование. Должности + статус инструктажей + рабочие места.
```
┌─────────────────────────────────────────────────────┐
│  Должности                  [+ Назначить должность]  │
│  Хирург · Хирургия         Ставка 1.0      [✏][✕]    │
│  Консультант · Терапия      Ставка 0.5     [✏][✕]    │
├─────────────────────────────────────────────────────┤
│  Рабочие места             [+ Назначить место]       │
│  Кабинет 204  ★ Основное                  [✏][✕]    │
├─────────────────────────────────────────────────────┤
│  Статус инструктажей                                 │
│  ✅ ТБ в операционной — пройден 12.01.2026           │
│  ⚠ Работа с оборудованием — требуется                │
└─────────────────────────────────────────────────────┘
```
* Назначение/снятие должности занимает/освобождает ставку (штатное расписание) + блок «Переход в ЭДО».
* Одно основное рабочее место; смена основного снимает метку с прежнего (с предупреждением).
* Статус инструктажей агрегирует требуемые/пройденные (источник — 1.6.6); «требуется» подсвечено.
* Права: `HR_POSITION_MANAGE`, `HR_WORKPLACE_MANAGE`.

### Секция «Специализации» (врач)
```
┌─────────────────────────────────────────────────────┐
│  Специализации              [+ Добавить]             │
│  🩺 Хирургия · Категория: Высшая   📎 документ [✏][✕]│
│  🩺 Эндоскопия · Категория: —      📎 документ [✕]   │
└─────────────────────────────────────────────────────┘
```
* Форма: специализация (Management Module) + категория (Management Module, необяз.) + удостоверяющий документ (File Module).
* Право — `HR_DOCTOR_SPECIALTY_MANAGE`.

### Секция «Допуски» (врач)
```
┌─────────────────────────────────────────────────────┐
│  Допуски к услугам          [+ Добавить]             │
│  Услуги: УЗИ, Гастроскопия    📎 сертификат  [✏][✕] │
└─────────────────────────────────────────────────────┘
```
* Форма: услуги (мульти-select, Services Submodule) + сертификат (может быть бессрочным).
* Право — `HR_DOCTOR_ADMISSION_MANAGE`.

### Секция «График»
Недельная таблица рабочих дней/промежутков/перерывов.
```
        Пн     Вт     Ср     Чт     Пт    Сб Вс
Раб.     ✓      ✓      ✓      ✓      ✓     –  –
Время  09–18  09–18  09–18  09–18  09–18
Перерыв13–14  13–14  13–14  13–14  13–14
```
* «Редактировать»: multi-toggle дней, динамические пары time-picker'ов (промежутки и перерывы), валидация «перерыв внутри рабочего времени».
* Пусто — «График не установлен» + «Установить график». Право — `HR_SCHEDULE_MANAGE`.

### Секция «Учёт рабочего времени» (календарь)
```
┌─────────────────────────────────────────────────────────────┐
│  Учёт времени          ◀ Июнь 2026 ▶        [+ Отметка]      │
├─────────────────────────────────────────────────────────────┤
│  Пн   Вт   Ср   Чт   Пт   Сб   Вс                            │
│  ┌──┐┌──┐┌──┐┌──┐┌──┐┌──┐┌──┐                                │
│  │1 ││2 ││3 ││4 ││5 ││6 ││7 │                                │
│  │09││09││— ││09││09││  ││  │  ← приход                       │
│  │18││18││  ││18││17││  ││  │  ← уход                         │
│  └──┘└──┘└──┘└──┘└──┘└──┘└──┘                                │
│  ...                                                         │
│  Итого за месяц: 168 ч · Опозданий: 2 · Пропусков: 1         │
└─────────────────────────────────────────────────────────────┘
```
* Месячный календарь: в каждой ячейке дня — приход/уход; день без отметок выделен; опоздания/недоработки — цветовой акцент.
* Переключение месяца стрелками; сводка по месяцу внизу.
* «+ Отметка» (корректирующее, secondary-стиль) — тип (Приход/Уход) + дата-время. Управление — `HR_TIME_TRACKING_MANAGE`; просмотр — `HR_TIME_TRACKING_GET`.

### Секция «Карьера» (таймлайн должностей)
История смены должностей в рамках клиники (источник — холодное хранилище, `employees_cold_storage.employees_positions`).
```
┌─────────────────────────────────────────────────────┐
│  Карьера                                             │
│  ● 2026 — наст. время                                │
│  │   Хирург · Хирургия · ставка 1.0                  │
│  │   📄 Приказ о переводе                            │
│  ● 2023 — 2026                                        │
│  │   Врач-ординатор · Хирургия · ставка 0.5          │
│  │   📄 Приказ о приёме                              │
│  ○ ...                                                │
└─────────────────────────────────────────────────────┘
```
* Вертикальный таймлайн: период, должность, отдел, ставка, ссылка на приказ (ЭДО). Только просмотр.

### Секция «Аттестации» (врач, таймлайн)
```
┌─────────────────────────────────────────────────────┐
│  Аттестации                 [+ Добавить]             │
│  ● 09.2026  Хирургия · Присвоение категории          │
│             ● Запланирована                          │
│  ● 03.2026  Терапия · Подтверждение категории         │
│             ● Просрочена                             │
└─────────────────────────────────────────────────────┘
```
* Таймлайн результатов; статус-бейдж (`scheduled`/`overdue`/`failed`). Форма: специализация + период + тип/статус. Право — `HR_CERTIFICATION_MANAGE`.

### Секция «Документы»
```
┌─────────────────────────────────────────────────────────────┐
│  Документы                        [+ Добавить документ]      │
│  📄 Диплом · Бессрочный                            [✏][✕]    │
│  📄 Удостоверение · до 12.05.2027  ● Действует     [✏][✕]    │
│  📄 Сертификат · до 01.02.2026  ● Истекает скоро   [✏][✕]    │
└─────────────────────────────────────────────────────────────┘
```
* Форма: тип документа (справочник `employees_documents_types`, управляемый мини-реестром из формы) + чекбокс «Бессрочный» (скрывает период) + срок + файл (File Module).
* Статус-бейдж по сроку (действует/истекает ≤30 дней/истёк/бессрочный). Право — `HR_EMPLOYEE_DOCS_MANAGE`.

### Модалки кадровых действий (шапка)
**В отпуск / В командировку:** период (date-range) + комментарий + блок «Переход в ЭДО» (folders `leave` / `business_trip`). Подтверждение → статус `on_leave` / `on_trip`; отпуск отражается в графике отпусков (1.6.9).
**На больничный:** дата начала (+ ожидаемая дата окончания, необяз.) + комментарий. Статус → `away` (с признаком больничного).
**Уволить:** предупреждение об архивировании + освобождении ставок + переход в ЭДО (folder `dismissal`); кнопка «Уволить» — деструктивный стиль; во время архивирования — спиннер «Архивируем данные…».

## Визуал: High-Fidelity mockups

1. Шапка доминирует: крупный аватар, ФИО, статус-дропдаун, ряд кнопок кадровых действий с понятными иконками.

6. Все статус-бейджи — единая семантика модуля.
7. Формы с загрузкой файла показывают превью документа.
8. Соответствие единой дизайн-системе.

## Состояния и edge cases

### Empty (по секциям)
Должности — «Должности не назначены»; Рабочие места — «Не назначены»; Специализации/Допуски — «Не добавлены»; График — «Не установлен»; Учёт времени — «Отметок за месяц нет»; Карьера — «История пуста»; Аттестации — «Аттестаций нет»; Документы — «Документы не добавлены».

### Loading
Скелетон содержимого секции; спиннер в формах и при архивировании (увольнение).

### Error
Сбой загрузки секции — блок ошибки + «Повторить»; сбой действия — toast; валидация — инлайн под полем.

### Offline
Кадровые действия и CRUD заблокированы, toast «Нет соединения»; незагруженная секция — Error.

### Permissions
Секция/действие скрыты без соответствующего права; своё дело сотрудник видит read-only; кадровые действия скрыты без `HR_EMPLOYEE_STATUS_UPDATE`/`HR_ARCHIVE_MANAGE`.

### Edge cases
* `employment_type` неизменяем — набор врачебных секций фиксирован.
* Снятие последней должности освобождает ставку (предупреждение).
* Смена основного рабочего места снимает метку с прежнего (предупреждение).
* Перерыв вне рабочего промежутка — блок валидацией.
* Отпуск/командировка с пересекающимися периодами — предупреждение о конфликте (проверка по графику отпусков).
* Возврат с больничного/отпуска/командировки — переход обратно в `at_work`.

## Interaction & Animation

* **Переключение секций** — fade ~150–200 мс, анимированный индикатор вкладки.
* **Статус-дропдаун** — раскрытие из бейджа; простой переход без подтверждения, бейдж перекрашивается, toast.
* **Кнопки кадровых действий** — открывают модалки fade+scale; «Уволить» — подтверждение + спиннер архивирования.
* **Календарь учёта** — слайд при смене месяца; hover дня показывает детали отметок.
* **Таймлайны** — последовательное появление узлов при прокрутке (reveal).
* **CRUD в секциях** — hover-подсветка строк, `[✏][✕]` на hover; удаление с подтверждением и анимацией исчезновения.
* **Toast** — выезд, авто-скрытие 3–5 с.

## Зависимости и ограничения

### API / интеграции
* `GET/PATCH /hr/employees/:id` — персональные/рабочие; `PATCH /…/status` — статус.
* `/positions`, `/contacts`, `/workplaces`, `/schedule`, `/specialties`, `/admissions`, `/certifications`, `/documents`, `/document-types`, `/briefings`, `/time-tracking` — секции.
* `GET /…/career` (cold storage) — таймлайн должностей.
* `POST /…/dismiss` — увольнение (архивирование).
* **Structure Module** (должности/отделы/места), **Auth Module** (роли), **Management Module** (специализации/категории/гендер), **Services Submodule** (услуги для допусков), **File Module** (фото, документы, объективка, архив), **EDM Module** (приказы по folders hire/transfer/leave/business_trip/dismissal).

### Роли / права (по секциям)
| Секция / действие | Право |
|---|---|
| Персональные | `HR_EMPLOYEE_UPDATE` |
| Контакты | `HR_CONTACTS_MANAGE` |
| Должности | `HR_POSITION_MANAGE` |
| Рабочие места | `HR_WORKPLACE_MANAGE` |
| График | `HR_SCHEDULE_MANAGE` |
| Специализации | `HR_DOCTOR_SPECIALTY_MANAGE` |
| Допуски | `HR_DOCTOR_ADMISSION_MANAGE` |
| Аттестации | `HR_CERTIFICATION_MANAGE` |
| Документы | `HR_EMPLOYEE_DOCS_MANAGE` |
| Учёт времени | `HR_TIME_TRACKING_GET` / `MANAGE` |
| Смена статуса (отпуск/больн./команд.) | `HR_EMPLOYEE_STATUS_UPDATE` |
| Увольнение | `HR_ARCHIVE_MANAGE` |

### Технические ограничения
* `employment_type`, системный пользователь, филиал — неизменяемы после создания.
* `occupied_tariff_rates` обновляется автоматически при назначении/снятии должности.
* Одно основное рабочее место; перерыв внутри рабочего времени.
* Карьера/история — read-only из холодного хранилища.

### Feature flags
* `gender_standard` — опции «Пол»/«Семейное положение»; стандарты специализаций/категорий — Management Module.

## Acceptance criteria

* Дело — единая страница с секциями; шапка показывает фото, ФИО, должность/отдел, статус и кнопки кадровых действий (отпуск, больничный, командировка, увольнение).
* Секции «Специализации», «Допуски», «Аттестации» видны только для врача.
* В «Персональных» и «Рабочих данных» данные редактируются напрямую.
* Специализации/допуски добавляются/удаляются с загрузкой удостоверяющего документа.
* График настраивается недельной формой с валидацией перерыва.
* «Учёт времени» — календарь с приходом/уходом по дням и выбором месяца.
* «Карьера» — таймлайн смены должностей; «Аттестации» — таймлайн результатов со статус-бейджами.
* «Документы» — загрузка/удаление с указанием типа; различение срочных/бессрочных.
* Кадровые действия меняют статус и предлагают переход в ЭДО; увольнение архивирует данные и освобождает ставки.
