# 1.6.1 HR Dashboard

## Спецификация страницы интерфейса

* **Название страницы:** Главный дашборд HR
* **Модуль:** HR (Human Resources)
* **Тип страницы:** Дашборд (оперативная аналитика + виджеты)
* **Основные роли:** Администратор / HR-менеджер, Руководитель отдела
* **Права доступа:** `HR_EMPLOYEE_GET` (база); виджеты дополнительно ограничены правами своих разделов (`HR_STAFFING_GET`, `HR_CERTIFICATION_*`, `HR_TIME_TRACKING_GET` и т.д.)
* **Маршрут:** `/hr`
* **Платформа:** веб-приложение медицинской системы
* **Статус документа:** Draft

> Общие паттерны (статус-бейдж, состояния, права) — см. `1.6.0. Overview`.

---

## Назначение страницы и сценарии использования

Дашборд даёт HR-менеджеру оперативную и краткую картину всей кадровой работы клиники: сводку по сотрудникам, ближайшие события и задачи, узкие места (свободные ставки, предстоящие аттестации/инструктажи/отпуска) — через статистические карточки, графики, диаграммы и списки. Это стартовая страница модуля и точка быстрого перехода в профильные разделы.

С помощью страницы пользователь может:

1. Видеть общую сводку по персоналу (на местах, отсутствуют, на больничном, в отпуске, в командировке).
2. Вести календарь ближайших событий/задач, добавлять события (в т.ч. рутинные — повторяющиеся).
3. Контролировать свободные штатные единицы по должностям.
4. Видеть списки: требуемые инструктажи, предстоящие аттестации, предстоящие отпуска, сотрудники на больничном, сотрудники в командировке.
5. Отслеживать графики: истечение испытательных сроков, предстоящие отчёты.
6. Переходить из любого виджета в соответствующий раздел (drill-down).

## User Flows / Task Flows

### Flow 1. Обзор и переход в раздел

1. Пользователь открывает HR-модуль — по умолчанию загружается дашборд.
2. Система отображает виджеты по доступным данным/правам.
3. Пользователь кликает заголовок виджета или элемент списка → переход в соответствующий раздел/эл. дело.

### Flow 2. Добавление события/задачи в календарь

1. Пользователь нажимает «+ Событие» в виджете календаря (или на ячейке дня).
2. Открывается модальная форма события.
3. Пользователь заполняет название, дату/время, тип, при необходимости включает «Повторять» и задаёт периодичность.
4. Сохранение → событие появляется в календаре; рутинное — раскладывается на будущие повторы.

### Flow 3. Реакция на узкое место

1. Пользователь видит в виджете «Свободные штатные единицы» вакантную должность.
2. Кликает строку → переход в штатное расписание (1.6.5) с предвыбранной должностью, либо в орг. структуру (1.6.2) для назначения.

### Flow 4. Настройка дашборда (опционально)

1. Пользователь нажимает «Настроить» → режим редактирования сетки виджетов.
2. Перетаскивает/скрывает виджеты, сохраняет раскладку (персональную).

### Виджет «Сводка по сотрудникам»

Набор статистических карточек (число + подпись + иконка + цвет статуса):

| Карточка | Источник |
|----------|----------|
| На рабочих местах | count `at_work` |
| Отсутствуют (вне места) | count `away` (без больничного контекста) |
| На больничном | count `away` с признаком больничного |
| В отпуске | count `on_leave` |
| В командировке | count `on_trip` |

* Клик по карточке → реестр сотрудников (1.6.3) с предустановленным фильтром статуса.
* Возможна сводная диаграмма (donut) распределения по статусам.

### Виджет «Календарь событий/задач»

* Месячный календарь с маркерами дней, где есть события.
* Под календарём — список ближайших событий (название, дата/время, тип).
* Кнопка «+ Событие» и клик по дню — создание события.
* Типы событий (пример): аттестация, инструктаж, отпуск, отчёт, произвольная задача.

**Форма «Событие/задача»:**

```
┌────────────────────────────────────────┐
│  Новое событие                       [✕]│
├────────────────────────────────────────┤
│  Название *                              │
│  Тип            ▾                        │
│  Дата *         Время                    │
│  Связано с сотрудником  ▾  (необяз.)     │
│  Описание                                │
│  ─────────────────────────────────────  │
│  [ ] Повторять                           │
│   Периодичность ▾ (день/неделя/месяц/…)  │  ← блок виден при «Повторять»
│   Интервал [N]   До даты [____]          │
├────────────────────────────────────────┤
│              [Отмена]      [Сохранить]  │
└────────────────────────────────────────┘
```

* «Повторять» раскрывает блок периодичности (рутинное событие): тип повтора, интервал, дата окончания/число повторов.

### Виджет «Свободные штатные единицы»

* Список должностей с вакантными ставками: должность · отдел · свободно ставок.
* Источник — штатное расписание (`tariff_rates − occupied_tariff_rates > 0`).
* «Все вакансии →» → штатное расписание (1.6.5, фильтр вакансий).

### Списочные виджеты

| Виджет | Элемент списка | Переход |
|--------|----------------|---------|
| Требуемые инструктажи | Сотрудник · должность · инструктаж | 1.6.6 / эл. дело |

## Состояния и edge cases

### Empty
* Нет данных в виджете — мини-empty внутри карточки («Нет предстоящих аттестаций», «Событий нет»).
* Календарь без событий — пустая сетка + подсказка «+ Событие».

### Loading
* Скелетоны карточек/виджетов при первой загрузке; виджеты грузятся независимо (по мере готовности данных).

### Error
* Сбой загрузки виджета — локальная ошибка внутри карточки + «Повторить», без падения всего дашборда.
* Ошибка сохранения события — toast.

### Offline
* Дашборд показывает последние закэшированные данные с пометкой «Данные могут быть неактуальны»; создание события заблокировано, toast «Нет соединения».

### Permissions
* Виджет скрыт, если у пользователя нет права на его раздел (например, нет `HR_STAFFING_GET` → скрыт виджет вакансий).
* Soon-виджеты скрыты/неактивны без feature flag.

### Edge cases
* Рутинное событие с очень частым повтором — ограничение горизонта генерации повторов (например, на 12 мес. вперёд) во избежание переполнения календаря.
* Большое число вакансий/аттестаций — виджет показывает топ-N + «Все →».

## Interaction & Animation

* **Загрузка виджетов** — поочерёдное появление (fade-in) по готовности данных.
* **Статистические карточки** — лёгкий hover-подъём; число анимируется счётчиком при первой загрузке.
* **Диаграммы** — анимация отрисовки (~300–400 мс), интерактивные тултипы.
* **Календарь** — переключение месяцев слайдом; выбор дня — подсветка.
* **Форма события** — fade+scale модалки; блок «Периодичность» разворачивается анимацией высоты.
* **Drill-down** — клик по виджету плавно переводит в раздел с предустановленным фильтром.
* **Режим «Настроить»** — drag-and-drop виджетов с плейсхолдером и плавным reflow сетки.

## Зависимости и ограничения

### API / интеграции
* `GET /hr/dashboard/summary` — агрегаты по статусам сотрудников.
* `GET /hr/dashboard/events`, `POST /hr/dashboard/events` — календарь событий/задач (вкл. правила повтора).
* Данные виджетов агрегируются из: `staffing_schedule` (вакансии), `employees_certifications` (аттестации), `employees_briefings`/`briefings` (инструктажи), статусов сотрудников (больничный/командировка), графика отпусков (1.6.9).
* **Structure Module** — названия отделов/должностей.

### Роли / права
* Базовый доступ — `HR_EMPLOYEE_GET`. Видимость каждого виджета дополнительно ограничена правом его раздела.

### Технические ограничения
* Виджеты загружаются независимо; ошибка одного не ломает дашборд.
* Горизонт генерации повторяющихся событий ограничен.
* Раскладка виджетов («Настроить») сохраняется per-user.

### Feature flags
* `hr_probation`, `hr_reports` — виджеты испытательных сроков и отчётов.
* `hr_vacations` — виджет предстоящих отпусков.

## Acceptance criteria

* Дашборд — стартовая страница HR-модуля, отображает сводку по статусам сотрудников статистическими карточками.
* Карточки сводки кликабельны и ведут в реестр с предустановленным фильтром статуса.
* Календарь показывает ближайшие события; форма создания поддерживает рутинные (повторяющиеся) события с периодичностью.
* Присутствуют виджеты: свободные штатные единицы, требуемые инструктажи, предстоящие аттестации, предстоящие отпуска, на больничном, в командировке.
* Виджеты «истечение испытательных сроков» и «предстоящие отчёты» присутствуют как графики-таймлайны (под feature-флагами, со статусом Soon).
* Каждый виджет поддерживает drill-down в соответствующий раздел.
* Виджеты загружаются независимо; ошибка одного не ломает остальные.
* Виджеты скрываются по отсутствию прав/флагов.
* Состояния empty / loading / error / offline / permissions обрабатываются корректно.
