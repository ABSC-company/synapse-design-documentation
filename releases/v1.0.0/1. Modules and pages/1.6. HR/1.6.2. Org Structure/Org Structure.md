# 1.6.2 Org Structure

## Спецификация страницы интерфейса

* **Название страницы:** Организационная структура
* **Модуль:** HR (Human Resources)
* **Тип страницы:** Интерактивная карта-область (дерево иерархии) + информационный сайдбар
* **Основные роли:** Администратор / HR-менеджер, Руководитель
* **Права доступа:** `HR_EMPLOYEE_GET` (просмотр), `HR_POSITION_MANAGE` / `HR_ARCHIVE_MANAGE` (перевод/назначение), справочники — Structure Module
* **Маршрут:** `/hr/structure`
* **Платформа:** веб-приложение медицинской системы
* **Статус документа:** Draft

> Общие паттерны — см. `1.6.0. Overview`. Орг. справочники (отделы, должности) ведутся в Structure Module; эта страница их визуализирует и инициирует кадровые действия.

---

## Назначение страницы и сценарии использования

Графическое представление иерархии клиники в виде интерактивного дерева. Каждый узел — структурная единица (отдел); связи между узлами обозначают подчинённость. Внутри карточек узлов отображаются сотрудники. Страница служит и для наглядного обзора структуры, и для интерактивного назначения/перевода сотрудников «перетаскиванием».

С помощью страницы пользователь может:

1. Просматривать дерево отделов и их подчинённость.
2. Видеть сотрудников внутри отделов (по должностям).
3. Искать отдел/сотрудника по карте.
4. Открывать сводную информацию о всей структуре (без выбранного узла).
5. Открывать детали отдела (при выбранном узле).
6. Видеть список не назначенных на должность сотрудников.
7. Перетаскивать сотрудника из сайдбара на узел карты → инициировать перевод/назначение через приказ.

## User Flows / Task Flows

### Flow 1. Навигация по карте

1. Пользователь открывает «Орг. структура».
2. Система отрисовывает дерево; сайдбар показывает общую сводку.
3. Пользователь панорамирует/масштабирует карту, разворачивает/сворачивает ветви.

### Flow 2. Просмотр отдела

1. Пользователь кликает узел (отдел).
2. Узел подсвечивается как активный.
3. Сайдбар переключается на детали отдела: сотрудники по должностям, вакансии, руководитель.

### Flow 3. Поиск

1. Пользователь вводит запрос в поиск по карте.
2. Совпадающие узлы/сотрудники подсвечиваются; карта центрируется на первом совпадении.

### Flow 4. Назначение/перевод перетаскиванием (ключевой сценарий)

1. Пользователь захватывает карточку сотрудника в любом списке сайдбара (не назначенные / сотрудники отдела / все сотрудники).
2. Тащит её на целевой узел отдела на карте.
3. Узел подсвечивается как валидная зона сброса.
4. При сбросе открывается модальное окно «Приказ о переводе/назначении».
5. Форма: отдел (авто-заполнен целевым узлом), должность (select из свободных/вакантных должностей этого отдела), ставка.
6. **Обязательный блок «Первичный документ»** (folder `transfer`, паттерн из 1.6.0): приказ о переводе/назначении (новый / из шаблона) либо загруженный/выбранный инициирующий документ (рапорт, ходатайство).
7. Кнопка «Перевести» активна только после привязки документа.
8. Подтверждение → сотрудник назначается/переводится, карта и сайдбар обновляются, приказ фиксируется в делопроизводстве (1.6.10).

### Карта-область (canvas)

* Дерево узлов сверху вниз (или слева направо); связи — линии подчинённости.
* **Узел отдела (карточка):** название отдела, кол-во сотрудников, индикатор вакансий, аватары сотрудников (стек с «+N»), руководитель (отметка).
* Управление картой: zoom in/out, fit-to-screen `⤢`, панорамирование drag по пустому полю, разворот/сворачивание ветвей.
* Поиск по карте — инпут сверху; подсвечивает и центрирует совпадения.

### Сайдбар — состояние «нет активного узла» (общая сводка)

```
┌──────────────────────────────┐
│  Обзор структуры             │
│  Отделов: 12                 │
│  Сотрудников: 128            │
│  Вакантных мест: 14          │
├──────────────────────────────┤
│  Не назначенные сотрудники(6)│
│  🖼 Иванов И.    ⠿ (drag)    │
│  🖼 Петров П.    ⠿           │
├──────────────────────────────┤
│  Все сотрудники              │
│  🔍 поиск...                 │
│  🖼 ... список ... ⠿         │
└──────────────────────────────┘
```

* **Сводка:** число отделов, сотрудников, вакантных мест.
* **Не назначенные сотрудники** — есть аккаунт/карточка, но нет должности; источник кандидатов на назначение. Каждая карточка — draggable (`⠿` хэндл).
* **Все сотрудники** — полный список с поиском, тоже draggable.

### Сайдбар — состояние «активный узел» (детали отдела)

```
┌──────────────────────────────┐
│  ← Сброс выбора              │
│  Отдел: Хирургия             │
│  Руководитель: Иванов И.     │
│  Сотрудников: 8 · Вакансий: 2│
├──────────────────────────────┤
│  По должностям:              │
│  Хирург (3/4)                │
│   🖼 Иванов И.  ⠿            │
│   🖼 Сидоров К. ⠿            │
│   ▢ вакансия (0.5 ставки)    │
│  Медсестра (2/2)             │
│   🖼 ...        ⠿            │
└──────────────────────────────┘
```

* Группировка сотрудников по должностям с индикатором заполненности (занято/всего ставок).
* Вакантные позиции показаны явной строкой-плейсхолдером.
* Карточки сотрудников отдела также draggable (перевод в другой отдел).

### Модальное окно «Приказ о переводе / назначении»

Открывается при сбросе сотрудника на узел.

```
┌────────────────────────────────────────────┐
│  Перевод сотрудника                      [✕]│
├────────────────────────────────────────────┤
│  Сотрудник:  🖼 Иванов Иван Иванович         │
│  Из: Терапия · Терапевт   →   В: Хирургия    │
│                                              │
│  Отдел *        [Хирургия] (авто, read-only) │
│  Должность *    ▾  ← свободные/вакантные в    │
│                     отделе (с остатком ставок)│
│  Рабочая ставка ○ 1.0  ○ 0.5  ○ 0.25         │
│   (недоступные — disabled по остатку)         │
│                                              │
│  ┌── Первичный документ * ────────────────┐  │
│  │ ◉ Создать новый  ○ Из шаблона          │  │
│  │ ○ Загрузить файл ○ Из входящих         │  │
│  │ 📄 Приказ о переводе · Draft [Заменить]│  │
│  └────────────────────────────────────────┘  │
├────────────────────────────────────────────┤
│              [Отмена]      [Перевести]      │
└────────────────────────────────────────────┘
```

* «Отдел» авто-заполнен целевым узлом, read-only.
* «Должность» — select из свободных/вакантных должностей целевого отдела (по штатному расписанию), с подсказкой остатка ставки; занятые полностью — недоступны.
* Для не назначенного сотрудника заголовок/семантика — «Назначение на должность» (без блока «Из»).

## Визуал: High-Fidelity mockups

1. Карта — крупная интерактивная область с плавными связями-линиями; узлы-карточки в стиле дизайн-системы.
2. Активный узел выделен обводкой/тенью акцентного цвета; ветви разворачиваются/сворачиваются иконкой.
3. Индикатор вакансий на узле — бейдж/точка (красный, если есть незанятые ставки).
4. Аватары сотрудников в узле — компактный стек с «+N».

6. Сайдбар чётко разделён на смысловые блоки; переключение «сводка ↔ отдел» визуально плавное.
7. Соответствие единой дизайн-системе.

## Состояния и edge cases

### Empty
* Структура не создана — карта показывает только root/пусто + подсказка «Структура управляется в разделе Структура» (Structure Module).
* Нет не назначенных сотрудников — блок «Все назначены».

### Loading
* Скелетон карты (размытые узлы) + скелетон сайдбара при первой загрузке.

### Error
* Сбой загрузки структуры — блок ошибки на месте карты + «Повторить».
* Сбой перевода — toast, модалка остаётся открытой.

### Offline
* Карта read-only из кэша; drag-операции заблокированы, toast «Нет соединения».

### Permissions
* Без `HR_POSITION_MANAGE`/`HR_ARCHIVE_MANAGE` drag-назначение отключено (карточки не захватываются, хэндл скрыт) — карта остаётся просматриваемой.
* Без `HR_EMPLOYEE_GET` страница недоступна.

### Edge cases
* Сброс на отдел без свободных ставок — модалка открывается, но select должностей пуст/«Нет свободных ставок»; предлагается открыть штатное расписание.
* Сброс на тот же отдел/должность — операция отклоняется с подсказкой.
* Перевод сотрудника со снятием прежней должности — освобождение ставки в исходном отделе; пользователь предупреждается.
* Очень большая структура — виртуализация рендера/кластеризация узлов, мини-карта (minimap) для навигации.

## Interaction & Animation

* **Панорамирование/зум** — инерционное перетаскивание, плавный зум колесом/кнопками.
* **Разворот ветви** — анимация раскрытия дочерних узлов с reflow линий.
* **Выбор узла** — плавная подсветка + анимированное переключение содержимого сайдбара (fade/slide).
* **Drag-and-drop** — карточка «отрывается» с тенью; зоны сброса подсвечиваются в реальном времени; невалидный сброс — возврат карточки анимацией.
* **Открытие модалки перевода** — fade+scale.
* **Обновление после перевода** — целевой узел кратко подсвечивается (flash), счётчики анимированно пересчитываются.
* **Поиск** — подсвеченные совпадения пульсируют, карта плавно центрируется.

## Зависимости и ограничения

### API / интеграции
* **Structure Module** — дерево отделов, должности, субординация (источник структуры).
* `GET /hr/structure` — структура с привязкой сотрудников и вакансий (агрегация HR + Structure).
* `GET /hr/employees?unassigned=true` — не назначенные сотрудники.
* `POST /hr/employees/:id/positions` — назначение/перевод на должность (занимает ставку).
* **Staffing Schedule (1.6.5)** — свободные ставки целевого отдела.
* **EDM Module** — приказ о переводе.

### Роли / права
* Просмотр — `HR_EMPLOYEE_GET`.
* Перевод/назначение перетаскиванием — `HR_POSITION_MANAGE` (+ `HR_ARCHIVE_MANAGE` для связанных кадровых действий при необходимости).

### Технические ограничения
* Сама орг. структура (создание/изменение отделов) редактируется в Structure Module, не здесь. При этом **структурное изменение (создание/изменение/упразднение отдела или должности) — тоже движение, требующее первичного документа** (паттерн из 1.6.0); соответствующая модалка Structure Module содержит обязательный блок «Первичный документ».
* Назначение ограничено наличием свободных ставок в штатном расписании.
* Производительность на больших деревьях — виртуализация/minimap.

## Acceptance criteria

* Структура отображается интерактивным деревом; связи между узлами отражают подчинённость; внутри узлов видны сотрудники.
* Есть поиск по карте, зум и панорамирование.
* Без выбранного узла сайдбар показывает общую сводку (отделы, сотрудники, вакансии), список не назначенных сотрудников и список всех сотрудников.
* При выбранном узле сайдбар показывает детали отдела — сотрудников по должностям и вакансии.
* Сотрудника можно перетащить из любого списка сайдбара на узел отдела; при сбросе открывается модальное окно приказа о переводе/назначении.
* В форме перевода отдел авто-заполнен целевым узлом, должности — из свободных/вакантных в отделе, присутствует обязательный блок «Первичный документ»; «Перевести» неактивна без привязанного документа.
* Назначение учитывает остаток ставок штатного расписания; при их отсутствии перевод не выполняется с понятной подсказкой.
* Drag-назначение скрыто без прав; карта остаётся просматриваемой.
* Состояния empty / loading / error / offline / permissions обрабатываются корректно.
