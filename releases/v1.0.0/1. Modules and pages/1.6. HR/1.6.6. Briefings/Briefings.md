# 1.6.6 Briefings (Инструктажи)

## Спецификация страницы интерфейса

* **Название страницы:** Инструктажи
* **Модуль:** HR (Human Resources)
* **Тип страницы:** Реестр (две вкладки: Шаблоны / Прохождение)
* **Основные роли:** Администратор / HR-менеджер
* **Права доступа:** `HR_EMPLOYEE_GET` (просмотр), `HR_BRIEFING_MANAGE` (шаблоны), `HR_EMPLOYEE_BRIEFING_MANAGE` (отметки прохождения)
* **Маршрут:** `/hr/briefings`
* **Платформа:** веб-приложение медицинской системы
* **Статус документа:** Draft

> Общие паттерны — см. `1.6.0. Overview`. Отметки прохождения конкретным сотрудником также доступны из его эл. дела (1.6.4).

---

## Назначение страницы и сценарии использования

Управление инструктажами клиники: шаблоны инструктажей, привязанные к должности, и контроль их прохождения сотрудниками. Шаблон не принадлежит конкретному сотруднику, поэтому раздел — отдельный пункт меню.

С помощью страницы пользователь может:

1. Вести реестр шаблонов инструктажей (по должностям).
2. Читать полный текст инструктажа.
3. Создавать/редактировать/удалять шаблоны.
4. Контролировать прохождение: кто прошёл, кому требуется (по должности).
5. Отмечать прохождение инструктажа сотрудником.

---

## User Flows / Task Flows

### Flow 1. Управление шаблонами
1. Вкладка «Шаблоны» → список шаблонов (должность + превью).
2. «+ Создать шаблон» / `[✏]` / `[✕]` → форма / подтверждение удаления.
3. Клик по карточке — полный текст.

### Flow 2. Контроль прохождения
1. Вкладка «Прохождение» → список сотрудников с требуемыми/пройденными инструктажами.
2. Фильтр по отделу/должности/статусу (пройдено/требуется/просрочено).
3. «Отметить прохождение» → форма (сотрудник, шаблон, дата).

## Структура: Wireframes

### Вкладка «Шаблоны»
```
┌─────────────────────────────────────────────────────────────┐
│  Инструктажи   [Шаблоны][Прохождение]      [+ Создать шаблон]│
├─────────────────────────────────────────────────────────────┤
│  🔍 Поиск по должности...                                    │
├─────────────────────────────────────────────────────────────┤
│  Должность: Хирург                                           │
│  «Инструктаж по технике безопасности в операционной...»      │
│                                                  [✏] [✕]      │
├─────────────────────────────────────────────────────────────┤
│  Должность: Медсестра                                        │
│  «Инструктаж по обработке инструментов...»          [✏][✕]   │
└─────────────────────────────────────────────────────────────┘
```
* Карточка: должность + превью содержания; клик — полный текст (модалка/разворот).

**Форма «Создать/редактировать шаблон»:** должность * (Structure Module) + содержание * (многострочный текст, `text`, без rich-text).

### Вкладка «Прохождение»
```
┌─────────────────────────────────────────────────────────────┐
│  Инструктажи   [Шаблоны][Прохождение]   [+ Отметить прохожд.]│
├─────────────────────────────────────────────────────────────┤
│  🔍 Поиск по сотруднику...                  [Фильтр ▾]       │
├─────────────────────────────────────────────────────────────┤
│ Сотрудник       │ Должность│ Инструктаж        │ Статус       │
│ Иванов И.       │ Хирург   │ ТБ в операционной │ ✅ 12.01.2026│
│ Петров П.       │ Хирург   │ ТБ в операционной │ ⚠ Требуется  │
└─────────────────────────────────────────────────────────────┘
```
* Сопоставление: для должности сотрудника есть шаблон(ы) → если нет отметки прохождения → «Требуется».
* Статусы: пройдено (дата), требуется, (опц.) просрочено — по семантике бейджей.

**Форма «Отметить прохождение»:** сотрудник * + шаблон инструктажа * (рекомендуется фильтр по должности сотрудника) + дата прохождения *.

## Визуал: High-Fidelity mockups

1. Две вкладки в шапке раздела; CTA меняется по вкладке.
2. Карточки шаблонов: должность-заголовок + превью с многоточием; `[✏][✕]` на hover при праве.

4. Полный текст инструктажа — комфортная типографика.
5. Единая дизайн-система.

## Состояния и edge cases

* **Empty:** «Шаблоны не созданы» (+создать) / «Нет данных о прохождении».
* **Loading:** скелетон карточек/строк; спиннер на сохранении.
* **Error:** блок ошибки + «Повторить»; toast при сбое действий.
* **Offline:** списки из кэша; действия заблокированы, toast.
* **Permissions:** без `HR_BRIEFING_MANAGE` — режим чтения шаблонов; без `HR_EMPLOYEE_BRIEFING_MANAGE` — отметка прохождения скрыта; без `HR_EMPLOYEE_GET` — раздел недоступен.
* **Edge cases:** несколько шаблонов на должность допустимы; удаление шаблона не стирает зафиксированную историю прохождений; смена должности сотрудника меняет набор требуемых инструктажей.

## Interaction & Animation

* Переключение вкладок — fade-смена контента.
* Поиск — debounce; разворот полного текста — анимация высоты/модалка fade+scale.
* Hover карточки/строки — подсветка, появление действий; удаление с подтверждением и анимацией.
* Toast — выезд, авто-скрытие 3–5 с.

---

## Зависимости и ограничения

* **API:** `GET/POST/PATCH/DELETE /hr/briefings` (шаблоны); `/hr/employees/:id/briefings` (отметки). **Structure Module** — должности.
* Связь с 1.6.4 (статус инструктажей в рабочих данных) и 1.6.1 (виджет «требуемые инструктажи»).
* **Роли:** просмотр — `HR_EMPLOYEE_GET`; шаблоны — `HR_BRIEFING_MANAGE`; отметки — `HR_EMPLOYEE_BRIEFING_MANAGE`.
* **Ограничения:** содержание — простой текст; «осиротевший» шаблон при удалении должности — поведение на стороне Structure/HR-интеграции.

---

## Acceptance criteria

* Раздел — отдельный пункт меню с вкладками «Шаблоны» и «Прохождение».
* Шаблоны показывают должность и превью; клик открывает полный текст; форма требует должность и содержание.
* Создание/редактирование/удаление шаблонов — только при `HR_BRIEFING_MANAGE`; удаление с подтверждением.
* Вкладка «Прохождение» сопоставляет требуемые инструктажи с фактическими отметками и показывает статус.
* Форма отметки прохождения предлагает выбор сотрудника, шаблона и даты.
* Состояния empty / loading / error / offline / permissions обрабатываются корректно.
