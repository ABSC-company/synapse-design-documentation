# 1.6.14 Reports (Отчёты) — Soon

> **Статус:** Soon (планируется). Раздел под feature flag `hr_reports`. До включения — пункт меню неактивен («Скоро»). Конкретный перечень отчётов в бэкенде на момент написания не специфицирован — спецификация ниже является проектным предложением и подлежит подтверждению продуктом.

## Спецификация страницы интерфейса

* **Название страницы:** Отчёты
* **Модуль:** HR (Human Resources)
* **Тип страницы:** Каталог отчётов + конструктор параметров + просмотр/экспорт
* **Основные роли:** Администратор / HR-менеджер, Руководитель
* **Права доступа (предлагаемые):** `HR_REPORTS_GET` (просмотр/выгрузка)
* **Маршрут:** `/hr/reports`
* **Feature flag:** `hr_reports`
* **Статус документа:** Draft / Soon

---

## Назначение страницы и сценарии использования

Формирование кадровых отчётов: численность и движение персонала, заполненность штата, отпуска/больничные, учёт времени, аттестации/инструктажи, предстоящие регуляторные отчёты. Поддержка параметров (период, отдел, тип) и экспорта (XLSX/PDF/печать).

Пользователь сможет:
1. Выбрать тип отчёта из каталога.
2. Задать параметры (период, отдел, фильтры).
3. Просмотреть результат (таблица/график) и выгрузить/распечатать.
4. Видеть расписание предстоящих регуляторных отчётов (связь с дашбордом 1.6.1).

---

## User Flows / Task Flows

1. **Построение отчёта:** каталог → выбрать отчёт → задать параметры → «Сформировать» → просмотр → «Экспорт».
2. **Предстоящие отчёты:** вкладка «Расписание» → список регуляторных отчётов с дедлайнами (виджет дашборда «предстоящие отчёты»).

## Состояния и edge cases

* **Empty:** «Выберите отчёт и параметры» (до построения); «Нет данных за период» (после).
* **Loading:** индикатор формирования; скелетон результата.
* **Error/Offline:** по общим паттернам (`1.6.0`); экспорт недоступен офлайн.
* **Permissions:** без `HR_REPORTS_GET` — раздел недоступен; конкретные отчёты могут ограничиваться правами своих доменов; раздел скрыт без флага `hr_reports`.
* **Edge cases:** большой объём данных — серверная генерация + уведомление о готовности; пустые параметры — валидация перед построением.

## Зависимости и ограничения

* **API (предлагаемые):** `GET /hr/reports/catalog`, `POST /hr/reports/build`, `GET /hr/reports/export`. Данные агрегируются из доменов HR (сотрудники, штат, табель, отпуска, аттестации, инструктажи).
* **Роли:** `HR_REPORTS_GET` (предлагаемое) + доменные права на источники.
* **Ограничения:** раздел под `hr_reports`; перечень и форматы отчётов не финализированы в бэкенде — требуют подтверждения.

---

## Acceptance criteria (предварительные)

* Раздел доступен при включённом `hr_reports`, иначе «Скоро».
* Каталог предлагает набор кадровых отчётов; параметры задаются перед построением.
* Результат отображается таблицей/графиком и экспортируется (XLSX/PDF/печать).
* Вкладка «Расписание» показывает предстоящие регуляторные отчёты с дедлайнами (согласовано с виджетом дашборда).
* Состояния empty / loading / error / offline / permissions обрабатываются корректно.
