# 1.6.13 Recruiting (Рекрутинг) — Soon

> **Статус:** Soon (планируется). Раздел под feature flag `hr_recruiting`. До включения — пункт меню неактивен («Скоро»). Бизнес-логика рекрутинга в бэкенде на момент написания не специфицирована — спецификация ниже является проектным предложением и подлежит подтверждению продуктом.

## Спецификация страницы интерфейса

* **Название страницы:** Рекрутинг
* **Модуль:** HR (Human Resources)
* **Тип страницы:** Канбан-доска вакансий/кандидатов + реестр
* **Основные роли:** Администратор / HR-менеджер, Рекрутер, Руководитель отдела
* **Права доступа (предлагаемые):** `HR_RECRUITING_GET` (просмотр), `HR_RECRUITING_MANAGE` (управление)
* **Маршрут:** `/hr/recruiting`
* **Feature flag:** `hr_recruiting`
* **Статус документа:** Draft / Soon

---

## Назначение страницы и сценарии использования

Подбор персонала: ведение вакансий (на базе свободных штатных единиц), воронка кандидатов по этапам (новый → скрининг → интервью → оффер → принят/отказ), хранение резюме и заметок. По найму — точка входа в приём сотрудника (1.6.3).

Пользователь сможет:
1. Открывать вакансии на основе вакантных ставок штатного расписания (1.6.5).
2. Вести кандидатов по этапам воронки (канбан).
3. Хранить резюме/контакты/заметки по кандидату.
4. При найме — конвертировать кандидата в нового сотрудника (приём).

---

## User Flows / Task Flows

1. **Создание вакансии:** «+ Вакансия» → выбор должности/отдела (вакантные ставки) → публикация.
2. **Движение кандидата:** drag карточки кандидата между колонками этапов; смена этапа фиксируется.
3. **Найм:** кандидат в колонке «Принят» → «Оформить приём» → форма приёма сотрудника (1.6.3) с предзаполнением.

## Состояния и edge cases

* **Empty:** «Нет открытых вакансий» / «Нет кандидатов».
* **Loading/Error/Offline:** по общим паттернам (`1.6.0`); drag заблокирован офлайн.
* **Permissions:** без `HR_RECRUITING_MANAGE` — просмотр; раздел скрыт без флага `hr_recruiting`.
* **Edge cases:** вакансия без свободной ставки — предупреждение; наём при отсутствии ставки — блок/переход в штатное расписание; кандидат-дубликат по контактам — предупреждение.

## Interaction & Animation

* Drag-and-drop карточек с подсветкой целевой колонки и плавным reflow; смена этапа — flash подтверждения.
* Карточка кандидата — выезжающая панель (drawer) fade/slide. Toast — выезд, авто-скрытие 3–5 с.

## Зависимости и ограничения

* **API (предлагаемые):** `GET/POST/PATCH /hr/recruiting/vacancies`, `/candidates`. Связь со штатным расписанием (1.6.5, вакансии), приёмом (1.6.3), File Module (резюме).
* **Роли:** `HR_RECRUITING_GET` / `HR_RECRUITING_MANAGE` (предлагаемые).
* **Ограничения:** раздел под `hr_recruiting`; модель воронки/кандидатов не финализирована в бэкенде — требует подтверждения.

---

## Acceptance criteria (предварительные)

* Раздел доступен при включённом `hr_recruiting`, иначе «Скоро».
* Вакансии создаются на базе вакантных ставок штатного расписания.
* Кандидаты ведутся по этапам воронки (канбан) с drag-and-drop.
* Карточка кандидата хранит резюме, контакты, историю этапов и заметки.
* Наём конвертирует кандидата в приём сотрудника (1.6.3) с предзаполнением.
* Состояния empty / loading / error / offline / permissions обрабатываются корректно.
