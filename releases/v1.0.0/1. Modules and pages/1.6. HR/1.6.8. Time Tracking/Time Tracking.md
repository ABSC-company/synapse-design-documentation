# 1.6.8 Time Tracking (Табель учёта времени)

## Спецификация страницы интерфейса

* **Название страницы:** Табель учёта рабочего времени
* **Модуль:** HR (Human Resources)
* **Тип страницы:** Табель-сетка (сотрудники × дни) + сводка
* **Основные роли:** Администратор / HR-менеджер, Руководитель отдела
* **Права доступа:** `HR_TIME_TRACKING_GET` (просмотр), `HR_TIME_TRACKING_MANAGE` (правка отметок)
* **Маршрут:** `/hr/time-tracking`
* **Платформа:** веб-приложение медицинской системы
* **Статус документа:** Draft

> Общие паттерны — см. `1.6.0. Overview`. Учёт времени конкретного сотрудника также доступен в его эл. деле (1.6.4, календарь).

---

## Назначение страницы и сценарии использования

Сводный табель по всем сотрудникам за выбранный период: отметки прихода/ухода (`check-in`/`check-out`), отработанные часы, опоздания, пропуски. Отметки обычно поступают автоматически (внешнее устройство/интеграция); ручная правка — корректирующий сценарий.

С помощью страницы пользователь может:

1. Просматривать табель за месяц/период по сотрудникам и дням.
2. Фильтровать по отделу/должности/сотруднику.
3. Видеть сводку: отработанные часы, опоздания, пропуски.
4. Корректировать отметки (добавить/изменить/удалить) при наличии прав.
5. Экспортировать/печатать табель.

---

## User Flows / Task Flows

### Flow 1. Просмотр табеля
1. Пользователь открывает «Табель», выбирает месяц/период.
2. Система показывает сетку «сотрудники × дни» с отметками и итогами.
3. Фильтрует по отделу/сотруднику.

### Flow 2. Корректировка отметки
1. Пользователь кликает ячейку дня сотрудника.
2. Открывается поповер/модалка с отметками дня (приход/уход).
3. Добавляет/меняет/удаляет отметку (при `HR_TIME_TRACKING_MANAGE`), сохраняет.

### Flow 3. Экспорт
1. «Экспорт» → выбор формата (XLSX/PDF/печать) → выгрузка по текущему фильтру/периоду.

## Структура: Wireframes

### Общий layout
```
┌──────────────────────────────────────────────────────────────────────┐
│  Табель учёта времени      ◀ Июнь 2026 ▶          [Фильтр ▾] [Экспорт]│
├──────────────┬──┬──┬──┬──┬──┬──┬──┬──┬───┬──────────────────────────┤
│  Сотрудник   │1 │2 │3 │4 │5 │6 │7 │…│31 │ Σ часов │ Опозд.│ Пропуски │
├──────────────┼──┼──┼──┼──┼──┼──┼──┼──┼───┼─────────┼───────┼──────────┤
│ Иванов И.    │✓ │✓ │— │✓ │L │  │  │…│✓  │  168 ч  │   2   │    1     │
│ Петрова А.   │✓ │О │О │О │О │  │  │…│✓  │  152 ч  │   0   │    0     │
└──────────────┴──┴──┴──┴──┴──┴──┴──┴──┴───┴─────────┴───────┴──────────┘
   Легенда: ✓ отработан · L опоздание · — пропуск · О отпуск · Б больничный · К командировка
```

### Управление
* Переключатель периода (месяц по умолчанию; диапазон опционально).
* «Фильтр» (Drawer): отдел, должность, сотрудник, тип дня.
* «Экспорт»: XLSX / PDF / печать.

### Ячейка дня
* Состояния: отработан (есть приход+уход), опоздание, пропуск, отпуск/больничный/командировка (из статуса/графика отпусков).
* Hover/клик — детали: время прихода/ухода, отработанные часы.

**Поповер/модалка дня:**
```
┌────────────────────────────────────┐
│  Иванов И. · 05.06.2026         [✕]│
├────────────────────────────────────┤
│  ⬇ Приход   09:12   [✏][✕]         │
│  ⬆ Уход     18:03   [✏][✕]         │
│  Отработано: 8 ч 51 мин · Опоздание │
│              [+ Добавить отметку]   │
├────────────────────────────────────┤
│              [Закрыть]   [Сохранить]│
└────────────────────────────────────┘
```
* «+ Добавить отметку» / `[✏]` — тип (Приход/Уход) + дата-время. Доступно при `HR_TIME_TRACKING_MANAGE`.

### Итоговые колонки
* Σ часов за период, число опозданий, число пропусков. Внизу таблицы — итоги по выборке.

## Состояния и edge cases

* **Empty:** «Нет данных за выбранный период».
* **Loading:** скелетон сетки; спиннер при экспорте.
* **Error:** блок ошибки + «Повторить»; toast при сбое сохранения отметки.
* **Offline:** табель из кэша; правка заблокирована, toast.
* **Permissions:** без `HR_TIME_TRACKING_GET` — раздел недоступен; без `HR_TIME_TRACKING_MANAGE` — правка/добавление отметок скрыты (только просмотр и экспорт).
* **Edge cases:** приход без ухода (открытая смена) — подсветка; ночные смены (уход на следующий день) — корректный учёт; день в отпуске/командировке/больничном помечается соответствующим типом, а не пропуском; большое число сотрудников — виртуализация строк.

---

## Interaction & Animation

* Смена месяца — слайд сетки; прокрутка дней с залипающей колонкой сотрудника.
* Hover ячейки — подсветка строки/столбца и тултип с деталями.
* Поповер дня — fade+scale у ячейки; сохранение — спиннер на кнопке.
* Экспорт — индикатор подготовки файла. Toast — выезд, авто-скрытие 3–5 с.

## Зависимости и ограничения

* **API:** `GET /hr/time-tracking?period=…` (сводно); `POST/PATCH/DELETE /hr/employees/:id/time-tracking`; `GET /hr/time-tracking/export`. Источник отметок — внешнее устройство/интеграция (`employees_time_tracking`).
* **Structure Module** — отделы/должности; график отпусков (1.6.9) и статусы — для типов дней.
* Связь с 1.6.4 (календарь учёта в деле) и 1.6.3 (экспорт учёта из контекстного меню).
* **Роли:** просмотр — `HR_TIME_TRACKING_GET`; правка — `HR_TIME_TRACKING_MANAGE`.
* **Ограничения:** ручная правка — корректирующий сценарий; норма часов/правила опозданий — из графика сотрудника (1.6.4) и настроек клиники.

---

## Acceptance criteria

* Табель показывает сетку «сотрудники × дни» за выбранный период с переключением месяца.
* Ячейки отражают состояние дня (отработан/опоздание/пропуск/отпуск/больничный/командировка) с легендой.
* Итоговые колонки показывают отработанные часы, опоздания и пропуски.
* Клик по ячейке открывает детали дня; правка/добавление отметок доступны при `HR_TIME_TRACKING_MANAGE`.
* Просмотр и экспорт доступны при `HR_TIME_TRACKING_GET`.
* Состояния empty / loading / error / offline / permissions обрабатываются корректно.
