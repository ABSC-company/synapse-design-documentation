# 1.6.7 Certifications (Аттестации)

## Спецификация страницы интерфейса

* **Название страницы:** Аттестации
* **Модуль:** HR (Human Resources)
* **Тип страницы:** Реестр (таблица) с агрегацией по сотрудникам
* **Основные роли:** Администратор / HR-менеджер, Руководитель
* **Права доступа:** `HR_EMPLOYEE_GET` (просмотр), `HR_CERTIFICATION_MANAGE` (управление)
* **Маршрут:** `/hr/certifications`
* **Платформа:** веб-приложение медицинской системы
* **Статус документа:** Draft

> Общие паттерны — см. `1.6.0. Overview`. Аттестации конкретного сотрудника также доступны в его эл. деле (1.6.4, таймлайн).

---

## Назначение страницы и сценарии использования

Общеклинический реестр аттестаций — плановых, предстоящих и просроченных. Раздел отвечает на вопрос «кому и когда нужна аттестация» и позволяет управлять записями централизованно (преимущественно для врачей; `specialty_id` обязателен).

С помощью страницы пользователь может:

1. Видеть все аттестации с сотрудником, специализацией, периодом и статусом.
2. Фильтровать по статусу (запланирована/просрочена/не пройдена), отделу, специализации, периоду.
3. Создавать/редактировать/удалять аттестации.
4. Переходить в эл. дело сотрудника.

---

## User Flows / Task Flows

### Flow 1. Контроль аттестаций
1. Пользователь открывает «Аттестации».
2. По умолчанию — сортировка по ближайшей дате; просроченные подсвечены.
3. Фильтрует по статусу/специализации/периоду.

### Flow 2. Создание / редактирование
1. «+ Добавить аттестацию» (или `⋯` → «Редактировать»).
2. Заполняет форму, сохраняет.

### Flow 3. Переход в дело
1. Клик по строке → эл. дело сотрудника, секция «Аттестации».

### Фильтры (Drawer)
| Поле | Тип |
|------|-----|
| Статус | `scheduled` / `overdue` / `failed` |
| Отдел | Multi-select (Structure Module) |
| Специализация | Select (Management Module) |
| Период | Date-range |

### Таблица
| Колонка | Содержание |
|---------|-----------|
| Сотрудник | Аватар + ФИО |
| Отдел | Отдел сотрудника |
| Специализация | `specialty_id` (Management Module) |
| Категория | Квалификационная категория (если есть) |
| Период | `date` (date-range) |
| Статус | Бейдж `scheduled`/`overdue`/`failed` |
| `⋯` | Контекстное меню |

Статус-бейджи: `scheduled` — зелёный; `overdue` — красный; `failed` — красный. (Близкие к истечению — жёлтый акцент.)

Контекстное меню: Редактировать, Удалить (`HR_CERTIFICATION_MANAGE`), Открыть эл. дело.

### Форма «Добавить/редактировать аттестацию»
| Поле | Тип | Обяз. |
|------|-----|:---:|
| Сотрудник | Select (поиск по ФИО) | ✓ |
| Специализация (`specialty_id`) | Select (Management Module) | ✓ |
| Категория | Select (Management Module) | — |
| Период (`date`) | Date-range | — |
| Статус | `scheduled` / `overdue` / `failed` | ✓ |

> При создании из эл. дела сотрудника поле «Сотрудник» предзаполнено и read-only.

## Состояния и edge cases

* **Empty:** «Аттестаций нет» (+добавить); по фильтру — «Ничего не найдено».
* **Loading:** скелетон строк; спиннер на сохранении.
* **Error:** блок ошибки + «Повторить»; toast при сбое.
* **Offline:** таблица из кэша; управление заблокировано, toast.
* **Permissions:** без `HR_CERTIFICATION_MANAGE` — режим чтения; без `HR_EMPLOYEE_GET` — раздел недоступен.
* **Edge cases:** аттестация без периода (`date` nullable) — статус задаётся вручную; авто-перевод `scheduled → overdue` по наступлении даты (на стороне бэкенда), UI отражает актуальный статус; аттестации преимущественно для врачей.

## Interaction & Animation

* Статусные вкладки — fade-смена; сортировка/фильтр — мягкий скелетон при обновлении.
* Drawer — slide-in; форма — fade+scale.
* Hover строки — подсветка, `⋯`; удаление с подтверждением.
* Toast — выезд, авто-скрытие 3–5 с.

---

## Зависимости и ограничения

* **API:** `GET /hr/certifications` (агрегированный список); `POST/PATCH/DELETE /hr/employees/:id/certifications`. **Management Module** — специализации/категории; **Structure Module** — отделы.
* Связь с 1.6.4 (таймлайн в деле) и 1.6.1 (виджет «предстоящие аттестации»).
* **Роли:** просмотр — `HR_EMPLOYEE_GET`; управление — `HR_CERTIFICATION_MANAGE`.
* **Ограничения:** `specialty_id` обязателен; статусы `scheduled`/`overdue`/`failed`.

---

## Acceptance criteria

* Реестр показывает сотрудника, специализацию, период и статус аттестации; сортируется по ближайшей дате.
* Фильтрация по статусу, отделу, специализации и периоду работает.
* Просроченные аттестации визуально выделены.
* Форма требует сотрудника, специализацию и статус; период необязателен.
* Клик по строке открывает эл. дело сотрудника.
* Управление доступно только при `HR_CERTIFICATION_MANAGE`.
* Состояния empty / loading / error / offline / permissions обрабатываются корректно.
