# 1.6.9 Vacation Schedule (График отпусков)

## Спецификация страницы интерфейса

* **Название страницы:** График отпусков
* **Модуль:** HR (Human Resources)
* **Тип страницы:** Таймлайн/диаграмма Ганта (сотрудники × календарь) + реестр
* **Основные роли:** Администратор / HR-менеджер, Руководитель отдела
* **Права доступа:** `HR_EMPLOYEE_GET` (просмотр), `HR_EMPLOYEE_STATUS_UPDATE` (оформление отпуска)
* **Маршрут:** `/hr/vacations`
* **Платформа:** веб-приложение медицинской системы
* **Статус документа:** Draft
* **Feature flag:** `hr_vacations`

> Общие паттерны — см. `1.6.0. Overview`. Отправка сотрудника в отпуск также выполняется из его эл. дела (1.6.4).

---

## Назначение страницы и сценарии использования

Годовой график отпусков клиники в виде диаграммы-таймлайна: кто и когда в отпуске, планируемые и фактические периоды, пересечения внутри отдела. Помогает планировать отпуска без оголения отделов и контролировать остаток отпускных дней.

С помощью страницы пользователь может:

1. Видеть отпуска всех сотрудников на годовой шкале (по отделам).
2. Различать запланированные и фактические (утверждённые) отпуска.
3. Выявлять пересечения отпусков в отделе.
4. Планировать/оформлять отпуск (период, тип) с переходом в ЭДО.
5. Отслеживать остаток отпускных дней по сотруднику.

---

## User Flows / Task Flows

### Flow 1. Просмотр графика
1. Пользователь открывает «График отпусков».
2. Система показывает таймлайн на год; строки — сотрудники, сгруппированные по отделам.
3. Фильтрует по отделу/сотруднику/году.

### Flow 2. Планирование отпуска
1. Пользователь нажимает «+ Запланировать отпуск» (или выделяет диапазон на шкале сотрудника).
2. Открывается форма: сотрудник, период, тип отпуска.
3. Сохранение → запланированный отпуск появляется на шкале.

### Flow 3. Утверждение / оформление
1. Пользователь открывает запланированный отпуск → «Оформить».
2. Форма с переходом в ЭДО (приказ folder `leave`).
3. Подтверждение → отпуск становится фактическим; при наступлении периода статус сотрудника → `on_leave`.

## Структура: Wireframes

### Общий layout (Гант)
```
┌──────────────────────────────────────────────────────────────────────────┐
│  График отпусков   [Год 2026 ▾]            [Фильтр ▾]  [+ Запланировать]   │
├───────────────────┬────────────────────────────────────────────────────────┤
│  Сотрудник        │ Янв Фев Мар Апр Май Июн Июл Авг Сен Окт Ноя Дек         │
├───────────────────┼────────────────────────────────────────────────────────┤
│  ▸ Хирургия                                                                 │
│    Иванов И.      │              ▓▓▓▓(план)        ███(факт)                │
│    Сидоров К.     │      ███                                                 │
│  ▸ Терапия                                                                  │
│    Петрова А.     │                         ▓▓▓▓▓                            │
└───────────────────┴────────────────────────────────────────────────────────┘
   Легенда: ███ утверждён · ▓▓▓ запланирован · ⚠ пересечение
```

### Управление
* Селектор года; «Фильтр» (Drawer): отдел, сотрудник, тип отпуска, статус (план/факт).
* Группировка строк по отделам (сворачиваемые группы).

### Полоса отпуска (на шкале)
* Цвет/паттерн: утверждён (сплошной), запланирован (штриховка).
* Пересечения в отделе подсвечиваются (⚠), чтобы избежать одновременного отсутствия.
* Hover — тултип: сотрудник, период, тип, статус; клик — карточка/действия.

### Форма «Запланировать/оформить отпуск»
```
┌────────────────────────────────────────┐
│  Отпуск                              [✕]│
├────────────────────────────────────────┤
│  Сотрудник *      ▾ (поиск по ФИО)       │
│  Тип отпуска *    ▾ (ежегодный/без сохр.│
│                      /учебный/…)         │
│  Период *         [с] – [по]             │
│  Остаток дней: 14 (подсказка)            │
│  Комментарий                             │
│  ┌── Кадровый документ ───────────────┐  │
│  │  [Создать приказ в ЭДО →]          │  │
│  └─────────────────────────────────────┘  │
├────────────────────────────────────────┤
│       [Отмена]  [Запланировать/Оформить]│
└────────────────────────────────────────┘
```
* «Остаток дней» — подсказка по доступным отпускным дням сотрудника.
* Тип отпуска — справочник (ежегодный, без сохранения, учебный и т.п.).
* Оформление сопровождается приказом ЭДО (folder `leave`).

### Реестр-режим (альтернативный вид)
Переключатель «Таймлайн / Список»: таблица отпусков (Сотрудник · Отдел · Тип · Период · Статус · Дней) с фильтрами и сортировкой.

## Визуал: High-Fidelity mockups

1. Годовая Гант-диаграмма с залипающей колонкой сотрудников и прокручиваемыми месяцами; «сегодня» — вертикальная линия.
2. Полосы отпусков различимы по статусу (план/факт); пересечения внутри отдела подсвечены предупреждающим цветом.
3. Группы отделов сворачиваются; легенда статусов видна.
4. Переключатель «Таймлайн/Список». Единая дизайн-система.

## Состояния и edge cases

* **Empty:** «Отпуска не запланированы» (+запланировать).
* **Loading:** скелетон таймлайна/таблицы; спиннер на сохранении.
* **Error:** блок ошибки + «Повторить»; toast при сбое.
* **Offline:** график из кэша; планирование заблокировано, toast.
* **Permissions:** без `HR_EMPLOYEE_GET` — раздел недоступен; без `HR_EMPLOYEE_STATUS_UPDATE` — планирование/оформление скрыты (только просмотр); раздел скрыт без флага `hr_vacations`.
* **Edge cases:** пересечение отпусков в отделе — предупреждение (не жёсткий запрет); превышение остатка дней — предупреждение/блок по политике клиники; перенос/отмена запланированного отпуска; авто-смена статуса в `on_leave` при наступлении периода и возврат в `at_work` по окончании.

## Interaction & Animation

* Прокрутка месяцев — горизонтальный слайд с залипающей колонкой; сворачивание групп — анимация высоты.
* Выделение диапазона на шкале — drag для создания отпуска с живым предпросмотром полосы.
* Hover полосы — тултип; клик — карточка действий (fade+scale).
* Предупреждение о пересечении — мягкая пульсация акцента. Toast — выезд, авто-скрытие 3–5 с.

---

## Зависимости и ограничения

* **API:** `GET /hr/vacations?year=…`; `POST/PATCH/DELETE /hr/vacations`; оформление → `PATCH /hr/employees/:id/status` (`on_leave`) + ЭДО (folder `leave`).
* **Structure Module** — отделы; **EDM Module** — приказы; связь со статусами сотрудника и табелем (1.6.8) и дашбордом (1.6.1, «предстоящие отпуска»).
* **Роли:** просмотр — `HR_EMPLOYEE_GET`; оформление — `HR_EMPLOYEE_STATUS_UPDATE`.
* **Ограничения:** раздел под feature flag `hr_vacations`; остаток дней/типы отпусков — по настройкам/политике клиники (детальная бизнес-логика отпусков в бэкенде на момент написания не специфицирована — заполнено проектным предположением, требует подтверждения продуктом).

---

## Acceptance criteria

* График отображает отпуска всех сотрудников на годовой шкале, сгруппированных по отделам.
* Запланированные и фактические отпуска визуально различимы; пересечения в отделе подсвечены.
* Планирование/оформление отпуска через форму с периодом, типом, остатком дней и переходом в ЭДО.
* Оформление отпуска при наступлении периода переводит сотрудника в `on_leave`.
* Доступен альтернативный список-вид с фильтрами.
* Раздел управляется feature-флагом `hr_vacations`; планирование скрыто без права.
* Состояния empty / loading / error / offline / permissions обрабатываются корректно.
