# 1.6.12 Interns / Residents (Интерны / резиденты) — Soon

> **Статус:** Soon (планируется). Раздел под feature flag `hr_interns`. До включения — пункт меню неактивен («Скоро»). Бизнес-логика интернатуры/резидентуры в бэкенде на момент написания не специфицирована — спецификация ниже является проектным предложением и подлежит подтверждению продуктом.

## Спецификация страницы интерфейса

* **Название страницы:** Интерны / резиденты
* **Модуль:** HR (Human Resources)
* **Тип страницы:** Реестр + карточка программы обучения
* **Основные роли:** Администратор / HR-менеджер, Руководитель отдела, Наставник
* **Права доступа (предлагаемые):** `HR_EMPLOYEE_GET` (просмотр), `HR_INTERN_MANAGE` (управление)
* **Маршрут:** `/hr/interns`
* **Feature flag:** `hr_interns`
* **Статус документа:** Draft / Soon

---

## Назначение страницы и сценарии использования

Учёт интернов (резидентов): программа обучения, наставник, период, этапы/ротации, прогресс. Интерн — особый тип сотрудника с обучающим треком; по завершении может переводиться в штатного сотрудника/врача.

Пользователь сможет:
1. Вести реестр интернов с программой, наставником и периодом.
2. Отслеживать прогресс по этапам/ротациям.
3. Назначать наставника и закреплять за отделом.
4. По завершении — перевести в штат (точка входа в назначение должности, 1.6.2/1.6.4).

---

## User Flows / Task Flows

1. **Просмотр:** реестр интернов → фильтр по отделу/наставнику/статусу программы.
2. **Карточка интерна:** программа, наставник, период, этапы/ротации, прогресс, аттестации.
3. **Завершение программы:** «Перевести в штат» → назначение должности (через орг. структуру/эл. дело) с приказом ЭДО.

## Состояния и edge cases

* **Empty:** «Интернов нет».
* **Loading/Error/Offline:** по общим паттернам (`1.6.0`).
* **Permissions:** без `HR_INTERN_MANAGE` — просмотр; раздел скрыт без флага `hr_interns`.
* **Edge cases:** интерн без назначенного наставника — подсветка; завершение программы → перевод в штат; досрочное прекращение программы.

## Зависимости и ограничения

* **API (предлагаемые):** `GET /hr/interns`, `POST/PATCH /hr/interns`. Связь с сотрудниками (тип занятости), аттестациями (1.6.7), орг. структурой (1.6.2), ЭДО.
* **Роли:** `HR_EMPLOYEE_GET` / `HR_INTERN_MANAGE` (предлагаемые).
* **Ограничения:** раздел под `hr_interns`; модель интернатуры не финализирована в бэкенде — требует подтверждения.

---

## Acceptance criteria (предварительные)

* Раздел доступен при включённом `hr_interns`, иначе «Скоро».
* Реестр показывает интернов с отделом, наставником, периодом и прогрессом программы.
* Карточка отражает этапы/ротации и связанные аттестации.
* Завершение программы ведёт к переводу в штат (назначение должности с приказом).
* Состояния empty / loading / error / offline / permissions обрабатываются корректно.
