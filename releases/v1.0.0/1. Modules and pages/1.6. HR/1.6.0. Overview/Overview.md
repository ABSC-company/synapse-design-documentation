# 1.6.0 HR Module — Обзор и навигация

## Спецификация модуля

* **Модуль:** HR Module (управление персоналом)
* **Тип:** раздел внутренней админки клиники
* **Платформа:** веб-приложение медицинской системы
* **Основные роли:** Администратор / HR-менеджер, Руководитель отдела, Сотрудник (просмотр своих данных)
* **Статус документа:** Draft

> Этот файл — навигационная и паттерн-сводка модуля. Он не описывает отдельную страницу по структуре спецификации, а фиксирует общие правила, на которые ссылаются страницы 1.6.1–1.6.14 без повторного описания механики.

## Что НЕ входит в модуль (точки интеграции)

* **Загрузка и хранение файлов** (фото, сканы документов, сертификаты, архивы, объективки) — File Module. В HR-экранах используется единый паттерн «Загрузить файл».
* **Кадровый документооборот (приказы)** — EDM Module. HR Module — bucket-owner документов категории `hr`. Кадровые действия (приём, перевод, отпуск, командировка, увольнение) сопровождаются приказом. Реестр и редактор приказов описаны в модуле 1.5.
* **Орг. структура как источник справочников** (отделы, должности, рабочие места, субординация) — Structure Module. Страница 1.6.2 визуализирует структуру и инициирует кадровые действия, но сами справочники ведутся в Structure Module.
* **Глобальные настройки** (стандарт гендера, списки специализаций и категорий врачей) — Management Module.
* **Системные аккаунты / базовые роли** — Auth Module.

## Общие UI-паттерны

Зафиксированы один раз; страницы ссылаются на них без повторного описания.

### Паттерн «Реестр» (список сущностей)

```
┌─────────────────────────────────────────────────────────────────┐
│  Заголовок страницы                          [+ Кнопка создания] │
├─────────────────────────────────────────────────────────────────┤
│  [Вкладки/табы, если есть]                                        │
│  🔍 Поиск...                              [Фильтр ▾] [Сортировка] │
├─────────────────────────────────────────────────────────────────┤
│  Таблица / список карточек                                        │
├─────────────────────────────────────────────────────────────────┤
│  Пагинация                                                        │
└─────────────────────────────────────────────────────────────────┘
```

* Поиск — debounce 300–500 мс; набор искомых полей указан на странице.
* Фильтр — боковая выезжающая панель (Drawer) или dropdown.
* Длинные списки — серверная пагинация или infinite scroll.

### Паттерн «Форма создания/редактирования» (модальное окно)

* Обязательные поля помечены `*`.
* «Сохранить» неактивна, пока форма не валидна.
* Необратимое удаление — отдельное подтверждающее окно.

### Паттерн «Массовый выбор + Action Bar»

* Чекбокс появляется при hover строки / в режиме выбора.
* При выборе ≥1 элемента снизу появляется Action Bar: «Выбрано: N» + массовые действия (например, «Удалить») + «Снять выбор».

### Паттерн «Контекстное меню»

* Открывается кнопкой `⋮` в строке и по правому клику (ПКМ) на строке.
* Набор действий зависит от сущности и прав.

### Паттерн «Статус-бейдж»

Единая цветовая семантика модуля:

| Семантика | Где используется | Цвет |
|-----------|------------------|------|
| Активно / в порядке | `at_work`, документ действующий, аттестация `scheduled` | Зелёный |
| Внимание / скоро истекает | `on_leave`, `on_trip`, документ/аттестация истекает ≤30 дней | Жёлтый/янтарный |
| Критично / просрочено | аттестация `overdue`/`failed`, документ истёк, просроченный инструктаж | Красный |
| Нейтрально / неактивно | `away`, `dismissed`, бессрочный документ | Серый |

### Паттерн «Пустое / Загрузка / Ошибка»

* **Empty** — иллюстрация + поясняющий текст + кнопка создания.
* **Loading** — скелетоны (списки/карточки/профиль); спиннер для точечных действий в формах.
* **Error** — блок с иконкой, текстом и кнопкой «Повторить» (для списков); toast (для действий); инлайн под полем (для валидации).

### Паттерн «Нет прав»

* Нет права на просмотр — раздел/вкладка скрыты из навигации.
* Есть просмотр, нет редактирования — интерактивные элементы скрыты или disabled с tooltip «Недостаточно прав».

### Паттерн «Загрузка файла» (File Module)

```
┌─────────────────────────────────────────┐
│   📎  Перетащите файл или нажмите         │
│   PDF, JPG, PNG до N МБ                   │
└─────────────────────────────────────────┘
```

После загрузки — превью + название + [Заменить] [Удалить]. Внутренняя логика — File Module.

### Паттерн «Переход в ЭДО» (EDM Module)

Везде, где кадровое действие требует приказа:

```
┌─────────────────────────────────────────┐
│  Кадровый документ                       │
│  Это действие требует приказа.           │
│  [Создать документ в ЭДО →]              │
└─────────────────────────────────────────┘
```

Приказ можно создать сразу или позже; привязка не блокирует завершение действия.

### Паттерн «Объективка»

Объективка — печатная сводная карточка сотрудника (персональные + рабочие данные). Доступна как действие «Скачать / Распечатать объективку» из реестра сотрудников и электронного дела. Генерация — на стороне бэкенда/File Module; на UI — кнопка и выбор формата (PDF / печать).

## Статусы сотрудника и матрица переходов

| Статус | Описание | Бейдж |
|--------|----------|-------|
| `at_work` | На рабочем месте | Зелёный |
| `away` | Вне рабочего места / больничный | Серый |
| `on_leave` | В отпуске | Жёлтый |
| `on_trip` | В командировке | Жёлтый |
| `dismissed` | Уволен (в архиве) | Серый |

```
at_work   → away | on_leave | on_trip | dismissed
away      → at_work
on_leave  → at_work
on_trip   → at_work
dismissed → at_work  (только через процедуру восстановления из архива)
```

* Переход в `dismissed` запускает архивирование (сборка полного дампа данных, перенос в `dismissed_employees_archives`).
* Уволенные не отображаются в активном реестре; для них — фильтр/раздел «Архив» внутри 1.6.3.
* Переходы `on_leave` / `on_trip` логически связаны с графиком отпусков (1.6.9) и приказами ЭДО (folders `leave` / `business_trip`).

## Acceptance criteria — общие (для всех экранов модуля)

* Навигация HR содержит активные пункты: Дашборд, Орг. структура, Сотрудники, Штатное расписание, Инструктажи, Аттестации, Табель, График отпусков, Кадровое делопроизводство; и Soon-пункты (Испытательные сроки, Интерны, Рекрутинг, Отчёты).
* Электронное дело — вложенный экран, открываемый из реестра/карты/списков, а не пункт меню.
* Все реестры поддерживают поиск, фильтрацию, пагинацию и состояния пусто/загрузка/ошибка.
* Статус-бейджи во всех разделах используют единую цветовую семантику.
* Формы — модальные окна с `*`-полями и неактивной кнопкой сохранения до валидации.
* Контекстное меню доступно по `⋮` и по ПКМ.
* Кадровые действия предлагают переход в ЭДО без встраивания редактора документов.
* Элементы без прав скрыты или disabled с tooltip.
* Soon-разделы управляются feature-флагами и до включения показываются как «Скоро».
