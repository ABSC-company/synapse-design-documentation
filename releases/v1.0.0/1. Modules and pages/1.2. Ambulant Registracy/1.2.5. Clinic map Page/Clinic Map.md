# 1.2.5. Clinic Map Page

## Спецификация страницы интерфейса

- Страница: Карта клиники
- Модуль: Амбулаторная регистратура
- Позиция в модуле: Четвертая страница интерфейса сотрудника регистратуры
- Тип страницы: Навигационная карта территории и внутренних помещений медицинского учреждения
- Основные роли: Регистратор, старший регистратор, администратор регистратуры
- Статус документа: Draft

---

# Назначение страницы и сценарии использования

Страница предназначена для поиска и навигации по территории медицинского учреждения, включая поиск зданий, корпусов, этажей, отделений, кабинетов и врачей.

Ключевые задачи страницы:

1. Просмотр карты территории медицинского учреждения.
2. Поиск отделений через строку поиска.
3. Определение расположения отделения:
   - здание;
   - корпус;
   - этаж.
4. Выбор корпуса и этажа.
5. Переход к карте выбранного этажа.
6. Поиск врача по ФИО.
7. Просмотр расположения кабинета врача.
8. Просмотр состава кабинета через Drawer.

Общий layout страницы:

1. Слева:
   - Sidebar в свернутом состоянии.

2. Сверху:
   - Header модуля регистратуры.

3. Контентная область:

Первый контент:

- Карта территории клиники
- Поиск отделений

Второй контент:

- Карта выбранного этажа
- Поиск врачей
- Drawer кабинета

# User Flows / Task Flows для страницы

## Flow 1. Просмотр карты территории

1. Пользователь открывает страницу Карта клиники.
2. Система отображает карту территории учреждения.
3. На карте отображаются:
   - здания;
   - входы;
   - парковка;
   - лифты;
   - навигационные элементы.
4. Пользователь просматривает карту.

---

## Flow 2. Поиск отделения

1. Пользователь вводит название отделения.
2. Система выполняет поиск.
3. Под инпутом отображается dropdown со списком найденных результатов.
4. Для каждого результата отображается:

- название отделения;
- здание;
- корпус;
- этаж.

Пример:

Хирургическое отделение  
Корпус А — 3 этаж

Хирургия взрослых  
Корпус Б — 2 этаж

Детская хирургия  
Корпус В — 1 этаж

5. Dropdown остается открытым пока:

- поле не очищено;
- пользователь не выбрал результат;
- пользователь не потерял фокус.

6. Пользователь выбирает найденный результат.

7. Система автоматически:

- выбирает здание;
- выбирает корпус;
- выбирает этаж;
- открывает карту этажа.

---

## Flow 3. Выбор здания и корпуса

1. Пользователь нажимает на здание.
2. Система выделяет выбранное здание.
3. Система отображает список корпусов.
4. Пользователь выбирает корпус.
5. Система отображает этажи корпуса.
6. Пользователь выбирает этаж.
7. Система открывает карту выбранного этажа.

---

## Flow 4. Поиск врача

1. Пользователь вводит ФИО врача.
2. Система выполняет поиск.
3. Отображается dropdown результатов.

Для каждого результата отображается:

- ФИО врача;
- отдел;
- кабинет.

Пример:

Иванов Александр Сергеевич  
Кардиология — кабинет 105

Петров Андрей Викторович  
Хирургия — кабинет 111

4. Пользователь выбирает врача.

5. Система:

- подсвечивает кабинет;
- центрирует карту;
- открывает Drawer кабинета.

---

## Flow 5. Просмотр информации кабинета

1. После выбора врача открывается Drawer.
2. Drawer отображает:

- номер кабинета;
- отдел;
- список врачей кабинета;
- специализации врачей.

Пример:

Кабинет: 111

Отдел:
Хирургия

Врачи:

Иванов А.С.
Хирург

Петров А.А.
Травматолог

Сидоров И.И.
Ортопед

# Состояния и edge cases

## Базовые состояния страницы

| Состояние | Ожидаемое поведение |
|------------|--------------------|
| loading | Skeleton карты и toolbar |
| empty | Данные отсутствуют |
| error | Ошибка загрузки |
| offline | Отсутствует сеть |
| permissions | Ограничение доступа |

---

## Edge cases: карта территории

1. У здания отсутствуют корпуса.
2. У корпуса отсутствуют этажи.
3. Карта здания недоступна.
4. Найдено слишком большое количество результатов.
5. Поиск не дал результатов.
6. Здание временно недоступно.

---

## Edge cases: поиск отделений

1. Поиск пустой.
2. Отделение не найдено.
3. Найдено несколько отделений с одинаковым названием.
4. Отделение существует, но этаж не определен.
5. Dropdown не открылся.
6. Dropdown не закрывается.

---

## Edge cases: поиск врача

1. Врач не найден.
2. Несколько врачей имеют одинаковое ФИО.
3. Врач работает в нескольких кабинетах.
4. Врач временно отсутствует.
5. Кабинет не найден.

---

## Edge cases: Drawer

1. Кабинет не содержит врачей.
2. Кабинет содержит врачей разных отделений.
3. Drawer не загрузился.
4. Отсутствуют данные кабинета.

---

# Interaction & Animation

## Правила поведения

1. Поиск работает без перезагрузки страницы.
2. Выбор здания выполняется мгновенно.
3. Выбор корпуса обновляет список этажей.
4. Выбор этажа открывает карту.
5. Выбор врача центрирует карту.
6. Drawer открывается автоматически.

---

## Переходы и микроанимации

1. Выбор здания сопровождается fade-анимацией.
2. Карта плавно центрируется на найденном объекте.
3. Drawer открывается slide-анимацией слева.
4. Подсветка кабинета появляется плавно.
5. Dropdown появляется через fade.

## Технические ограничения

1. Карта должна поддерживать масштабирование.
2. Карта должна поддерживать панорамирование.
3. При большом количестве объектов требуется виртуализация.
4. Поиск работает с debounce.
5. Центрирование карты должно работать без полной перезагрузки.
