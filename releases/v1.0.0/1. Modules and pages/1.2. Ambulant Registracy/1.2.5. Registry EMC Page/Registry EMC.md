# 1.2.5. Registry EMC Page

## Спецификация страницы интерфейса

- Страница: ЭМК пациента (регистратура)
- Модуль: Амбулаторная регистратура
- Позиция в модуле: Пятая по счёту страница интерфейса сотрудника регистратуры
- Тип страницы: Просмотр данных пациента
- Основные роли: Регистратор, старший регистратор, администратор регистратуры
- Статус документа: Draft

## Назначение страницы и сценарии использования

Страница предназначена для управления всеми визитами, консультациями, инструментальными и лабораторными исследованиями пациента. Единая рабочая область с переключаемым контентом через Tab-меню.

Ключевые задачи страницы:

1. Просмотр всех визитов пациента в одном месте.
2. Просмотр консультаций врачей с информацией о враче, услуге и статусе.
3. Просмотр инструментальных исследований с доступом к результатам и файлам.
4. Просмотр лабораторных анализов с индикаторами результатов и графиком тренда.
5. Поиск по различным параметрам (врач, название услуги, дата).
6. Фильтрация записей по статусу, типу, периоду дат.
7. Просмотр деталей каждого элемента через модальное окно.
8. Управление статусами записей в соответствии с ролью пользователя.

Общий layout страницы:

1. Слева: Sidebar в свернутом состоянии.
2. Сверху: **Page Header** — заголовочный блок с кнопками управления (см. [Page Header.md](../1.2.0. Module layout components/1.2.0.1. Page Header/Page Header.md)).
3. Справа: **Electronic Queue** — блок электронной очереди по правому краю (см. [Electronic Queue.md](../1.2.0. Module layout components/1.2.0.2. Electronic Queue/Electronic queue.md)).
4. Контентная область: Tab-меню, Toolbar (поиск, фильтр, кнопка действия), таблица/список контента.

**Page Header содержит две основные кнопки действий:**

1. **Зарегистрировать пациента** — открывает модаль регистрации нового пациента (см. подробнее в [Page Header.md — «Кнопка «Зарегистрировать пациента»**](../1.2.0. Module layout components/1.2.0.1. Page Header/Page Header.md))
2. **Запись пациента** — открывает модаль создания новой записи пациента к врачу (см. подробнее в [Page Header.md — «Кнопка «Запись пациента к врачу»**](../1.2.0. Module layout components/1.2.0.1. Page Header/Page Header.md))

## User Flows / Task Flows для страницы

### Flow 1. Просмотр всех визитов (вкладка "Все визиты")

1. Пользователь открывает страницу Визиты пациента.
2. По умолчанию открыта вкладка "Все визиты".
3. Система загружает таблицу со всеми записями пациента (консультации, исследования).
4. Таблица содержит колонки: Дата, Клиника, Врач, Направление, Статус, Тип визита.
5. Пользователь просматривает все визиты в хронологическом порядке (новые сверху).
6. Каждый элемент может быть раскрыт для просмотра полной информации.

### Flow 2. Просмотр консультаций врачей

1. Пользователь нажимает на вкладку "Консультация врача".
2. Таблица переключается на отображение только консультаций.
3. Таблица содержит колонки: Дата, Клиника, Врач, Услуга, Статус, Тип документа.
4. Каждая строка содержит информацию о консультации, врача и услуге.
5. Пользователь может просмотреть статусы: Ожидание, На приеме, Завершено, Отменено.
6. Доступна информация о типах документов (DCM, EXE, DOC и т.д.).

### Flow 3. Просмотр инструментальных исследований

1. Пользователь нажимает на вкладку "Инструментальные исследования".
2. Таблица переключается на отображение только инструментальных тестов.
3. Таблица содержит колонки: Дата, Наименование, Врач, Кабинет, Статус, Тип файла.
4. Каждая строка содержит информацию об исследовании, враче-специалисте и кабинете.
5. Пользователь может просмотреть статусы: Назначено, Проводится, Завершено, Выдано.
6. Доступна информация о типах прикрепленных файлов (протоколы, снимки, графики).

### Flow 4. Просмотр лабораторных исследований

1. Пользователь нажимает на вкладку "Лабораторные исследования".
2. Таблица переключается на отображение только лабораторных анализов.
3. Таблица содержит колонки: Дата, Наименование, Лаборатория, Статус, Результат.
4. Каждая строка содержит цветной индикатор результата слева от названия:
    - Зеленый: результат в норме
    - Желтый: результат требует внимания
    - Красный: критичное отклонение
5. Пользователь может просмотреть статусы: Ожидание сдачи, Обработка, Завершено, Утверждено.
6. Цветовой индикатор дает быструю визуальную оценку результата.

### Flow 5. Поиск по названию услуги или врачу

1. Пользователь вводит текст в инпут поиска (название услуги, ФИО врача или лаборатория).
2. Система фильтрует таблицу в реальном времени (debounce 300ms).
3. Совпадения ищутся в названии, враче, услуге, в зависимости от текущей вкладки.
4. При отсутствии результатов отображается сообщение "Результаты не найдены".
5. Пользователь может очистить поиск кнопкой X в инпуте.

### Flow 6. Применение расширенных фильтров

1. Пользователь нажимает кнопку Фильтр в toolbar.
2. Открывается Drawer фильтрации справа.
3. Доступные фильтры зависят от текущей вкладки:
    - Все визиты: тип визита, статус, период дат
    - Консультация: врач, услуга, статус, период дат
    - Инструментальные: тип исследования, статус, врач, период дат
    - Лабораторные: тип анализа, статус, лаборатория, результат, период дат
4. Пользователь выбирает нужные параметры.
5. Нажимает кнопку Применить для обновления таблицы.
6. Кнопка Сброс очищает все фильтры.

### Flow 7. Просмотр деталей записи через модальное окно

1. Пользователь нажимает на строку таблицы.
2. Открывается модальное окно с полной информацией о записи.
3. Модаль содержит разные разделы в зависимости от типа:
    - **Консультация**: врач, услуга, дата, время, результаты, прикрепленные документы, история статусов
    - **Инструментальное исследование**: название, врач, кабинет, дата, результаты, файлы (протоколы, снимки), история статусов
    - **Лабораторный анализ**: название, лаборатория, дата, результаты с нормами, индикаторы, история анализов, график тренда
4. Пользователь может просмотреть все детали без перехода на другую страницу.
5. Модаль закрывается кнопкой X или кликом на backdrop.

### Flow 8. Управление статусами записей

1. Пользователь находится в модали записи (Flow 7).
2. При наличии прав роли доступна кнопка Изменить статус или быстрые кнопки действий.
3. Система отображает список возможных переходов статуса для текущей записи.
4. Пользователь выбирает новый статус.
5. Система обновляет статус и добавляет запись в историю.
6. Отображается toast-сообщение об успешном обновлении.

### Flow 9. Скачивание файлов (для Консультаций и Инструментальных исследований)

1. Пользователь находится в модали записи с файлами.
2. В разделе Файлы отображаются прикрепленные документы с иконками типов.
3. Пользователь нажимает на файл или кнопку Скачать.
4. Система загружает файл на компьютер.
5. При большом размере файла отображается прогресс-бар.

### Flow 10. Просмотр графика тренда (для Лабораторных анализов)

1. Пользователь находится в модали лабораторного анализа.
2. Нажимает на вкладку "История" или "График".
3. Система загружает все предыдущие анализы этого же типа.
4. Отображается линейный график с историей значений.
5. На графике видны нормативные границы (горизонтальные линии).
6. Пользователь может наводить на точки для просмотра точных значений.

## Состояния и edge cases

### Базовые состояния страницы

| Состояние   | Ожидаемое поведение                                                                                     |
| ----------- | ------------------------------------------------------------------------------------------------------- |
| loading     | Skeleton профиля, toolbar и таблицы. Skeleton таблицы, блокировка интерактивных действий              |
| empty       | Нет данных на выбранную вкладку или период. Сообщение "Нет результатов" с CTA смены фильтров         |
| error       | Ошибка загрузки данных с сообщением об ошибке и кнопкой Повторить                                     |
| offline     | Баннер отсутствия интернет-соединения. Локальные данные доступны только для чтения                     |
| permissions | Скрытие кнопок редактирования для ролей без прав                                                      |

### Edge cases: таблица и переключение вкладок

1. При смене вкладки таблица загружается заново (loading state).
2. Если для выбранной вкладки нет данных: отображается empty state с соответствующим сообщением.
3. Поиск и фильтры из предыдущей вкладки не применяются к новой вкладке (каждая вкладка имеет собственное состояние).
4. Очень длинное название услуги/исследования обрезается с ellipsis, полный текст доступен в tooltip.
5. Таблица содержит 500+ записей: виртуализация для оптимизации производительности.
6. При быстром переключении вкладок ответы API приходят в разном порядке: отменяется предыдущий запрос.
7. Доктор не назначен исследованию: колонка Врач отображает "—" или "Не назначен".
8. Исследование без файлов: колонка Тип файла остается пустой.

### Edge cases: консультация врача

1. Консультация без заполненных результатов: раздел Результаты в модали показывает "Результаты не заполнены".
2. Документ не может быть загружен: отображение иконки ошибки и кнопка Повторить.
3. Очень большой файл: отображение прогресс-бара при скачивании.
4. История статусов очень длинная: таблица истории становится scrollable.
5. Комплексная услуга с несколькими документами разных типов: все отображаются в файлах.

### Edge cases: инструментальное исследование

1. Исследование в статусе "Назначено": кабинет может быть не определен до проведения.
2. Исследование в статусе "Проводится": отображается прогресс или статус выполнения.
3. Исследование с множественными файлами (10+): файлы могут быть сгруппированы или спрятаны в expander.
4. Снимок/протокол очень большого размера: предпросмотр может быть недоступен, только скачивание.
5. Результаты исследования содержат специальные символы или графики: корректное отображение.

### Edge cases: лабораторный анализ

1. Анализ в статусе "Ожидание сдачи": результат не показан, индикатор серый или пропущен.
2. Анализ в статусе "Обработка": прогресс-бар вместо индикатора результата.
3. История анализа пуста (первый в истории): вкладка История отключена или показывает сообщение.
4. Результат содержит очень много показателей (100+): таблица результатов становится scrollable.
5. Нормативный диапазон имеет специальный формат ("< 5.0" или "> 100"): корректное отображение.
6. График тренда содержит большой диапазон времени (5+ лет): нужно ограничить или дать опцию масштабирования.
7. На графике очень много точек: агрегация или пропуск некоторых точек для оптимизации.

### Edge cases: поиск и фильтрация

1. Поиск пустой: отображаются все записи согласно фильтрам.
2. Поиск не находит совпадений: сообщение "Результаты не найдены".
3. При вводе поиска одновременно множество совпадений: таблица фильтруется в реальном времени.
4. Выбраны несовместимые фильтры: предупреждение или пустой результат.
5. Фильтры из одной вкладки не применяются к другой (правильное поведение).
6. Drawer фильтра не открывается: попытка открыть повторяется.
7. После применения фильтра таблица пуста: сообщение "Нет результатов по фильтрам".
8. После сброса фильтра исходные данные восстанавливаются корректно.

### Edge cases: модальные окна

1. Модаль открывается, но данные еще загружаются: skeleton внутри модали.
2. Запись удалена другим пользователем: сообщение об ошибке и закрытие модали.
3. При открытии модали произошла ошибка сети: сообщение об ошибке и кнопка Повторить.
4. Очень длинный текст в модали: становится scrollable внутри модали.
5. Модаль открыта на узком экране: адаптивное масштабирование и переупорядочение элементов.
6. История действий в модали очень длинная: становится scrollable.

### Edge cases: управление статусами

1. Кнопка Изменить статус недоступна из-за прав: кнопка скрывается или disabled.
2. Попытка изменить на недопустимый статус: сообщение об ошибке с доступными переходами.
3. За время открытия модали статус изменился другим пользователем: синхронизация или сообщение.
4. При изменении статуса произошла ошибка: сообщение об ошибке и кнопка Повторить.
5. История статусов содержит множество записей с похожими временами: корректная сортировка.

## Interaction & Animation

Правила поведения:

1. Переключение Tab-меню без полной перезагрузки страницы (lazy-load контента).
2. Поиск применяется без перезагрузки, с debounce 300ms.
3. Результаты поиска обновляются автоматически при вводе.
4. Фильтры открываются в Drawer и применяются после нажатия Применить (не в реальном времени).
5. Сортировка таблицы по клику на заголовок колонки (циклически: asc → desc → off).
6. Выделение строки при наведении (hover) для лучшей интерактивности.
7. Клик на строку открывает модаль с информацией.
8. После успешного обновления статуса таблица обновляется и показывается toast.
9. Переключение вида (таблица/карточки) сохраняется в localStorage.
10. Каждая вкладка имеет собственное состояние поиска, фильтров и вида.
11. Скачивание файла сопровождается прогресс-баром.
12. Наведение на точку графика тренда показывает tooltip.

Переходы и анимации:

1. Переключение Tab-меню: cross-fade контента (duration: 200ms).
2. Drawer фильтра: slide-in справа (duration: 300ms), backdrop затемнение.
3. Модаль деталей: fade-in с масштабированием (duration: 250ms).
4. Закрытие модали: fade-out (duration: 200ms).
5. Выделение строки при hover: плавное затемнение фона (transition: 200ms).
6. Загрузка таблицы: skeleton-лоадеры с pulse-анимацией.
7. Toast-сообщение: slide-up анимация, автоскрытие через 3 сек.
8. Переключение вида (таблица/карточки): cross-fade.
9. Прогресс-бар при скачивании: linear animation.
10. График тренда: плавное появление линий и точек.

## Зависимости и ограничения

### API

1. API загрузки всех визитов пациента с пагинацией.
2. API загрузки только консультаций врачей.
3. API загрузки только инструментальных исследований.
4. API загрузки только лабораторных анализов.
5. API фильтрации и поиска (с параметрами, зависящими от вкладки).
6. API получения полной информации о одной записи.
7. API скачивания файлов (для консультаций и исследований).
8. API изменения статуса записи.
9. API получения истории изменений статусов.
10. API получения графика тренда (для лабораторных анализов).
11. API получения доступных переходов статусов.

### Роли и права

1. Регистратор: просмотр всех данных, поиск, фильтрация, скачивание файлов, создание записей.
2. Врач-специалист: просмотр своих записей и исследований, редактирование некоторых полей.
3. Администратор: полный доступ ко всем функциям и редактированию.
4. Ограничения на изменение статусов в зависимости от роли.
5. Доступ к определенным вкладкам может быть ограничен по ролям.

### Feature flags

1. Включение страницы Визиты в меню модуля.
2. Включение вкладки Все визиты.
3. Включение вкладки Консультация врача.
4. Включение вкладки Инструментальные исследования.
5. Включение вкладки Лабораторные исследования.
6. Включение функции Поиск с debounce.
7. Включение Drawer фильтрации.
8. Включение режима переключения вида (таблица/карточки).
9. Включение скачивания файлов.
10. Включение тренд-анализа для лабораторных анализов.
11. Включение изменения статусов.
12. Включение виртуализации таблицы при большом количестве данных.

### Технические ограничения

1. При более чем 1000 записей требуется виртуализация таблицы.
2. Максимальный размер файла для скачивания: 500MB.
3. Таймаут запроса API: 30 секунд (с retry).
4. Кэширование данных на стороне клиента (TTL: 5 минут).
5. Пагинация по умолчанию: 50 элементов на странице.
6. WebSocket для real-time обновления статусов (опционально).
7. Максимальный период для графика тренда: 5 лет.
8. График тренда отображает максимум 100 точек (с агрегацией).
