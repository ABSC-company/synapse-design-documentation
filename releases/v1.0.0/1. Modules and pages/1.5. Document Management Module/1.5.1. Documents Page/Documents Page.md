# 1.5.1 Documents Page

## Спецификация страницы интерфейса

* Название страницы: Документы
* Модуль: EDM (Electronic Document Management)
* Тип страницы: Реестр документов
* Основные роли: Администратор, Руководитель, Сотрудник
* Статус документа: Draft
* Платформа: веб-приложение медицинской системы

---

## Назначение страницы и сценарии использования

Страница предназначена для просмотра и контроля реестра документов внутри клиники. Документы организованы в папки внутри вкладок по принадлежности. Просмотр документа, его подписание/отказ, история подписания и комментарии — на отдельной странице (→ 1.5.2 Document Page).

С помощью страницы пользователь может:

1. Просматривать входящие документы, ожидающие его действий (вкладка «Входящие»).
2. Перемещаться по папкам документов в разрезе принадлежности (HR, Медицинские, Прочие).
3. Создавать новые папки.
4. Создавать новые документы: через редактор (→ 1.5.4) или на основе шаблона (→ 1.5.3).
5. Открывать краткую карточку документа в модальном окне.
6. Переходить к полному просмотру документа (→ 1.5.2).
7. Отслеживать статус документооборота и прогресс подписания.
8. Выполнять действия с папками и документами через контекстное меню.
9. Выбирать несколько элементов и выполнять массовое удаление через Action Bar.
10. Переключаться между режимами отображения: карточки и таблица.

## User Flows / Task Flows

### Flow 1. Навигация по папкам

1. Пользователь открывает раздел «Документы» — по умолчанию активна вкладка «Входящие».
2. Пользователь переключается на вкладку принадлежности (например, «HR»).
3. Система отображает корневые папки вкладки.
4. Пользователь нажимает на папку — система переходит внутрь папки и отображает её содержимое (вложенные документы).
5. В заголовке обновляется путь через компонент Breadcrumbs.
6. Пользователь может вернуться на любой уровень, нажав на элемент в Breadcrumbs.

---

### Flow 2. Создание папки

1. Пользователь находится внутри уровня папок (вкладки принадлежности).
2. Нажимает «Новая папка».
3. Открывается модальное окно с полем названия папки.
4. Пользователь вводит название и подтверждает.
5. Папка создаётся в текущем расположении и отображается в списке.

---

### Flow 3. Создание документа

1. Пользователь нажимает кнопку «Создать документ».
2. Открывается модальное окно с выбором способа создания:
   * **Новый документ** — переход в редактор документов (→ 1.5.4). Документ создаётся с нуля.
   * **На основе шаблона** — отображается список доступных шаблонов. Пользователь выбирает шаблон и переходит на страницу заполнения (→ 1.5.3 Document Filling Page).
3. Созданный документ помещается в текущую папку со статусом «Черновик».

---

### Flow 4. Открытие карточки документа

1. Пользователь нажимает на элемент документа в списке.
2. Открывается модальное окно с подробной информацией.
3. Пользователь знакомится с деталями: метаданными, подписантами, прогрессом, просмотрами.
4. При необходимости нажимает «Открыть документ» — переход на страницу документа (→ 1.5.2).

---

### Flow 5. Действия через контекстное меню

1. Пользователь наводит курсор на элемент (папку или документ) — появляется иконка ⋮.
2. Пользователь нажимает ⋮ — открывается контекстное меню.
3. Пользователь выбирает действие. При удалении — запрашивается подтверждение.

---

### Flow 6. Множественная выборка и массовое удаление

1. Пользователь наводит курсор на элемент — появляется чекбокс.
2. Пользователь отмечает один или несколько элементов.
3. Снизу страницы появляется Action Bar с количеством выбранных элементов и доступными массовыми действиями.
4. Пользователь нажимает «Удалить» в Action Bar.
5. Открывается подтверждение: «Удалить N элементов?».
6. После подтверждения элементы удаляются, Action Bar скрывается.

### Header

Header меняется в зависимости от текущего уровня навигации.

**На корневом уровне вкладки:**
```
Документы                               [Новая папка]  [Создать документ]
[===] [---]  🔍 Поиск...  [= Фильтры]
```

**Внутри папки (с Breadcrumbs):**
```
Документы  >  HR  >  Приказы  >  2026   [Новая папка]  [Создать документ]
[===] [---]  🔍 Поиск...  [= Фильтры]
```

Элементы Header:

* **Breadcrumbs** — путь от корня до текущей папки. Каждый элемент кликабелен и переводит на соответствующий уровень. Корень («Документы») всегда является первым элементом.
* **Кнопка «Новая папка»** — открывает модальное окно создания папки. Скрыта на вкладке «Входящие».
* **Кнопка «Создать документ»** — открывает модальное окно выбора способа создания.
* **Переключатель режима отображения** — две кнопки: `[===]` карточки (grid) и `[---]` таблица (list). Активный режим подсвечен.
* **Поиск** — поле поиска по названию в текущей папке.
* **Кнопка «Фильтры»** — открывает drawer с расширенными фильтрами.

---

### Вкладки (Tab — первый уровень)

| Вкладка     | Описание                                                                                      |
| ----------- | --------------------------------------------------------------------------------------------- |
| Входящие    | Документы, ожидающие действий текущего пользователя (подписание или ознакомление). После выполнения действия документ покидает вкладку. Папочная структура не используется. |
| HR          | Папки и документы с принадлежностью «HR».                                                     |
| Медицинские | Папки и документы с принадлежностью «Медицинский».                                            |
| Прочие      | Папки и документы с другой принадлежностью.                                                   |

* Счётчик документов, ожидающих действий, отображается рядом с названием вкладки «Входящие»: `Входящие (12)`.

### Режим карточек (Grid)

Папки и документы отображаются в виде карточек в сетке.

**Карточка папки:**
```
┌──────────────────────────────────┐
│ [ ]  ← чекбокс при hover/выборе │
│                                  │
│             📁                   │
│                                  │
│  Название папки                  │
│  12 документов  📅 12.06.2026    │
└──────────────────────────────────┘
                                    ← ⋮ появляется при hover
```

**Карточка документа:**
```
┌──────────────────────────────────┐
│ [ ]  ← чекбокс при hover/выборе │
│                                  │
│         [превью документа]       │  ← thumbnail первой страницы
│                                  │
│  Название документа              │
│  Иванов И.И.                     │  ← автор
│  📅 12.06.2026  👁 47            │  ← дата создания / просмотры
└──────────────────────────────────┘
                                    ← ⋮ появляется при hover
```

* Чекбокс появляется в левом верхнем углу при hover или при активном режиме выборки.
* При выборе элемента карточка визуально выделяется (обводка акцентным цветом).
* Кнопка ⋮ — контекстное меню элемента.
* Клик на карточку документа (вне чекбокса и ⋮) — открывает модальное окно с информацией.
* Клик на карточку папки (вне чекбокса и ⋮) — переход внутрь папки.

---

### Режим таблицы (List)

Папки и документы отображаются строками таблицы.

**Шапка таблицы:**

| Чекбокс | Иконка | Название | Автор | Дата создания | Изменён | Просмотры | Статус | |
| ------- | ------ | -------- | ----- | ------------- | ------- | --------- | ------ |-|

**Строка папки:**
```
[ ]  📁  Название папки      —           12.06.2026   12.06.2026   —     —         ⋮
```

**Строка документа:**
```
[ ]  📄  Название документа  Иванов И.И.  12.06.2026   12.06.2026   47    Processed ⋮
```

* Чекбокс в первой колонке — для выборки строки.
* Чекбокс в шапке — выбрать/снять все элементы текущего уровня.
* При выборе строка подсвечивается.
* Клик на строку документа (вне чекбокса и ⋮) — открывает модальное окно с информацией.
* Клик на строку папки (вне чекбокса и ⋮) — переход внутрь папки.
* Колонки «Автор», «Статус» и «Просмотры» не применимы к папкам и отображаются как `—`.

---

### Action Bar (массовые действия)

Появляется снизу страницы при выборе одного или нескольких элементов. Скрывается при снятии всего выбора.

```
┌──────────────────────────────────────────────────────────────┐
│  Выбрано: 5 элементов         [🗑️ Удалить]  [✕ Снять выбор] │
└──────────────────────────────────────────────────────────────┘
```

Элементы:

* **Счётчик** — «Выбрано: N элементов».
* **Кнопка «Удалить»** — удаляет все выбранные элементы после подтверждения. Доступна только если пользователь имеет право удаления для всех выбранных элементов; иначе — элементы без права удаляются пропускаются с предупреждением.
* **Кнопка «Снять выбор»** — снимает выборку и скрывает Action Bar.

### Контекстное меню папки

```
┌───────────────────────────┐
│  ✏️  Переименовать        │
│  🗑️  Удалить              │
└───────────────────────────┘
```

| Действие        | Условие доступности                                   |
| --------------- | ----------------------------------------------------- |
| Переименовать   | Пользователь с правом редактирования папки            |
| Удалить         | Пользователь с правом удаления; папка должна быть пустой либо удаление каскадное |

---

### Контекстное меню документа

```
┌─────────────────────────┐
│  ✏️  Редактировать      │
│  📁  Переместить        │
│  🖨️  Распечатать        │
│  ⬇️  Скачать            │
│  ─────────────────────  │
│  🗑️  Удалить            │
└─────────────────────────┘
```

| Действие      | Условие доступности                                              |
| ------------- | ---------------------------------------------------------------- |
| Редактировать | Только для автора; статус Draft                                  |
| Переместить   | Автор или пользователь с правом на редактирование документа      |
| Распечатать   | Любой пользователь с доступом на просмотр                        |
| Скачать       | Любой пользователь с доступом на просмотр                        |
| Удалить       | Автор или пользователь с правом на удаление; статус Draft        |

При выборе «Переместить» открывается модальное окно с деревом папок. Пользователь выбирает целевую папку и подтверждает перемещение.

При выборе «Удалить» — запрашивается подтверждение в модальном окне.

## Статусы документа

### Draft

Черновик.

Особенности:

* Документ создан.
* Доступен только автору.
* Разрешено редактирование.

---

### Pending

Ожидает первую подпись.

Особенности:

* Документ отправлен на согласование.
* Виден автору и текущему подписанту.

---

### Processed

В процессе подписания.

Особенности:

* Получена минимум одна подпись.
* Отображается прогресс подписания.

Пример:

2 из 5 подписей.

---

### Signed

Подписан.

Особенности:

* Все подписи получены.
* Документооборот завершён.

---

### Delivered

Отправлен на ознакомление.

Особенности:

* Подписание не требуется.
* Документ доступен получателям для просмотра.

---

### Rejected

Отказан.

Особенности:

* Один из подписантов отказал в подписании.
* Отображается причина отказа.
* Документ возвращён автору.

---

### Aborted

Отозван.

Особенности:

* Автор отменил процесс подписания.
* Документооборот остановлен.

## Визуал: High-Fidelity Mockups

Визуальные требования:

1. Папки и документы визуально различимы иконками и структурой карточки.
2. Активный режим отображения (карточки / таблица) выделен в переключателе.
3. Breadcrumbs отображаются сразу под названием вкладки при навигации внутри папок.
4. Чекбокс выборки появляется при hover — ненавязчиво, но доступно.
5. Action Bar фиксируется снизу страницы поверх контента при активной выборке.
6. Статус документа отображается метками-значками в карточке и в модальном окне.
7. Прогресс подписания отображается визуально (например, `1 из 3`).
8. Документы, ожидающие действий пользователя, собраны в отдельной вкладке «Входящие».
9. Счётчик ожидающих отображается в названии вкладки «Входящие».
10. Интерфейс поддерживает большое количество элементов (пагинация или бесконечная прокрутка).
11. Все действия соответствуют единой дизайн-системе продукта.

## Состояния и Edge Cases

### Empty

* Пустая папка — Empty State: иллюстрация + «Папка пуста» + кнопки «Новая папка» и «Создать документ».
* Вкладка «Входящие» без ожидающих документов: «Нет документов, ожидающих ваших действий».
* Нет результатов по фильтрам/поиску: «Ничего не найдено» + кнопка «Сбросить фильтры».

### Loading

* Загрузка списка: карточки-заглушки (skeleton) или строки-заглушки в зависимости от режима.
* Загрузка модального окна карточки.
* Выполнение удаления: spinner, Action Bar заблокирован.

### Error

* Ошибка загрузки списка — сообщение с кнопкой «Повторить».
* Ошибка удаления — всплывающее уведомление с текстом ошибки.
* Удаление непустой папки без каскадного права — предупреждение: «Папка содержит документы».
* Шаблоны недоступны — сообщение в модальном окне выбора шаблона.

### Permissions

* Элементы, к которым у пользователя нет доступа, не отображаются.
* Недоступные действия в контекстном меню скрыты.
* Массовое удаление: если часть выбранных элементов нельзя удалить — выводится предупреждение с перечнем.

### Locked

Для документов в статусе Signed, Aborted, Rejected пункт «Редактировать» в контекстном меню недоступен.

## Acceptance Criteria

* Вкладка «Входящие» содержит только документы, ожидающие действий текущего пользователя; после выполнения действия документ покидает вкладку.
* Вкладки принадлежности (HR, Медицинские, Прочие) при открытии показывают папки корневого уровня.
* Навигация по папкам работает: клик на папку переходит внутрь, Breadcrumbs обновляется.
* Клик на элемент Breadcrumbs переводит на соответствующий уровень.
* Создание папки через кнопку «Новая папка» создаёт папку в текущем расположении.
* При нажатии «Создать документ» открывается модальное окно с выбором: «Новый документ» или «По шаблону».
* «Новый документ» — переход в редактор (1.5.4).
* «По шаблону» — отображается список шаблонов с поиском; клик на шаблон переводит на 1.5.3.
* Переключатель режима отображения переключает между карточками и таблицей.
* В обоих режимах при hover появляется чекбокс выборки.
* При выборе одного или нескольких элементов появляется Action Bar с количеством и кнопкой «Удалить».
* Массовое удаление запрашивает подтверждение; после подтверждения элементы удаляются.
* «Снять выбор» в Action Bar снимает все отметки и скрывает Action Bar.
* Клик на карточку/строку документа открывает модальное окно с полной информацией.
* Модальное окно содержит: название, описание, автора, даты, статус, принадлежность, тип доступа, просмотры, подписи, список просмотревших.
* Кнопка «Открыть документ» в модальном окне переводит на 1.5.2.
* Контекстное меню (⋮) появляется при hover и содержит корректный набор действий для папки или документа.
* Действия контекстного меню доступны только при наличии прав.
* Кнопка «Фильтры» открывает drawer с фильтрами; фильтры применяются к текущему уровню.
* Поиск фильтрует элементы по названию в режиме реального времени.
* Все статусы документов отображаются корректно.
* Empty, Loading и Error состояния обрабатываются и отображаются корректно.
