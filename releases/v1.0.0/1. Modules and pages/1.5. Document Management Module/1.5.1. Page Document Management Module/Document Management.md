# 1.5.1 Documents Page

## Спецификация страницы интерфейса

* Название страницы: Документы
* Модуль: EDM (Electronic Document Management)
* Тип страницы: Реестр документов
* Основные роли: Администратор, Руководитель, Сотрудник
* Статус документа: Draft
* Платформа: веб-приложение медицинской системы

---

## Назначение страницы и сценарии использования

Страница предназначена для просмотра, поиска и управления документами внутри системы.

С помощью страницы пользователь может:

1. Просматривать список документов.
2. Искать документы по названию.
3. Фильтровать документы.
4. Просматривать информацию о документе.
5. Открывать документ для просмотра.
6. Скачивать документ.
7. Распечатывать документ.
8. Подписывать документ.
9. Отказывать в подписании документа.
10. Оставлять комментарии к документу.

---

## User Flows / Task Flows

### Flow 1. Просмотр списка документов

1. Пользователь открывает раздел «Документы».
2. Система отображает список документов.
3. Пользователь может использовать вкладки для фильтрации списка.
4. Пользователь может воспользоваться поиском.
5. Пользователь может открыть карточку документа.

---

### Flow 2. Фильтрация документов

1. Пользователь нажимает кнопку «Фильтр».
2. Открывается Driver-компонент.
3. Пользователь выбирает параметры фильтрации.
4. Система обновляет список документов.

---

### Flow 3. Просмотр информации о документе

1. Пользователь нажимает на карточку документа.
2. Открывается модальное окно.
3. Система отображает информацию о документе.
4. Пользователь может перейти к просмотру документа.

---

### Flow 4. Просмотр документа

1. Пользователь нажимает кнопку «Посмотреть документ».
2. Открывается страница просмотра документа.
3. Пользователь просматривает содержимое документа.
4. Пользователь может оставить комментарий.
5. Пользователь может подписать или отклонить документ.

---

### Flow 5. Отказ от подписания документа

1. Пользователь нажимает кнопку «Отказать».
2. Открывается модальное окно.
3. Пользователь вводит комментарий.
4. Пользователь нажимает кнопку «Отправить комментарий».
5. Система сохраняет комментарий и выполняет отказ от подписания.

---

## Структура: Wireframes

### Header Page

Элементы:

* Заголовок страницы «Документы».
* Кнопка «Создать документ».

Расположение:

* Заголовок расположен слева.
* Кнопка создания документа расположена справа.

---

## Основной Layout

### Панель управления документами

Элементы:

* Tabs:

  * Все документы
  * Подписанные
  * Не подписанные
* Поле поиска по названию документа.
* Кнопка фильтрации.

Расположение:

* Tabs расположены слева.
* Поиск расположен рядом с Tabs.
* Кнопка фильтрации расположена справа.

---

### Driver фильтрации

При нажатии на кнопку фильтрации открывается Driver-компонент.

Содержимое:

* Фильтр по статусу.
* Фильтр по категории.
* Фильтр по дате.
* Фильтр по отделу.
* Фильтр по автору.

Для автора используется текстовое поле ввода.

---

## Основной контент

### Карточки документов

Для каждой карточки отображается:

* Название документа.
* Автор документа.

Пример автора:

* Google Docs

Дополнительно:

* Контекстное меню (три вертикальные точки).

---

### Контекстное меню документа

Элементы:

* Скачать документ.
* Распечатать документ.
* Открыть в новом окне.

---

## Модальное окно документа

При нажатии на карточку документа открывается модальное окно.

### Header модального окна

Элементы:

* Название документа.
* Кнопка закрытия модального окна.

Расположение:

* Название документа слева.
* Кнопка закрытия справа.

---

### Основная информация о документе

Информация отображается в виде таблицы.

Поля:

| Поле                | Описание                      |
| ------------------- | ----------------------------- |
| Автор               | Автор документа               |
| Подписанты          | Список подписантов            |
| Прогресс подписания | Например: 0 из 3              |
| Дата создания       | Дата создания документа       |
| Дата изменения      | Последнее изменение документа |

Для прогресса подписания используется Dropdown-компонент с отображением списка подписантов.

---

### Footer модального окна

Элементы:

* Кнопка «Посмотреть документ».

---

# Страница просмотра документа

После нажатия кнопки «Посмотреть документ» открывается отдельная страница просмотра документа.

Редактирование документа недоступно.

---

## Header страницы просмотра документа

Элементы:

* Логотип системы.
* Кнопка «Подписать».
* Кнопка «Отказать».

Расположение:

* Логотип расположен слева.
* Кнопки действий расположены справа.

---

## Основной Layout страницы просмотра

Страница состоит из трёх областей.

### Левая панель

Навигация по документу.

Отображается:

* Список разделов документа.
* Список страниц документа.

---

### Центральная область

Отображается содержимое документа.

Пользователь может:

* Просматривать документ.
* Перемещаться между страницами документа.

Редактирование документа недоступно.

---

### Правая панель

Блок комментариев.

Отображается:

* Поле добавления комментария.
* Список комментариев.

Для каждого комментария отображается:

* Автор.
* Дата создания.
* Текст комментария.

---

## Модальное окно отказа от подписания

Открывается после нажатия кнопки «Отказать».

Элементы:

* Поле ввода комментария.
* Кнопка «Отправить комментарий».
* Кнопка «Отмена».

Требования:

* Комментарий обязателен.
* Кнопка отправки недоступна до заполнения комментария.

---

## Состояния и Edge Cases

### Empty

* Документы отсутствуют.
* Отображается Empty State.

---

### Loading

* Загрузка списка документов.
* Загрузка документа.
* Загрузка комментариев.

---

### Error

* Ошибка загрузки документов.
* Ошибка загрузки документа.
* Ошибка сохранения комментария.

---

### Permissions

* Пользователь не имеет доступа к документу.
* Пользователь не может подписать документ.

---
