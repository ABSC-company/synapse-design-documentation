# 1.5.1 Documents Page

## Спецификация страницы интерфейса

* **Название страницы:** Документы
* **Модуль:** EDM (Electronic Document Management)
* **Тип страницы:** Реестр документов
* **Основные роли:** Администратор, Руководитель, Сотрудник
* **Статус документа:** Draft
* **Платформа:** веб-приложение медицинской системы

---

# Назначение страницы и сценарии использования

Страница предназначена для просмотра, поиска и управления документами внутри системы, а также создания новых документов.

С помощью страницы пользователь может:

1. Просматривать список документов
2. Искать документы по названию
3. Фильтровать документы
4. Просматривать информацию о документе
5. Открывать документ для просмотра
6. Скачать документ
7. Распечатать документ
8. Подписывать документ
9. Отказывать в подписании документа
10. Оставлять комментарии к документу
11. Создавать новый документ (через Header action)

---

# User Flows / Task Flows

## Flow 1. Просмотр списка документов

1. Пользователь открывает раздел «Документы»
2. Система отображает список документов
3. Пользователь использует вкладки для фильтрации списка
4. Пользователь использует поиск
5. Пользователь открывает карточку документа

---

## Flow 2. Фильтрация документов

1. Пользователь нажимает кнопку «Фильтр»
2. Открывается Driver-компонент
3. Пользователь выбирает параметры
4. Система обновляет список документов

---

## Flow 3. Просмотр информации о документе

1. Пользователь нажимает на карточку документа
2. Открывается модальное окно
3. Система отображает информацию
4. Пользователь переходит к просмотру документа

---

## Flow 4. Просмотр документа

1. Пользователь открывает документ
2. Открывается Document View Page
3. Пользователь просматривает содержимое
4. Пользователь оставляет комментарий
5. Пользователь подписывает или отклоняет документ

---

## Flow 5. Отказ от подписания документа

1. Пользователь нажимает «Отказать»
2. Открывается модальное окно
3. Пользователь вводит комментарий
4. Нажимает «Отправить»
5. Система сохраняет комментарий и фиксирует отказ

---

## Flow 6. Создание документа (новый сценарий)

### 6.1 Открытие создания документа

1. Пользователь нажимает кнопку **«Создать документ»** в Header
2. Открывается модальное окно выбора режима создания

---

### 6.2 Выбор режима создания

В модальном окне доступны два варианта:

* **Создать из шаблона (Template Document)**
* **Создать с нуля (Blank Document)**

---

### 6.3 Создание документа из шаблона

1. Пользователь выбирает «Шаблон документа»
2. Система открывает список доступных шаблонов
3. Пользователь выбирает шаблон
4. Система перенаправляет на страницу:

### Template Document Builder (страница конструктора шаблонного документа)

#### Layout страницы:

* Левая панель: форма заполнения
* Правая панель: live preview (финальный результат документа)

#### Поведение:

* При вводе данных слева, справа обновляется итоговый документ в реальном времени
* Пользователь может сохранить документ или продолжить редактирование

---

### 6.4 Создание документа с нуля

1. Пользователь выбирает «Создать с нуля»
2. Система перенаправляет на:

### Document Editor (страница редактирования документа)

#### Особенности:

* Полностью пустой документ
* Пользователь самостоятельно формирует структуру
* Поддержка базовых блоков документа
* Возможность сохранения Draft

---

## Расположение

* Заголовок слева
* Кнопка справа

---

# Модальное окно создания документа

## Элементы

* Выбор режима:

  * Template Document
  * Blank Document
* Кнопка закрытия

## Поведение

* Без выбора режима кнопка продолжения недоступна
* После выбора выполняется редирект на соответствующую страницу

---

## Layout

### Левая панель

## Элементы

* Поле комментария (обязательное)

## Правила

---

# Состояния

## Empty

* Нет документов

## Loading

* Загрузка списка / документа / комментариев

## Error

* Ошибка загрузки / сохранения

## Permissions

* Нет доступа к документу
* Нет прав подписи

---

# Acceptance Criteria

* Пользователь может просматривать документы
* Может фильтровать через Tabs и Driver
* Может искать документы
* Может открывать карточки
* Может просматривать документ
* Может скачивать / печатать / открывать в новом окне
* Может создавать документ через Header
* Может выбрать шаблон или создать с нуля
* Может редактировать шаблон через Builder
* Может работать с Document Editor
* Может подписывать / отклонять документ
* Комментарий обязателен при отказе
* Все состояния отображаются корректно
