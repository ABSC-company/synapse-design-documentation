# 1.5.2 Document Page

## Спецификация страницы интерфейса

* Название страницы: Просмотр документа
* Модуль: EDM (Electronic Document Management)
* Тип страницы: Детальная страница документа
* Основные роли: Администратор, Руководитель, Сотрудник, Подписант
* Статус документа: Draft
* Платформа: веб-приложение медицинской системы

---

## Назначение страницы и сценарии использования

Страница открывается при переходе по клику на документ из реестра (← 1.5.1 Documents Page). Предназначена для детального взаимодействия с конкретным документом.

С помощью страницы пользователь может:

1. Просматривать содержимое документа.
2. Подписывать документ.
3. Отказывать в подписании с указанием причины.
4. Просматривать историю подписания.
5. Читать комментарии к документу.
6. Добавлять комментарии к документу.
7. Скачивать и распечатывать документ.

---

## User Flows / Task Flows

### Flow 1. Просмотр документа

1. Пользователь переходит на страницу из реестра документов.
2. Система загружает содержимое документа и его метаданные.
3. Пользователь читает содержимое, листает страницы (если документ многостраничный).
4. Пользователь может просмотреть очередь подписантов, историю подписания, комментарии.

---

### Flow 2. Подписание документа

1. Подписант открывает документ, ожидающий его подписи.
2. Система отображает блок действий с кнопками «Подписать» и «Отказать».
3. Подписант знакомится с содержимым.
4. Нажимает «Подписать».
5. Система сохраняет подпись с датой и временем.
6. Документ переходит следующему подписанту (последовательный режим) либо получает статус «Подписан» (если подписи завершены).
7. Блок действий скрывается, статус подписи текущего пользователя обновляется до «Подписано».

---

### Flow 3. Отказ в подписании

1. Подписант открывает документ, ожидающий его подписи.
2. Нажимает «Отказать».
3. Появляется форма с обязательным полем комментария.
4. Подписант вводит причину отказа и подтверждает действие.
5. Статус документа меняется на «Отказан».
6. Автор документа получает уведомление с причиной отказа.
7. Документ возвращается автору, дальнейшее подписание недоступно.

---

### Flow 4. Добавление комментария

1. Пользователь прокручивает страницу до секции комментариев (или переходит через навигацию).
2. Нажимает «Добавить комментарий».
3. Вводит текст в поле ввода.
4. Нажимает «Отправить».
5. Комментарий появляется в списке с именем пользователя и временной меткой.

## Структура: Wireframes

### Общий layout

```
┌────────────────────────────────────────────────────────────────────┐
│  HEADER                                                            │
│  [← Документы]  Название документа  [Скачать] [Печать] [...]      │
├───────────────────────────────────────┬────────────────────────────┤
│                                       │  ПРАВАЯ ПАНЕЛЬ             │
│  ОБЛАСТЬ ПРОСМОТРА ДОКУМЕНТА                │  ┌──────────────────────┐  │
│  (рендер HTML-страниц)                │  │  Метаданные          │  │
│                                       │  ├──────────────────────┤  │
│  ┌───────────────────────────────┐    │  │  Блок действий       │  │
│  │  [содержимое страницы 1]      │    │  │  (Подписать/Отказать)│  │
│  └───────────────────────────────┘    │  ├──────────────────────┤  │
│                                       │  │  Очередь подписантов │  │
│  [Страница 1 из N]  [← →]            │  ├──────────────────────┤  │
│                                       │  │  История подписания  │  │
│                                       │  └──────────────────────┘  │
├───────────────────────────────────────┴────────────────────────────┤
│  СЕКЦИЯ КОММЕНТАРИЕВ                                               │
│  [список комментариев]                                             │
│  [поле добавления комментария]  [Отправить]                        │
└────────────────────────────────────────────────────────────────────┘
```

---

### Header

Элементы:

* Кнопка «← Документы» — возврат в реестр.
* Название документа.
* Кнопка «Скачать» — скачать PDF/исходный формат.
* Кнопка «Печать».
* Меню «...» — дополнительные действия (по правам): редактировать, отозвать, удалить.

---

### Область просмотра документа

Основная область страницы. Отображает содержимое документа в HTML-формате (отображается из шаблона с подставленными значениями).

Элементы:

* Белый «лист» документа с тенью (имитация страницы A4), центрированный на фоне нейтрального серого.
* Все переменные, поля ввода и подписи подставлены — пользователь видит итоговый документ, а не шаблон.
* Навигация по страницам (если документ многостраничный):
  * Текущий номер и общее количество страниц: `Страница 1 из N`.
  * Кнопки «←» и «→» (или листание страниц).

---

### Правая панель

Фиксированная панель справа от области просмотра. Состоит из четырёх блоков.

#### Блок «Метаданные документа»

| Поле               | Описание                            |
| ------------------ | ----------------------------------- |
| Автор              | Создатель документа                 |
| Категория          | Категория документа                 |
| Статус             | Текущий статус (бейдж)              |
| Дата создания      | Дата создания                       |
| Дата обновления    | Дата последнего изменения           |
| Тип доступа        | Общий / Ограниченный                |
| Количество просмотров | Счётчик просмотров               |

---

#### Блок «Действия» (только для текущего подписанта)

Отображается только если документ ожидает подписи текущего пользователя.

```
┌──────────────────────────────┐
│  Ожидает вашей подписи       │
│                              │
│  [Подписать]                 │
│  [Отказать]                  │
└──────────────────────────────┘
```

* «Подписать» — основная кнопка (акцентный цвет).
* «Отказать» — второстепенная кнопка (деструктивный цвет или outlined).
* Если пользователь уже подписал — блок заменяется статусом «Вы подписали документ» с датой.
* Если пользователь отказал — блок заменяется статусом «Вы отказали в подписании» с датой и причиной.
* Если пользователь не является подписантом — блок не отображается.

---

#### Блок «Очередь подписантов»

Список всех подписантов шаблона в порядке очереди.

```
┌──────────────────────────────────────────┐
│  Подписанты                              │
├──────────────────────────────────────────┤
│  ✅  1. Иванов И.И.  Хирург              │  ← Подписано
│       12.06.2026, 14:32                  │
├──────────────────────────────────────────┤
│  ●   2. Петрова А.С.  Главный врач       │  ← Текущий (выделен)
│       Ожидает подписи                    │
├──────────────────────────────────────────┤
│  ○   3. Сидоров К.Р.  Администратор      │  ← Ожидает
│       —                                  │
└──────────────────────────────────────────┘
```

Для каждого подписанта:

* Порядковый номер.
* ФИО и должность.
* Статус подписания с иконкой:

| Статус              | Иконка | Описание                        |
| ------------------- | ------ | ------------------------------- |
| Ожидает             | ○      | Ещё не дошла очередь            |
| Текущий подписант   | ●      | Визуально выделен (акцентный)   |
| Подписано           | ✅     | Дата подписания                 |
| Отказано            | ✕      | Красный цвет, причина отказа    |

---

#### Блок «История подписания»

Хронологический лог событий документооборота.

```
┌──────────────────────────────────────────┐
│  История                                 │
├──────────────────────────────────────────┤
│  12.06.2026 14:32                        │
│  Иванов И.И. подписал документ.         │
├──────────────────────────────────────────┤
│  12.06.2026 10:00                        │
│  Документ отправлен на подписание.       │
├──────────────────────────────────────────┤
│  11.06.2026 18:45                        │
│  Документ создан. Автор: Козлова Т.В.   │
└──────────────────────────────────────────┘
```

Каждая запись: дата/время + описание события + участник.

---

### Секция комментариев

Расположена под областью просмотра документа (полная ширина).

```
┌────────────────────────────────────────────────────────────┐
│  Комментарии  (3)                                          │
├────────────────────────────────────────────────────────────┤
│  Козлова Т.В.   12.06.2026, 09:15                          │
│  Необходимо уточнить дозировку в разделе 3.               │
├────────────────────────────────────────────────────────────┤
│  Иванов И.И.   12.06.2026, 11:40                          │
│  Исправлено, пересмотрите пожалуйста.                     │
├────────────────────────────────────────────────────────────┤
│  ┌──────────────────────────────────────────────────────┐  │
│  │  Добавьте комментарий...                             │  │
│  └──────────────────────────────────────────────────────┘  │
│                                         [Отправить]        │
└────────────────────────────────────────────────────────────┘
```

* Комментарии отображаются в хронологическом порядке (от старых к новым).
* Каждый комментарий: имя, дата/время, текст.
* Поле ввода — многострочный textarea.
* Кнопка «Отправить» недоступна, пока поле пустое.
* Если комментариев нет — Empty State с текстом «Комментариев пока нет».
* Доступность комментирования определяется правами пользователя на документ.

## Форма отказа в подписании

При нажатии «Отказать» поверх страницы открывается модальное окно.

```
┌────────────────────────────────────────┐
│  Отказать в подписании             [✕] │
├────────────────────────────────────────┤
│                                        │
│  Укажите причину отказа *              │
│  ┌──────────────────────────────────┐  │
│  │                                  │  │
│  │  (многострочное поле)            │  │
│  │                                  │  │
│  └──────────────────────────────────┘  │
│                                        │
├────────────────────────────────────────┤
│       [Отмена]    [Отказать]           │
└────────────────────────────────────────┘
```

Требования:

* Поле причины обязательное.
* Кнопка «Отказать» в модальном окне недоступна, пока поле не заполнено.
* После подтверждения статус документа меняется на «Отказан».
* Причина отказа фиксируется в истории подписания и видна автору документа.
* Кнопка «Отмена» закрывает модальное окно без изменений.

## Доступность действий по роли и статусу

| Действие         | Кто видит                                           | При каком статусе             |
| ---------------- | --------------------------------------------------- | ----------------------------- |
| Подписать        | Текущий активный подписант                          | Pending, Processed            |
| Отказать         | Текущий активный подписант                          | Pending, Processed            |
| Скачать          | Все, у кого есть доступ на просмотр                 | Любой                         |
| Печать           | Все, у кого есть доступ на просмотр                 | Любой                         |
| Добавить комментарий | Все, у кого есть право на просмотр             | Любой (кроме Aborted)         |
| Редактировать    | Автор                                               | Draft                         |
| Отозвать         | Автор                                               | Pending, Processed            |
| Удалить          | Автор или пользователь с правом на удаление         | Draft                         |

## Состояния и Edge Cases

### Loading

* Загрузка содержимого документа: скелетон-заглушка в области области просмотра и правой панели.
* Выполнение подписания/отказа: кнопки переходят в состояние loading, повторный клик заблокирован.

### Error

* Документ не найден — страница с сообщением об ошибке и кнопкой «← Назад к документам».
* Ошибка загрузки содержимого — сообщение с кнопкой «Повторить».
* Ошибка подписания/отказа — toast с текстом ошибки, состояние кнопок восстанавливается.

### Permissions

* Пользователь не имеет доступа к документу — страница 403 с кнопкой «← Назад к документам».
* Пользователь не является текущим подписантом — блок действий скрыт.
* Пользователь уже подписал — блок действий заменён статусом с датой.

### Locked (Signed / Aborted / Rejected)

* Блок действий не отображается.
* История подписания и комментарии доступны для просмотра.
* При статусе «Отказан» — причина отказа выделена в истории и в блоке подписантов.

## Визуал: High-Fidelity Mockups

Визуальные требования:

1. Документ отображается как белый лист на сером фоне — аналогично редактору (1.5.3).
2. Текущий подписант в очереди выделен акцентным цветом и/или иконкой.
3. Статус документа отображается меткой-значком в метаданных.
4. Причина отказа выделена красным цветом в истории и в блоке подписантов.
5. Кнопки «Подписать» и «Отказать» достаточно крупные и хорошо различимы по цвету.
6. Счётчик комментариев отображается рядом с заголовком секции.
7. Все переходы состояний (загрузка, успех, ошибка) плавные.
8. Все действия соответствуют единой дизайн-системе продукта.

## Acceptance Criteria

* Документ открывается и содержимое отображается корректно.
* Многостраничные документы поддерживают навигацию между страницами.
* Метаданные документа отображаются в правой панели.
* Блок действий («Подписать» / «Отказать») виден только текущему активному подписанту.
* После нажатия «Подписать» статус подписи обновляется, блок действий заменяется подтверждением.
* Форма отказа открывается модальным окном; кнопка «Отказать» недоступна без причины.
* После отказа документ получает статус «Отказан», причина видна автору.
* Очередь подписантов отображает корректные статусы для каждого участника.
* История подписания содержит все события в хронологическом порядке.
* Комментарии отображаются в хронологическом порядке.
* Добавление комментария работает: поле обязательное, после отправки комментарий появляется в списке.
* Доступ к действиям соответствует правам пользователя и статусу документа.
* Состояния Loading, Error и Permissions обрабатываются и отображаются корректно.
