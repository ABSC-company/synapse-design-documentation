# 5. Увольнение и Архив уволенных сотрудников

## Техническое задание для дизайнера

* **Раздел:** HR Module
* **Страницы/флоу:** Увольнение сотрудника (модальное окно из профиля), Архив уволенных (отдельная страница), Восстановление сотрудника
* **Основные роли:** Администратор / HR-менеджер
* **Статус документа:** Draft

> Общие паттерны — см. `00-overview.md`. Точка входа в увольнение — шапка профиля сотрудника, см. `01-employees-list-and-profile.md`, раздел 1.2.1.1.

---

## Контекст

При увольнении сотрудник переводится в статус `dismissed`, система собирает весь его дамп данных (должности, контакты, рабочие места, график, специализации, допуски, аттестации, документы, инструктажи, учёт времени), упаковывает в архив и **физически переносит запись из активного списка сотрудников в архив**. Поэтому в реестре сотрудников (`01-employees-list-and-profile.md`) уволенные не отображаются как один из фильтров статуса — для них отдельная страница.

Восстановление — обратный процесс: данные распаковываются, сотрудник возвращается в активный список со статусом `at_work`.

---

## User Flows

**Flow 1. Увольнение сотрудника**
1. Пользователь в профиле сотрудника открывает меню статуса или `⋮` → «Уволить».
2. Открывается модальное окно увольнения.
3. Пользователь подтверждает (опционально указывает причину/комментарий, опционально создаёт приказ в ЭДО).
4. Система переводит сотрудника в `dismissed`, архивирует данные, освобождает занятые ставки в штатном расписании.
5. Пользователь возвращается в реестр сотрудников; уволенный больше не отображается в активном списке.

**Flow 2. Просмотр архива**
1. Пользователь открывает «Архив уволенных» в боковом меню (или по ссылке «Уволенные →» из реестра сотрудников).
2. Видит список архивных записей с возможностью поиска и просмотра деталей.

**Flow 3. Восстановление сотрудника**
1. Пользователь в архиве открывает карточку уволенного сотрудника.
2. Нажимает «Восстановить».
3. Подтверждает действие.
4. Система распаковывает архив, возвращает сотрудника в активный список со статусом `at_work`, восстанавливает связанные записи (должности, контакты и т.д.) и увеличивает занятые ставки в штатном расписании.
5. Пользователь видит сотрудника снова в общем реестре.

## 5.1 Модальное окно «Увольнение сотрудника»

```
┌──────────────────────────────────────────────────┐
│  Увольнение сотрудника                        [✕]│
├──────────────────────────────────────────────────┤
│                                                   │
│  Вы увольняете:                                   │
│  🖼 Иванов Иван Иванович — Хирург, Хирургия        │
│                                                   │
│  ⚠ Это действие архивирует все данные сотрудника  │
│  (должности, документы, аттестации, историю       │
│  учёта времени) и освободит занятые им ставки      │
│  в штатном расписании. Сотрудника можно            │
│  восстановить позже из раздела «Архив».            │
│                                                   │
│  ┌──────────────────────────────────────────┐    │
│  │  Кадровый документ                       │    │
│  │  Увольнение сопровождается приказом.      │    │
│  │  [Создать документ в ЭДО →]               │    │
│  └──────────────────────────────────────────┘    │
│                                                   │
├──────────────────────────────────────────────────┤
│                   [Отмена]      [Уволить]         │
└──────────────────────────────────────────────────┘
```

- Кнопка «Уволить» — деструктивный/предупреждающий стиль (например, красная или с предупреждающей иконкой), так как действие не тривиально обратимо в моменте (требует отдельной процедуры восстановления).
- Блок перехода в ЭДО — по общему паттерну (`00-overview.md`), не блокирует завершение увольнения: пользователь может создать приказ сразу или позже из самого ЭДО.
- После подтверждения — короткое состояние загрузки внутри модального окна (архивирование может занять заметное время из-за сборки дампа данных), например, спиннер с текстом «Архивируем данные сотрудника...», прежде чем модальное окно закроется.

## 5.2 Страница «Архив уволенных сотрудников»

### Layout

```
┌─────────────────────────────────────────────────────────────────┐
│  Архив уволенных сотрудников                                     │
├─────────────────────────────────────────────────────────────────┤
│  🔍 Поиск по имени, фамилии...                                   │
├─────────────────────────────────────────────────────────────────┤
│  🖼  Иванов Иван Иванович                                         │
│      Был: Хирург, Хирургия · Уволен: 12.05.2026                  │
│                                                  [Подробнее →]    │
├─────────────────────────────────────────────────────────────────┤
│  🖼  Петрова Анна Сергеевна                                       │
│      Была: Медсестра, Терапия · Уволена: 03.02.2026               │
│                                                  [Подробнее →]    │
└─────────────────────────────────────────────────────────────────┘
```

- Строка показывает фото, ФИО, последнюю должность и отдел на момент увольнения, дату увольнения.
- Поиск — по имени/фамилии/отчеству.
- Нет права удалять записи архива физически из этого реестра (тех. задание не описывает такой use case) — только просмотр и восстановление.

### 5.2.1 Карточка архивной записи (детали)

Открывается по «Подробнее →».

```
┌───────────────────────────────────────────────────────────┐
│  ← Назад к архиву                                            │
├───────────────────────────────────────────────────────────┤
│  🖼   Иванов Иван Иванович                                    │
│       Был: Хирург, Хирургия · Филиал на Юнусобод               │
│       Уволен: 12.05.2026                      [Восстановить] │
├───────────────────────────────────────────────────────────┤
│  Личные данные                                                │
│  Дата рождения, гражданство, контакты — read-only снимок       │
│  на момент увольнения                                          │
├───────────────────────────────────────────────────────────┤
│  📦 Полный архив данных                                        │
│  Содержит: должности, контакты, рабочие места, график,          │
│  специализации, допуски, аттестации, документы,                │
│  инструктажи, учёт рабочего времени                            │
│  [Скачать архив]                                                │
└───────────────────────────────────────────────────────────┘
```

- Содержимое архива (`archive_id` из File Module) для просмотра доступно одним действием «Скачать архив» — детальный построчный просмотр содержимого архива не описан в тех. задании как отдельный UI-сценарий, поэтому здесь не детализируется: достаточно указать на наличие архива и кнопку получения файла (поведение самой выгрузки — на стороне File Module).
- Кнопка «Восстановить» — главное действие карточки, расположена в шапке.

## 5.3 Модальное окно «Восстановление сотрудника»

```
┌──────────────────────────────────────────────────┐
│  Восстановление сотрудника                    [✕]│
├──────────────────────────────────────────────────┤
│                                                   │
│  Вы восстанавливаете:                              │
│  🖼 Иванов Иван Иванович — Хирург, Хирургия        │
│                                                   │
│  Сотрудник вернётся в активный список со статусом │
│  «На рабочем месте». Будут восстановлены           │
│  должности, контакты, рабочие места, график и      │
│  прочие данные на момент увольнения. Ставки по      │
│  ранее занятым должностям будут зарезервированы     │
│  заново в штатном расписании.                       │
│                                                   │
│  ⚠ Если ставка по прежней должности уже занята      │
│  другим сотрудником, восстановление может            │
│  потребовать ручной корректировки штатного           │
│  расписания.                                         │
│                                                   │
├──────────────────────────────────────────────────┤
│                 [Отмена]      [Восстановить]      │
└──────────────────────────────────────────────────┘
```

- Предупреждение про занятую ставку — превентивная UX-подсказка для дизайнера; точная обработка конфликта (что показывает система, если ставки не хватает) не зафиксирована в тех. задании на уровне бизнес-логики и должна быть уточнена отдельно с продуктом/разработкой. На уровне дизайна достаточно предусмотреть, что сообщение об ошибке восстановления (если оно возникнет) выводится отдельным toast/баннером после неудачной попытки.
- После подтверждения — состояние загрузки аналогично увольнению («Восстанавливаем данные сотрудника...»).
- После успешного восстановления — пользователь перенаправляется в профиль восстановленного сотрудника в общем реестре (не остаётся в архиве).

## Состояния

| Состояние | Поведение |
|-----------|-----------|
| Empty (Архив) | «Уволенных сотрудников нет» |
| Loading | Скелетон списка архива / спиннер во время архивирования и восстановления |
| Error | Toast «Не удалось уволить/восстановить сотрудника. Попробуйте снова» |
| Permissions | Страница архива и действия увольнения/восстановления скрыты без `HR_ARCHIVE_GET` / `HR_ARCHIVE_MANAGE` соответственно |

## Acceptance Criteria

- Увольнение запускается из профиля сотрудника (меню статуса или контекстное меню), открывает модальное окно с явным предупреждением о последствиях.
- Кнопка подтверждения увольнения визуально маркирована как необратимое/предупреждающее действие.
- Модальное окно увольнения содержит необязательный переход в ЭДО для создания приказа, не блокирующий завершение действия.
- Уволенные сотрудники не отображаются в основном реестре сотрудников и его статусных фильтрах — для них отдельная страница «Архив уволенных».
- Страница архива поддерживает поиск по ФИО и показывает последнюю должность/отдел и дату увольнения для каждой записи.
- Карточка архивной записи показывает read-only снимок личных данных и предоставляет скачивание полного архива одной кнопкой.
- Восстановление сотрудника доступно из карточки архивной записи, сопровождается предупреждением о восстанавливаемых данных и потенциальном конфликте по ставкам.
- После восстановления пользователь перенаправляется в профиль сотрудника в общем (не архивном) реестре.
- Для всех экранов раздела предусмотрены состояния: пусто, загрузка, ошибка, нет прав.
