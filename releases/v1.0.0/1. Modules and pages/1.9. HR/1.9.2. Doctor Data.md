# P-03 — Staffing Schedule / Штатное расписание

## Спецификация страницы интерфейса

- **Название страницы:** Штатное расписание
- **Модуль:** HR (Human Resources)
- **Тип страницы:** Реестр
- **Основные роли:** Администратор, HR-менеджер, Руководитель
- **Права доступа:** `HR_STAFFING_GET`, `HR_STAFFING_MANAGE`
- **Платформа:** веб-приложение медицинской системы
- **Маршрут:** `/hr/staffing`

---

# Назначение страницы и сценарии использования

Страница предназначена для просмотра и управления штатным расписанием клиники — перечнем должностей, количеством ставок, окладами и уровнем заполненности.

С помощью страницы пользователь может:

1. Просматривать список должностей в штатном расписании
2. Видеть количество ставок и уровень заполненности по каждой должности
3. Фильтровать расписание по отделу, наличию вакансий
4. Создавать новую запись штатного расписания
5. Редактировать существующую запись
6. Удалять запись

---

# User Flows / Task Flows

## Flow 1. Просмотр штатного расписания

1. Пользователь открывает раздел «Штатное расписание»
2. Система отображает список записей
3. Пользователь фильтрует по отделу или наличию вакансий

---

## Flow 2. Создание записи

1. Пользователь нажимает «Добавить позицию»
2. Открывается модальное окно M-16
3. Пользователь заполняет форму
4. Нажимает «Сохранить»
5. Система создаёт запись и обновляет список

---

## Flow 3. Редактирование записи

1. Пользователь нажимает «⋯» на строке
2. Выбирает «Редактировать»
3. Открывается M-16 с заполненными данными
4. Пользователь вносит изменения и сохраняет

---

## Flow 4. Удаление записи

1. Пользователь нажимает «⋯» → «Удалить»
2. Система запрашивает подтверждение (inline confirm или отдельный модал)
3. Запись удаляется

# Основной контент

## Таблица штатного расписания

| Колонка | Описание |
|---------|----------|
| Должность | Название из Structure Module |
| Отдел | Из Structure Module |
| Всего ставок | `tariff_rates` |
| Занято ставок | `occupied_tariff_rates` |
| Вакантно | `tariff_rates - occupied_tariff_rates` |
| Заполненность | Прогресс-бар: `occupied / tariff_rates × 100%` |
| Оклад (1 ставка) | `salary` |
| Базовая роль | Название роли из Auth Module (если задана) |
| Действия | Кнопка «⋯» |

## Цветовая индикация заполненности

| Состояние | Цвет |
|-----------|------|
| Полностью занято | Зелёный |
| Частично занято | Жёлтый |
| Пусто / вакантно | Красный |

## Контекстное меню

- Редактировать (право `HR_STAFFING_MANAGE`)
- Удалить (право `HR_STAFFING_MANAGE`)

# Модальное окно — Создать / редактировать запись штатного расписания (M-16)

## Header

- Заголовок: «Новая позиция» / «Редактирование позиции»
- Кнопка закрытия

## Форма

| Поле | Тип | Обязательное |
|------|-----|:---:|
| Должность (`position_id`) | Select из Structure Module | ✓ |
| Количество ставок (`tariff_rates`) | Number (шаг 0.25) | ✓ |
| Оклад за 1 ставку (`salary`) | Number | ✓ |
| Базовая роль (`base_role_id`) | Select из Auth Module | — |

## Footer

- Кнопка «Сохранить»
- Кнопка «Отмена»

## Поведение

- `occupied_tariff_rates` не редактируется вручную — вычисляется автоматически при назначении / снятии должностей сотрудников
- При редактировании нельзя установить `tariff_rates` меньше текущего значения `occupied_tariff_rates`

# Состояния страницы

## Empty

- Текст: «Штатное расписание не заполнено»
- Кнопка «Добавить позицию»

## Loading

- Скелетон строк таблицы

## Error

- Баннер с ошибкой и кнопкой «Повторить»

## Permissions

- Нет права `HR_STAFFING_GET` — редирект на 403
- Нет права `HR_STAFFING_MANAGE` — кнопка «Добавить позицию» и контекстное меню скрыты

---

# Acceptance Criteria

- Таблица отображает все записи штатного расписания
- Прогресс-бар заполненности корректно отражает соотношение занятых и общих ставок
- Фильтр по вакансиям показывает только должности с незанятыми ставками
- M-16 не позволяет установить `tariff_rates` меньше `occupied_tariff_rates`
- После назначения / снятия должности у сотрудника `occupied_tariff_rates` обновляется автоматически
- Все состояния (empty, loading, error, permissions) отображаются корректно
