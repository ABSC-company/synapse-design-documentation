# 4. Штатное расписание клиники

## Техническое задание для дизайнера

* **Раздел:** HR Module
* **Страница:** Штатное расписание (отдельный пункт бокового меню)
* **Основные роли:** Администратор / HR-менеджер
* **Статус документа:** Draft

> Общие паттерны — см. `00-overview.md`.

---

## Назначение страницы

Штатное расписание клиники по должностям: сколько ставок предусмотрено на каждую должность, сколько уже занято, какой оклад за полную ставку, какая базовая роль выдаётся при назначении. Это справочно-управляющий экран для HR/администратора — не привязан к конкретному сотруднику, используется как основа при назначении должностей сотрудникам (вкладка «Должности» в профиле сотрудника читает остаток ставки отсюда — см. `01-employees-list-and-profile.md`, раздел 1.2.3).

## User Flows

**Flow 1. Просмотр штатного расписания**
1. Пользователь открывает «Штатное расписание» в боковом меню.
2. Система показывает список должностей с количеством ставок (всего/занято/свободно) и окладом.
3. Пользователь может отфильтровать по отделу/филиалу.

**Flow 2. Создание записи штатного расписания**
1. Пользователь нажимает «+ Добавить запись».
2. Выбирает должность, указывает количество ставок и оклад.
3. Опционально указывает базовую роль, выдаваемую при назначении.
4. Сохраняет — запись появляется в списке с `occupied_tariff_rates = 0`.

**Flow 3. Редактирование ставок**
1. Пользователь меняет общее количество ставок по должности (например, расширение штата).
2. Система не позволяет установить значение меньше, чем уже занято (`occupied_tariff_rates`) — показывает ошибку валидации.

## Layout

```
┌─────────────────────────────────────────────────────────────────┐
│  Штатное расписание                       [+ Добавить запись]   │
├─────────────────────────────────────────────────────────────────┤
│  🔍 Поиск по должности...                  [Фильтр ▾]           │
├─────────────────────────────────────────────────────────────────┤
│  Должность      │ Отдел      │ Ставки      │ Оклад/ставка │ ⋮  │
│  ────────────────────────────────────────────────────────────── │
│  Хирург         │ Хирургия   │ 1.5 / 2.0   │ 8 000 000 UZS│ ⋮  │
│                 │            │ ▓▓▓░ 75%    │              │    │
│  ────────────────────────────────────────────────────────────── │
│  Медсестра      │ Хирургия   │ 4.0 / 4.0   │ 4 000 000 UZS│ ⋮  │
│                 │            │ ▓▓▓▓ 100%   │              │    │
│  ────────────────────────────────────────────────────────────── │
│  Администратор  │ Ресепшн    │ 0.0 / 2.0   │ 3 500 000 UZS│ ⋮  │
│                 │            │ ░░░░ 0%     │              │    │
└─────────────────────────────────────────────────────────────────┘
```

### Элементы строки

- **Должность / Отдел** — справочные данные из Structure Module.
- **Ставки** — формат «занято / всего» (`occupied_tariff_rates` / `tariff_rates`), плюс мини-progress-bar заполненности.
  - Прогресс-бар на 100% — нейтральный/зелёный цвет (штат закрыт).
  - Прогресс-бар на 0% — выделяется как «вакантно» (например, серый с акцентной подписью «Вакансия» рядом).
- **Оклад/ставка** — `salary`, сумма за одну полную ставку.
- **Контекстное меню `⋮`**: «Редактировать», «Удалить запись» (удаление доступно только если `occupied_tariff_rates = 0`, иначе пункт меню disabled с подсказкой «Нельзя удалить: на должность назначены сотрудники»).

## Форма «Добавить/редактировать запись штатного расписания»

```
┌────────────────────────────────────────────┐
│  Добавить запись штатного расписания     [✕]│
├────────────────────────────────────────────┤
│  Должность *                                 │
│  ┌──────────────────────────────────────┐   │
│  │ Выберите должность ▾                 │   │  ← справочник Structure Module;
│  └──────────────────────────────────────┘   │     одна должность — одна запись
│                                              │     расписания (предупреждение,
│                                              │     если запись уже существует)
│                                              │
│  Количество ставок *                         │
│  ┌──────────────────────────────────────┐   │
│  │  2.0                                 │   │  ← числовое поле, шаг 0.25
│  └──────────────────────────────────────┘   │
│  Занято сейчас: 1.5                          │  ← только в режиме редактирования,
│                                              │     read-only подсказка
│                                              │
│  Оклад за полную ставку *                    │
│  ┌──────────────────────────────────────┐   │
│  │  8 000 000                    UZS    │   │
│  └──────────────────────────────────────┘   │
│                                              │
│  Базовая роль при назначении                 │
│  ┌──────────────────────────────────────┐   │
│  │ Выберите роль (необязательно) ▾      │   │  ← справочник Auth Module
│  └──────────────────────────────────────┘   │
│  Эта роль будет автоматически выдана          │
│  сотруднику при назначении на должность.      │
│                                              │
├────────────────────────────────────────────┤
│                  [Отмена]      [Сохранить]  │
└────────────────────────────────────────────┘
```

- При редактировании: если пользователь пытается установить «Количество ставок» меньше текущего занятого значения — инлайн-ошибка под полем: «Нельзя установить меньше занятых ставок (1.5)».
- Шаг числового поля — 0.25, чтобы соответствовать допустимым значениям ставки (1 / 0.5 / 0.25) при назначении на должность.

## Состояния

| Состояние | Поведение |
|-----------|-----------|
| Empty | «Штатное расписание не заполнено» + кнопка «Добавить запись» |
| Loading | Скелетон таблицы |
| Error | Блок ошибки + «Повторить» |
| Permissions | Страница не отображается в меню без `HR_STAFFING_GET`; кнопки управления скрыты без `HR_STAFFING_MANAGE` |

## Acceptance Criteria

- Страница показывает список должностей с количеством ставок «занято / всего», окладом и визуальным индикатором заполненности.
- Заполненность 0% и 100% визуально выделены (вакансия / закрыто).
- Удаление записи недоступно, если по должности есть занятые ставки; в интерфейсе это отражено через disabled-пункт меню с поясняющей подсказкой.
- Форма редактирования не позволяет установить количество ставок ниже текущего занятого значения, с инлайн-валидацией.
- Поле «Базовая роль» — необязательное, с поясняющей подсказкой о её автоматической выдаче.
- Фильтр позволяет искать по отделу и по уровню заполненности ставок.
- Для страницы предусмотрены состояния: пусто, загрузка, ошибка, нет прав.
