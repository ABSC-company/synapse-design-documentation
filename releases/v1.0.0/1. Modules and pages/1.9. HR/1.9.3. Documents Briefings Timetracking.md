## 3.1 Вкладка «Документы» (личные документы сотрудника)

### Назначение

Личные документы сотрудника общего назначения: дипломы, удостоверения и прочее (не путать со специализациями врача из `02-doctor-data.md` — там документ привязан к конкретной специализации/допуску; здесь — самостоятельные документы личности и квалификации).

### Layout

```
┌─────────────────────────────────────────────────────────────┐
│  Документы                                  [+ Добавить документ]│
├─────────────────────────────────────────────────────────────┤
│  📄 Диплом о высшем образовании                                │
│  Тип: Диплом · Бессрочный                                     │
│  Паспорт.pdf                                      [✏] [✕]      │
├─────────────────────────────────────────────────────────────┤
│  📄 Удостоверение водителя                                    │
│  Тип: Удостоверение · до 12.05.2027     ● Действует            │
│                                                    [✏] [✕]      │
├─────────────────────────────────────────────────────────────┤
│  📄 Сертификат курсов первой помощи                            │
│  Тип: Сертификат · до 01.02.2026        ● Истекает скоро       │
│                                                    [✏] [✕]      │
└─────────────────────────────────────────────────────────────┘
```

- Тип документа — выбирается из справочника `employees_documents_types` (управляется отдельно, см. 3.1.2).
- Срочность: бессрочный документ помечается «Бессрочный» без бейджа статуса; срочный документ показывает дату окончания и статус-бейдж:

| Условие | Текст | Цвет |
|---------|-------|------|
| Срок действия далёкий (> 30 дней) | Действует | Зелёный |
| Срок истекает в ближайшие 30 дней (порог уточняется с продуктом) | Истекает скоро | Жёлтый |
| Срок истёк | Истёк | Красный |
| `indefinite = true` | Бессрочный | Серый, без срока |

### 3.1.1 Форма «Добавить/редактировать документ»

```
┌────────────────────────────────────────┐
│  Добавить документ                   [✕]│
├────────────────────────────────────────┤
│  Тип документа *                         │
│  ┌──────────────────────────────────┐   │
│  │ Выберите тип ▾                   │   │  ← справочник employees_documents_types
│  └──────────────────────────────────┘   │
│                                          │
│  [ ] Бессрочный документ                  │
│                                          │
│  Срок действия                            │  ← скрывается, если «Бессрочный» отмечен
│  ┌──────────┐    –    ┌──────────┐       │
│  │ от       │         │ до       │       │
│  └──────────┘         └──────────┘       │
│                                          │
│  Файл документа                           │
│  [Загрузить файл]                        │  ← паттерн File Module
│                                          │
├────────────────────────────────────────┤
│              [Отмена]      [Сохранить]  │
└────────────────────────────────────────┘
```

- Чекбокс «Бессрочный документ» переключает видимость полей периода (toggle, не одновременное отображение).

### 3.1.2 Управление типами документов (справочник)

Тех. задание выделяет отдельный CRUD для `employees_documents_types` (`/hr/document-types`). Для дизайна это можно реализовать как:
- ссылку «Управление типами документов» внутри формы добавления документа (рядом с выбором типа, иконка шестерёнки), открывающую простой реестр-модалку:

```
┌────────────────────────────────────────┐
│  Типы документов                     [✕]│
├────────────────────────────────────────┤
│  Диплом                          [✏][✕]│
│  Удостоверение                   [✏][✕]│
│  Сертификат                      [✏][✕]│
│                       [+ Добавить тип] │
├────────────────────────────────────────┤
│                              [Закрыть] │
└────────────────────────────────────────┘
```

- Форма добавления типа — название + описание, два текстовых поля, без дополнительной сложности.

## 3.2 Реестр шаблонов инструктажей (отдельная страница)

### Назначение

Управление шаблонами инструктажей, привязанных к должности. Находится в боковом меню отдельно от карточки сотрудника, так как шаблон не принадлежит конкретному сотруднику.

### Layout

```
┌─────────────────────────────────────────────────────────────┐
│  Шаблоны инструктажей                  [+ Создать шаблон]   │
├─────────────────────────────────────────────────────────────┤
│  🔍 Поиск по названию должности...                            │
├─────────────────────────────────────────────────────────────┤
│  Должность: Хирург                                            │
│  «Инструктаж по технике безопасности в операционной...»       │
│                                                  [✏] [✕]        │
├─────────────────────────────────────────────────────────────┤
│  Должность: Медсестра                                         │
│  «Инструктаж по обработке инструментов...»                    │
│                                                  [✏] [✕]        │
└─────────────────────────────────────────────────────────────┘
```

- Каждая карточка — должность + превью содержания инструктажа (обрезанный текст).
- Клик по карточке открывает полный текст в модальном окне или разворачивает inline.

### Форма «Создать/редактировать шаблон инструктажа»

```
┌────────────────────────────────────────────┐
│  Создать шаблон инструктажа              [✕]│
├────────────────────────────────────────────┤
│  Должность *                                 │
│  ┌──────────────────────────────────────┐   │
│  │ Выберите должность ▾                 │   │  ← справочник Structure Module
│  └──────────────────────────────────────┘   │
│                                              │
│  Содержание инструктажа *                    │
│  ┌──────────────────────────────────────┐   │
│  │                                      │   │  ← многострочный текстовый
│  │                                      │   │     редактор (rich text не
│  │                                      │   │     требуется по тех. заданию —
│  └──────────────────────────────────────┘   │     поле типа `text`)
│                                              │
├────────────────────────────────────────────┤
│                  [Отмена]      [Сохранить]  │
└────────────────────────────────────────────┘
```

## 3.3 Вкладка «Инструктажи» (в профиле сотрудника)

### Назначение

Фиксация фактов прохождения сотрудником инструктажей из общего реестра шаблонов.

### Layout

```
┌─────────────────────────────────────────────────────────────┐
│  Инструктажи                          [+ Отметить прохождение]│
├─────────────────────────────────────────────────────────────┤
```

- Список истории прохождений, отсортирован по дате (новые сверху).

## 3.4 Вкладка «Учёт рабочего времени» (в профиле сотрудника)

### Назначение

История отметок check-in/check-out сотрудника.

### Layout

```
┌─────────────────────────────────────────────────────────────┐
│  Учёт рабочего времени              [+ Добавить отметку]      │
├─────────────────────────────────────────────────────────────┤
│  🔍 Период:  [01.06.2026] – [25.06.2026]                       │
├─────────────────────────────────────────────────────────────┤
│  25.06.2026   09:02   ⬇ Приход                       [✕]      │
│  24.06.2026   18:05   ⬆ Уход                          [✕]      │
│  24.06.2026   09:00   ⬇ Приход                       [✕]      │
└─────────────────────────────────────────────────────────────┘
```

- Записи — таймстампы с типом `check-in` (Приход) / `check-out` (Уход), визуально различимы иконкой направления (вниз = приход, вверх = уход) и/или цветом.
- Фильтр по периоду — два date-picker'а сверху списка.
- Право на просмотр (`HR_TIME_TRACKING_GET`) отдельно от права на управление (`HR_TIME_TRACKING_MANAGE`) — кнопка добавления и `[✕]` показываются только при наличии второго права; список виден при наличии первого.

### Форма «Добавить отметку»

```
┌────────────────────────────────────────┐
│  Добавить отметку времени            [✕]│
├────────────────────────────────────────┤
│  Тип *                                   │
│  ○ Приход (check-in)  ○ Уход (check-out) │
│                                          │
│  Дата и время *                          │
│  ┌──────────────────────────────────┐   │
│  │  📅  🕐                          │   │
│  └──────────────────────────────────┘   │
├────────────────────────────────────────┤
│              [Отмена]      [Сохранить]  │
└────────────────────────────────────────┘
```

> Примечание: ручное добавление отметок — это административный сценарий (исправление пропущенной отметки), а не основной способ учёта времени (предполагается, что check-in/check-out обычно фиксируется автоматически через внешнее устройство/интеграцию, вне рамок этого ТД). Дизайнеру важно показать эту форму как редкое корректирующее действие, а не основной CTA экрана — например, кнопка может быть менее акцентной (secondary-стиль), чем обычно.

## Acceptance Criteria

- Вкладка «Документы» отличает срочные и бессрочные документы; срочные показывают статус-бейдж в зависимости от близости даты окончания.
- Чекбокс «Бессрочный» в форме документа скрывает поля периода.
- Справочник типов документов управляется через отдельный мини-реестр, доступный из формы добавления документа.
- Реестр шаблонов инструктажей — отдельная страница в боковом меню, не часть профиля сотрудника.
- Форма шаблона инструктажа требует выбора должности и текста содержания.
- Вкладка «Инструктажи» в профиле показывает историю прохождений с датой; форма отметки прохождения предлагает выбор из реестра шаблонов.
- Вкладка «Учёт времени» различает приход/уход визуально (иконка/цвет) и поддерживает фильтр по периоду.
- Право на просмотр учёта времени отделено от права на управление: кнопки добавления/удаления видны только при наличии прав на управление.
- Ручное добавление отметки времени визуально подано как редкое корректирующее действие, а не основной сценарий экрана.
