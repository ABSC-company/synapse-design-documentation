# 1.2.3. EMC Database Page

## Спецификация страницы интерфейса

-   Страница: База ЭМК
-   Модуль: Амбулаторная регистратура
-   Позиция в модуле: Третья страница интерфейса сотрудника регистратуры
-   Тип страницы: Рабочая область списка электронных медицинских карт
-   Основные роли: Регистратор, старший регистратор, администратор
    регистратуры
-   Статус документа: Draft

## Назначение страницы и сценарии использования

Страница предназначена для поиска, просмотра и управления списком
электронных медицинских карт пациентов.

### Ключевые задачи страницы:

1.  Просмотр списка медицинских карт пациентов
2.  Поиск записей по идентификатору ЭМК и данным пациента
3.  Фильтрация списка ЭМК
4.  Массовое управление выбранными записями
5.  Просмотр информации пациента
6.  Просмотр статусов и меток пациента
7.  Переход в детальную карточку ЭМК
8.  Выполнение действий через контекстное меню

## User Flows

### Flow 1. Просмотр списка ЭМК

1.  Пользователь открывает страницу База ЭМК
2.  Система отображает список медицинских карт
3.  Система загружает записи по умолчанию
4.  Пользователь просматривает список

### Flow 2. Поиск ЭМК

1.  Пользователь вводит значение в поиск
2.  Поиск выполняется по:

-   ID ЭМК
-   ФИО пациента
-   номеру телефона

3.  Система отображает результаты

### Flow 3. Выбор записей

1.  Пользователь отмечает чекбокс строки
2.  Система отображает Action Bar

Доступные действия:

-   экспорт
-   изменение статуса
-   изменение метки
-   удаление

## Структура: Wireframes

### Page Header

**Page Header** — заголовочный блок с кнопками управления (см. [Page Header.md](../1.2.0. Module layout components/1.2.0.1. Page Header/Page Header.md)):
- Кнопка **Зарегистрировать пациента** — открывает модаль регистрации нового пациента
- Кнопка **Запись пациента** — открывает модаль создания записи пациента к врачу

### Electronic Queue

**Electronic Queue** — блок электронной очереди по правому краю (см. [Electronic Queue.md](../1.2.0. Module layout components/1.2.0.2. Electronic Queue/Electronic queue.md))

### Контент

#### Toolbar

-   Поиск
-   Фильтр
-   Добавить
-   Массовые действия

#### Таблица ЭМК

  Элемент                  Описание
  ------------------------ ---------------------
  Checkbox                 Выбор записи
  ID                       Идентификатор ЭМК
  Пациент                  ФИО и телефон
  Дата последнего визита   Последнее посещение
  Метка                    Категория пациента
  Действия                 Контекстное меню

## Состояния

  Состояние     Поведение
  ------------- ---------------------
  loading       Skeleton таблицы
  empty         Нет данных
  error         Ошибка загрузки
  offline       Нет сети
  permissions   Ограничения по роли

## Acceptance criteria

1.  Отображается список ЭМК
2.  Поиск работает по ID и данным пациента
3.  Фильтр работает корректно
4.  Работают массовые действия
5.  Контекстное меню работает
6.  Реализованы состояния loading, empty, error, offline
