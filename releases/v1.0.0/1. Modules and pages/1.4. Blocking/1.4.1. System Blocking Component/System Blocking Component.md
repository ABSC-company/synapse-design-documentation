# 1.4.1. System Blocking Component

## Спецификация компонента

- Название компонента: Блокировка системы
- Тип: Глобальный компонент лейаута (оверлей)
- Область отображения: Поверх всего интерфейса, на любой странице системы
- Основные роли: любой авторизованный пользователь системы
- Статус документа: Draft
- Платформа: веб-приложение для медицинской регистратуры

## Назначение компонента и сценарии использования

Компонент предназначен для автоматической блокировки рабочего сеанса пользователя после 15 минут отсутствия активности на любой странице системы. Он накрывает весь интерфейс непрозрачным оверлеем и требует подтверждения личности пользователя через ввод пароля. Разблокировка не создаёт новый сеанс — она продолжает текущий.

Аналог: блокировка экрана в десктопном приложении Telegram.

Сценарии использования:

1. Пользователь оставил рабочее место, не выйдя из системы — через 15 минут система блокируется автоматически.
2. Пользователь возвращается к рабочему месту, видит экран блокировки и вводит пароль для продолжения работы.
3. Пользователь ввёл неверный пароль — система сообщает об ошибке и даёт повторить попытку.
4. После нескольких неверных попыток система завершает сеанс и перенаправляет на страницу авторизации.
5. Пользователь решает выйти из системы прямо с экрана блокировки, не разблокируя его.

## User Flows / Task Flows

### Flow 1. Автоматическая блокировка после неактивности

1. Пользователь работает в системе на любой странице.
2. Пользователь перестаёт взаимодействовать с интерфейсом (нет движения мыши, кликов, ввода с клавиатуры).
3. Через 14 минут появляется предупреждение: "Сессия будет заблокирована через 1 минуту".
4. Если пользователь не реагирует — через 15 минут компонент блокировки активируется поверх всего интерфейса.
5. Вся активность за оверлеем становится недоступной и визуально скрытой.

### Flow 2. Разблокировка через ввод пароля

1. Пользователь видит экран блокировки с именем текущего пользователя и полем ввода пароля.
2. Пользователь вводит пароль в поле.
3. Нажимает кнопку "Разблокировать" или клавишу Enter.
4. Система отправляет запрос на верификацию пароля.
5. При верном пароле оверлей плавно скрывается, интерфейс восстанавливается в том же состоянии, в котором был до блокировки.
6. Таймер неактивности сбрасывается.

### Flow 3. Неверный пароль

1. Пользователь вводит неверный пароль и нажимает "Разблокировать".
2. Система отображает сообщение об ошибке под полем ввода: "Неверный пароль. Осталось попыток: N".
3. Поле ввода очищается, фокус возвращается в поле.
4. Пользователь может повторить попытку.
5. После 5 неверных попыток подряд — сеанс завершается принудительно, пользователь перенаправляется на страницу авторизации.

### Flow 4. Выход из системы с экрана блокировки

1. Пользователь видит экран блокировки.
2. Пользователь нажимает кнопку "Выйти из системы".
3. Система показывает подтверждение: "Выйти из системы? Текущий сеанс будет завершён."
4. Пользователь подтверждает — сеанс завершается, пользователь перенаправляется на страницу авторизации.

### Flow 5. Предупреждение о скорой блокировке

1. За 1 минуту до блокировки в правом нижнем углу появляется toast-уведомление: "Сессия будет заблокирована через 60 секунд из-за неактивности."
2. В уведомлении есть кнопка "Продолжить работу".
3. Если пользователь нажимает "Продолжить работу" — таймер сбрасывается, уведомление скрывается.
4. Если пользователь не реагирует — по истечении 60 секунд активируется экран блокировки.

### Flow 6. Истечение сессии без возможности разблокировки

1. Серверный токен авторизации истёк, пока пользователь был заблокирован.
2. Пользователь возвращается и вводит верный пароль.
3. Сервер возвращает ошибку истечения сессии.
4. Система показывает сообщение: "Сессия истекла. Войдите заново."
5. Пользователь перенаправляется на страницу авторизации.

## Структура: Wireframes

### Глобальная структура оверлея

Компонент блокировки — это полноэкранный оверлей, который располагается в стеке отображения (`z-index`) поверх всех других элементов интерфейса, включая модальные окна, сайдбары, дропдауны и тосты.

```
┌─────────────────────────────────────────────────────────────┐
│                    ЭКРАН БЛОКИРОВКИ                         │
│              (перекрывает весь интерфейс)                   │
│                                                             │
│                                                             │
│              ┌─────────────────────────┐                   │
│              │                         │                   │
│              │    [Иконка замка]        │                   │
│              │                         │                   │
│              │  Система заблокирована  │                   │
│              │                         │                   │
│              │  Иван Иванов            │                   │
│              │  Регистратор            │                   │
│              │                         │                   │
│              │  [Поле: Пароль]  [👁]   │                   │
│              │                         │                   │
│              │  [Кнопка: Разблокировать] │                  │
│              │                         │                   │
│              │  Выйти из системы       │                   │
│              └─────────────────────────┘                   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### Фон оверлея

- Фон: размытый (`backdrop-filter: blur`) снимок интерфейса + тёмный полупрозрачный слой поверх.
- Это визуально сигнализирует пользователю, что интерфейс заблокирован, но он на нём находится.
- Контент за оверлеем не должен быть читаем — только размытая тень интерфейса.

### Центральная карточка

- Карточка выровнена строго по центру экрана (горизонтально и вертикально).
- Имеет скруглённые углы, тень, белый или светлый фон.
- Минимальный набор элементов карточки:

```
┌─────────────────────────────────┐
│                                 │
│          🔒                     │  ← иконка замка (крупная, по центру)
│                                 │
│   Система заблокирована         │  ← заголовок
│                                 │
│   ┌───────────────────────┐     │
│   │  Аватар  Иван Иванов  │     │  ← аватар + имя + роль текущего пользователя
│   │          Регистратор  │     │
│   └───────────────────────┘     │
│                                 │
│   Введите пароль для            │
│   продолжения работы            │  ← подсказка
│                                 │
│   ┌─────────────────────[👁]┐   │
│   │  Пароль                 │   │  ← поле ввода пароля
│   └─────────────────────────┘   │
│                                 │
│   [   Разблокировать    ]       │  ← primary кнопка
│                                 │
│   Выйти из системы              │  ← текстовая ссылка / secondary action
│                                 │
└─────────────────────────────────┘
```

### Предупреждающий toast (за 1 минуту до блокировки)

```
┌──────────────────────────────────────────────┐
│  ⏱  Сессия будет заблокирована через 60 сек  │
│                         [Продолжить работу]  │
└──────────────────────────────────────────────┘
```

- Располагается в правом нижнем углу экрана.
- Не перекрывает рабочую область критически, но привлекает внимание.
- Содержит обратный отсчёт в секундах.

## Визуал: High-Fidelity mockups

Визуальные правила:

1. Оверлей: размытый фон с тёмным полупрозрачным слоем (`rgba(0,0,0,0.6)` + blur ~12px).
2. Центральная карточка имеет белый или слегка тёплый фон, тень `box-shadow`, скруглённые углы (12–16px).
3. Иконка замка крупная (48–64px), нейтральный или акцентный цвет (серый или primary brand).
4. Имя пользователя и роль — чёткие, читаемые. Аватар — либо инициалы в круге, либо фотография.
5. Поле пароля — стандартный стиль полей системы с иконкой показа/скрытия символов.
6. Кнопка "Разблокировать" — primary action, выделяется цветом бренда.
7. Ссылка "Выйти из системы" — второстепенное действие, визуально менее акцентное (текстовая ссылка или ghost-кнопка).
8. Сообщение об ошибке — красный текст под полем пароля, мягкое появление.
9. Состояние загрузки — spinner на кнопке, поле и кнопка временно недоступны.
10. Toast предупреждения — стандартный компонент уведомлений системы, с иконкой таймера и кнопкой действия.

Обязательные визуальные компоненты:

- Полноэкранный оверлей с blur-фоном.
- Центральная карточка блокировки.
- Иконка замка.
- Блок идентификации пользователя (аватар, имя, роль).
- Поле ввода пароля с toggle видимости.
- Кнопка "Разблокировать".
- Ссылка "Выйти из системы".
- Сообщение об ошибке (под полем).
- Счётчик оставшихся попыток (при ошибках).
- Toast-уведомление за 60 секунд до блокировки.
- Состояние loading кнопки.

## Состояния и edge cases

### idle (экран заблокирован, ожидание ввода)

- Оверлей отображается поверх всего интерфейса.
- Пользователь видит карточку с полем пароля.
- Никакие элементы под оверлеем недоступны для клика, скролла, взаимодействия.
- Вкладка браузера остаётся активной (пользователь не вышел из системы).
- Фокус при появлении автоматически устанавливается на поле ввода пароля.

### loading (запрос верификации пароля)

- Кнопка "Разблокировать" переходит в состояние загрузки (spinner).
- Поле пароля становится недоступным для редактирования.
- Ссылка "Выйти из системы" остаётся доступной.
- Повторная отправка заблокирована.

### error (неверный пароль)

- Поле пароля очищается и подсвечивается красной рамкой.
- Под полем появляется сообщение об ошибке: "Неверный пароль. Осталось попыток: N."
- Фокус возвращается в поле ввода.
- Кнопка снова становится активной.

### locked-out (исчерпаны попытки)

- После 5 неверных попыток кнопка "Разблокировать" и поле пароля скрываются.
- Отображается сообщение: "Превышено количество попыток. Войдите заново."
- Остаётся только кнопка "Войти в систему", перенаправляющая на страницу авторизации.
- Сессия принудительно завершается.

### session-expired (серверный токен истёк)

- Пользователь вводит верный пароль, но сервер возвращает ошибку истечения сессии.
- Отображается сообщение: "Сессия истекла. Необходимо войти заново."
- Пользователь перенаправляется на страницу авторизации.

### offline (отсутствие сети)

- Пользователь пытается разблокировать систему без подключения к сети.
- Поле и кнопка остаются видимыми.
- При попытке отправки появляется сообщение: "Нет подключения к сети. Проверьте соединение."
- Кнопка "Разблокировать" становится неактивной, пока нет сети.
- Ссылка "Выйти из системы" остаётся доступной (выполняется локально).

### warning (60 секунд до блокировки)

- Toast-уведомление появляется в правом нижнем углу за 60 секунд до блокировки.
- Содержит обратный отсчёт (60, 59, 58...).
- Кнопка "Продолжить работу" сбрасывает таймер неактивности.
- Если пользователь закрывает toast без нажатия кнопки — таймер продолжается.
- Если toast закрыт и время истекло — блокировка активируется штатно.

### activity-reset (возобновление активности до блокировки)

- Если пользователь совершает любое действие (клик, ввод, движение мыши) до истечения 15 минут — таймер неактивности сбрасывается.
- Предупреждающий toast (если был показан) скрывается.
- Блокировка не активируется.

### tab-visibility (скрытая вкладка браузера)

- Если вкладка переведена в фон (пользователь переключился на другой таб) — таймер неактивности продолжает отсчёт.
- Блокировка активируется в фоновой вкладке и будет видна при возврате к ней.

### multiple-tabs (несколько вкладок с системой)

- Блокировка применяется к каждой вкладке независимо на основе активности в этой вкладке.
- Разблокировка одной вкладки не разблокирует остальные.

### keyboard-only (доступность)

- Оверлей полностью управляем с клавиатуры.
- Tab перемещает фокус между полем пароля, кнопкой "Разблокировать" и ссылкой "Выйти".
- Enter в поле пароля отправляет форму.
- Escape не закрывает оверлей (блокировка не может быть снята без ввода пароля).
- Фокус замкнут внутри карточки (focus trap) — нельзя перейти к элементам за оверлеем.

## Interaction & Animation

Правила поведения и переходы:

- Появление оверлея: плавный fade-in (200–300 мс), одновременно применяется blur на фон.
- Появление карточки: лёгкий scale от 0.96 до 1.0 + fade-in (200 мс). Даёт ощущение "выдвигания" карточки.
- Исчезновение оверлея (после разблокировки): fade-out (200–300 мс), blur снимается. Интерфейс восстанавливается плавно, без перезагрузки.
- Поле пароля получает focus-ring при фокусировке.
- Иконка toggle видимости пароля меняется мгновенно при клике.
- Ошибка валидации: красная рамка и текст появляются через fade-in (150 мс), без прыжков лейаута.
- Состояние loading кнопки: spinner заменяет текст, кнопка остаётся той же ширины.
- Toast предупреждения: выезжает снизу-справа (slide-up + fade-in, 250 мс), скрывается через slide-down + fade-out.
- Обратный отсчёт в toast обновляется каждую секунду без анимации (замена числа).
- Все переходы между состояниями карточки: 150–250 мс, без резких изменений размера карточки.

## Зависимости и ограничения

- API:
  - POST /auth/verify-password — верификация пароля пользователя без завершения сессии.
  - POST /auth/logout — завершение сессии при выборе "Выйти из системы".
- Таймер:
  - 15 минут (900 секунд) отсутствия событий: `mousemove`, `keydown`, `click`, `scroll`, `touchstart`.
  - Предупреждение показывается за 60 секунд до блокировки (на 14-й минуте).
  - Таймер работает только пока пользователь авторизован и компонент блокировки не активен.
- Стек отображения (z-index):
  - Компонент должен иметь максимальный z-index в системе, выше любых модальных окон, тостов, дропдаунов и сайдбаров.
- Ограничение попыток:
  - Максимум 5 неверных попыток ввода пароля, после чего — принудительный logout.
- Роли:
  - Компонент активен для любого авторизованного пользователя.
  - Страница авторизации и страница инициализации — исключения, на которых компонент не монтируется.
- Feature flags:
  - `session_lock_enabled` — включение/отключение механизма автоблокировки.
  - `session_lock_timeout_minutes` — переопределение таймаута (по умолчанию 15 минут).
- Технические ограничения:
  - Компонент монтируется на уровне корневого лейаута приложения (`root layout`).
  - Blur-эффект должен деградировать на браузерах без поддержки `backdrop-filter` (использовать полупрозрачный тёмный фон без blur).
  - Фокус-ловушка (focus trap) обязательна для соответствия WCAG 2.1 AA.
  - Форма должна поддерживать автозаполнение паролей браузером и менеджерами паролей.
  - В поле пароля запрещена вставка из буфера обмена через контекстное меню на shared-машинах (опционально).

## Acceptance criteria

- Оверлей появляется автоматически через 15 минут отсутствия пользовательской активности.
- Оверлей перекрывает весь интерфейс, включая модальные окна, сайдбары и тосты.
- Никакие элементы интерфейса за оверлеем не доступны для взаимодействия.
- За 60 секунд до блокировки появляется toast-предупреждение с обратным отсчётом.
- Кнопка "Продолжить работу" в toast сбрасывает таймер и скрывает предупреждение.
- На экране блокировки отображается имя, роль и аватар текущего пользователя.
- Ввод верного пароля снимает блокировку и восстанавливает интерфейс в исходное состояние.
- После разблокировки таймер неактивности сбрасывается.
- При неверном пароле отображается понятное сообщение с количеством оставшихся попыток.
- После 5 неверных попыток сессия завершается, пользователь перенаправляется на авторизацию.
- При истёкшей серверной сессии — пользователь перенаправляется на авторизацию с пояснением.
- При отсутствии сети кнопка "Разблокировать" неактивна, отображается сообщение об отсутствии сети.
- Кнопка "Выйти из системы" доступна всегда и завершает сессию с подтверждением.
- Фокус при появлении оверлея автоматически устанавливается на поле пароля.
- Фокус-ловушка работает корректно: Tab не позволяет выйти за пределы карточки.
- Все переходы и анимации плавные (150–300 мс) без резких изменений макета.
- Компонент не монтируется на страницах авторизации и инициализации.
