## Функциональные требования

| ID | Требование | Приоритет |
|---|---|---|
| `CASH-FR-01` | Система требует открытия смены перед любым кассовым действием | must |
| `CASH-FR-02` | Открытие смены автоматически создаёт связанные Z-отчёт и первый X-отчёт дня | must |
| `CASH-FR-20` | После закрытия текущего X-отчёта система автоматически открывает новый X-отчёт в рамках той же смены, без ручного создания | must |
| `CASH-FR-21` | Восстановление ошибочно закрытого Z-отчёта доступно самому кассиру, если в филиале/корпусе меньше 2 касс; при 2 и более кассовых рабочих местах в рамках этого филиала (мультикассовый режим) — только роли «Старший кассир» | must |
| `CASH-FR-22` | Список «ЭМК к оплате» показывает только визиты с непогашенной задолженностью и обновляется сразу после оплаты | must |
| `CASH-FR-03` | Кассир может найти визит/ЭМК к оплате по любому из параметров: ФИО, ПИНФЛ, номер ЭМК, номер визита, дата | must |
| `CASH-FR-04` | Кассиру доступны только административные данные ЭМК (пациент, услуги, сумма), клиническая часть скрыта | must |
| `CASH-FR-05` | Система поддерживает приём оплаты наличными, через терминал и через онлайн-системы (Payme, Click, Uzum) | must |
| `CASH-FR-06` | Статус оплаты для онлайн-систем подтверждается только колбэком платёжной системы, не действием кассира | must |
| `CASH-FR-07` | После успешной оплаты система автоматически формирует и печатает чек | must |
| `CASH-FR-08` | После успешной оплаты у визита/ЭМК снимается признак задолженности, визит пропадает из списка «ЭМК к оплате» | must |
| `CASH-FR-09` | Кассир может отложить оплату визита без его закрытия | must |
| `CASH-FR-10` | Кассир может отменить платёж до его перехода в статус «оплачен» | must |
| `CASH-FR-11` | Возврат по оплаченному платежу доступен только старшему кассиру и требует указания причины | must |
| `CASH-FR-23` | Кассир может провести оплату на сумму меньше остатка по счёту (частичная оплата); счёт переходит в статус «частично оплачен», а не закрывается | must |
| `CASH-FR-24` | Старший кассир может оформить возврат по конкретным услугам счёта (а не только произвольной суммой), если пациенту оказана группа услуг и нужно вернуть одну из них | must |
| `CASH-FR-25` | Каждый чек и итоговый Z-отчёт фискализируются через онлайн-ККМ и получают фискальный номер | must |
| `CASH-FR-26` | Пока счёт не оплачен/не отменён, в него может быть добавлена новая услуга того же визита с пересчётом суммы к оплате; для услуги в рамках другого визита всегда создаётся отдельный счёт | should |
| `CASH-FR-12` | Кассир может сформировать X-отчёт в любой момент открытой смены без влияния на её статус | must |
| `CASH-FR-13` | Закрытие Z-отчёта фиксирует итоги по способам оплаты и закрывает смену | must |
| `CASH-FR-14` | Старший кассир может восстановить ошибочно закрытый Z-отчёт с указанием причины | must |
| `CASH-FR-15` | Все денежные операции и восстановление Z-отчёта фиксируются в аудит-трейле | must |
| `CASH-FR-16` | Повторный колбэк платёжной системы с тем же идентификатором транзакции не создаёт повторного платежа/чека (идемпотентность) | must |
| `CASH-FR-17` | Перечень услуг и цены фиксируются в счёте на момент его выставления и не пересчитываются позже | must |
| `CASH-FR-18` | Сумма возвратов по одному платежу/счёту не может превышать фактически оплаченную сумму | must |
| `CASH-FR-19` | При закрытии смены система показывает расчётную сумму наличных; ввод фактической суммы необязателен, но при расхождении требует обязательного комментария | should |
