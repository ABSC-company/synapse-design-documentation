#### «Z-отчёт / Закрытие смены»

- **Назначение**: конец рабочего дня (`CASH-FR-13`, `CASH-FR-19`, `CASH-FR-25`).
- **Состав**: сводка нарастающим итогом за весь день по всем X-периодам; расчётная сумма наличных; необязательное поле фактической суммы с обязательным комментарием при расхождении; кнопка «Закрыть смену».

**Wireframe:**
```
┌────────────────────────────────────────────────┐
│  Закрытие смены (Z-отчёт) · 05.07.2026            │
├────────────────────────────────────────────────┤
│  Способ        Сумма       Чеков                 │
│  Наличные      2 340 000    18                   │
│  Терминал        980 000     7                   │
│  Payme           410 000     3                   │
│  Итого         3 730 000    28                   │
├────────────────────────────────────────────────┤
│  Расчётная сумма наличных:      2 590 000 UZS     │
│  Фактическая сумма (опц.):  [_________]           │
│  ⚠ Комментарий к расхождению:  [_____________]    │
│     (обязательно, если суммы не совпадают)        │
├────────────────────────────────────────────────┤
│                              [Закрыть смену]      │
└────────────────────────────────────────────────┘
```

**Примечания для дизайнера:**
- Поле «Фактическая сумма» — необязательное, но если заполнено и не равно расчётной, поле комментария появляется динамически и блокирует «Закрыть смену» пока не заполнено — показать оба состояния (без расхождения / с расхождением) в макете.
- «Закрыть смену» — необратимое действие с подтверждением (модалка «Смену нельзя будет продолжить, только восстановить с указанием причины»).
- После закрытия — Z-отчёт сразу уходит на фискализацию и печать, в UI это должно быть одним действием для кассира, а не двумя отдельными шагами.
