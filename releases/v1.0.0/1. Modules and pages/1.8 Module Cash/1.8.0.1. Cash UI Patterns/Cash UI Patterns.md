#### Общие UI-паттерны и обозначения

> Зафиксированы один раз, дальше разделы 6.1–6.9 ссылаются на них без повторного описания механики.

**Типы экранов модуля:**
- **Блокирующий модальный шаг** — «Открытие смены»: перекрывает весь интерфейс кассы, пока не выполнен.
- **Рабочий реестр** — «Список ЭМК к оплате», «Отложенные платежи», «Управление сменами»: таблица + фильтры + действие строки.
- **Одношаговая форма-действие** — «Оплата визита», «Возврат»: форма с явным основным действием внизу.
- **Документ-предпросмотр** — «Чек», «X-отчёт», «Z-отчёт»: неизменяемый снимок данных перед/после печати.

**Цветовая семантика статус-бейджей** (единая для модуля):
| Цвет | Значения |
|---|---|
| Зелёный | «оплачен», «открыта»/«открыт» (смена, X/Z-отчёт), «напечатан», подтверждение платёжной системы |
| Жёлтый | «частично_оплачен», «ожидает_подтверждения», «отложен», «черновик» |
| Красный | «отменён», «просрочен» (таймаут онлайн-оплаты), расхождение наличности сверх нормы |
| Серый | «закрыта»/«закрыт», «аннулирован», «отменён» (визит целиком, редкий случай) |
| Синий/акцент | «восстановлен» (Z-отчёт) — визуально должен отличаться от «открыт», это ручное вмешательство, а не обычный рабочий статус |

**Иконки экранов:** 🔓 Открытие смены · 💳 Список ЭМК к оплате · 🧾 Оплата визита · 🖨️ Чек · ⏸️ Отложенные платежи · 🔁 X-отчёт/пересменка · 🔒 Z-отчёт · 🗂️ Управление сменами · ↩️ Возврат.

**Компоненты:**
- **Модальное окно, блокирующее интерфейс** — только «Открытие смены»; ни один другой экран кассы не должен быть доступен, пока смена не открыта (в т.ч. через прямые ссылки/навигацию).
- **Индикатор остатка долга** — везде, где показана сумма счёта (список к оплате, форма оплаты), при `частично_оплачен` показываются одновременно «Итого», «Оплачено», «Остаток» — не только итоговая сумма.
- **Индикатор ожидания внешней системы** — единый компонент (спиннер + текст + таймер) для ожидания POS-терминала и колбэка платёжной системы; по истечении таймаута автоматически меняется на состояние «не подтверждено» с кнопкой «Проверить статус».
- **Toast** — печать чека/отчёта поставлена в очередь фискального регистратора, ошибка фискализации.
- **Переключатель роли рабочего места** — компактный контрол в шапке (не отдельная страница) для перехода «Регистратура ⇄ Касса» на местах, где роли совмещены (см. §4).

**Права доступа и видимость:**
- «Управление сменами» → действие «Восстановить» видно кассиру только если в филиале < 2 активных кассовых рабочих мест; при ≥ 2 — действие скрыто для кассира и видно только «Старшему кассиру» (см. `CASH-FR-21`).
- «Возврат» — пункт действия виден только «Старшему кассиру»; кассир видит оплаченный визит в режиме просмотра без кнопки возврата.
- Клиническая часть ЭМК нигде в кассовых экранах не отображается и не запрашивается (см. `CASH-FR-04`) — на макетах карточки визита не должно быть даже свёрнутых/задизейбленных клинических блоков.
