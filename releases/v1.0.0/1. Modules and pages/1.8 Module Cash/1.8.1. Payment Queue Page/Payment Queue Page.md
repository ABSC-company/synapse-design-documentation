#### «Список ЭМК к оплате» (поиск визитов к оплате)

- **Назначение**: главный рабочий экран кассира (`CASH-FR-03`, `CASH-FR-22`).
- **Состав**: по умолчанию — все визиты клиники с непогашенной задолженностью; поле поиска (ФИО, ПИНФЛ, номер ЭМК, номер визита) + быстрые фильтры (дата, отделение, статус: не оплачен/частично оплачен/отложен).
- **Колонки**: *Пациент, № визита, Отделение, Сумма итого, Оплачено, Остаток (долг), Статус*.
- **Действие строки**: «Оплатить».

**Wireframe:**
```
┌──────────────────────────────────────────────────────────────────────┐
│  ЭМК к оплате                                                          │
│  🔍 Поиск: ФИО / ПИНФЛ / № ЭМК / № визита    [Дата▾][Отделение▾][Статус▾]│
├───────────────────┬────────┬───────────┬────────┬─────────┬─────────┬──┤
│ Пациент           │№визита │ Отделение │ Итого  │ Оплачено│ Остаток │  │
├───────────────────┼────────┼───────────┼────────┼─────────┼─────────┼──┤
│ Каримов А.А.      │ 10245  │ Хирургия  │ 450 000│    0    │ 450 000 │●не оплач.│[Оплатить]│
│ Юсупова Н.Б.      │ 10231  │ Терапия   │ 320 000│  120 000│ 200 000 │●частично │[Оплатить]│
└───────────────────┴────────┴───────────┴────────┴─────────┴─────────┴──┘
```

**Примечания для дизайнера:**
- Строка исчезает из списка сразу после полного погашения — без ручного обновления/подтверждения (реактивно, по событию `CASH.ОплатаПринята`).
- Частично оплаченные строки — жёлтый статус-бейдж «частично» и видимая колонка «Остаток», чтобы не заходить в карточку ради суммы долга.
- Строка «Отложенные» может быть либо тем же статус-фильтром здесь, либо явной ссылкой на отдельный экран 6.5 — выбрать один способ и не дублировать оба одновременно.
