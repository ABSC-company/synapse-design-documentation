#### «Управление сменами»

- **Назначение**: восстановление ошибочно закрытых смен (`CASH-FR-14`, `CASH-FR-21`).
- **Состав**: список (*Дата, Кассир, Рабочее место, Статус, Итог*); фильтры (дата/кассир/рабочее место); действие «Восстановить» (обязательное поле «Причина») для закрытых смен.

**Wireframe:**
```
┌───────────────────────────────────────────────────────────┐
│  Управление сменами                     [Дата▾][Кассир▾]    │
├──────────┬─────────┬────────────┬─────────┬─────────┬──────┤
│ Дата     │ Кассир  │ Раб. место │ Статус  │ Итог    │      │
├──────────┼─────────┼────────────┼─────────┼─────────┼──────┤
│04.07.2026│ Иванов  │ Касса №1   │●Закрыта │3 730 000│[Восстановить]│
│05.07.2026│ Петров  │ Касса №2   │●Открыта │  820 000│      │
└──────────┴─────────┴────────────┴─────────┴─────────┴──────┘

Модалка восстановления:
┌──────────────────────────────────────────┐
│  Восстановить смену от 04.07.2026?         │
│  Причина * [_____________________]        │
├──────────────────────────────────────────┤
│                  [Отмена]  [Восстановить] │
└──────────────────────────────────────────┘
```

**Примечания для дизайнера:**
- Этот экран у кассира (режим одной кассы) и у старшего кассира (мультикасса) — один и тот же компонент; разница только в видимости пункта меню/действия (см. §6.0, «Права доступа»), не в вёрстке экрана.
- Восстановленная смена получает отдельный статус-бейдж «восстановлена» (акцентный цвет, не «открыта») — важно, чтобы в списке было видно, что смена не обычная, а была экстренно возвращена.
