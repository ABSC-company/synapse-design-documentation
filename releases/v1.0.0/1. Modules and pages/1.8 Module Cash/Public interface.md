## Публичный интерфейс (контракт с другими модулями)

**Синхронные методы:**

| Метод | Вход | Выход | Назначение |
|---|---|---|---|
| `ПолучитьСтатусОплаты` | visit_id/эмк_id | статус счёта (открыт / частично_оплачен / оплачен / отменён), сумма_итого, сумма_оплачено, остаток | Для `EMR`/`REG`, чтобы знать, погашена ли задолженность по визиту и в каком объёме |

**Публикуемые доменные события:**

| Событие | Когда | Полезная нагрузка |
|---|---|---|
| `CASH.СменаОткрыта` / `CASH.СменаЗакрыта` | открытие/закрытие смены | смена_id, кассир_id, рабочее_место_id |
| `CASH.СчётВыставлен` | создан `Invoice` по событию от `EMR`/`REG` | invoice_id, visit_id, сумма_итого |
| `CASH.ОплатаПринята` | платёж перешёл в `оплачен` | invoice_id, visit_id, сумма, способ_оплаты, номер_чека, остаток_по_счёту (0, если счёт закрыт полностью) |
| `CASH.ОплатаНеПодтверждена` | таймаут/отказ онлайн-оплаты (для уведомления через `NOTIF`) | invoice_id, visit_id, способ_оплаты, причина |
| `CASH.ПлатёжВозвращён` | проведён возврат (по сумме или по услугам) | invoice_id, visit_id, сумма_возврата, позиции_возврата (если есть), причина |
| `CASH.XОтчётЗакрыт` | X-отчёт закрыт (перерыв/пересменка) | x_отчёт_id, смена_id, кассир_id, итоги_по_способам_оплаты |
| `CASH.ZОтчётЗакрыт` | Z-отчёт закрыт | смена_id, итоги_по_способам_оплаты |
| `CASH.ZОтчётВосстановлен` | Z-отчёт восстановлен (кассиром либо старшим кассиром — см. §4) | смена_id, причина, кем |

**Зависимости от других модулей:** `EMR`/`REG` — публикуют событие «визит готов к оплате» с перечнем оказанных услуг, по которому `CASH` создаёт `Invoice`; `SVC` — синхронный запрос актуальных цен услуг в момент выставления счёта (замораживаются в `Invoice`, далее не пересчитываются); `HR` (кассир, старший кассир как сотрудники); `CLINIC` (рабочие места кассы, используются также для подсчёта порога мультикассового режима); `AUTH` (роли и права); `NOTIF` (уведомление кассира при таймауте/отказе онлайн-оплаты); внешние интеграции — платёжные системы Payme/Click/Uzum и фискальный регистратор/онлайн-ККМ (не модули системы, внешние сервисы).
