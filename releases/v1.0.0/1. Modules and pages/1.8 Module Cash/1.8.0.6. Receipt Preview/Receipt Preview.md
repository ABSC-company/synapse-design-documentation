####  «Чек»

- **Назначение**: подтверждение оплаты (`CASH-FR-07`, `CASH-FR-25`).
- **Состав**: предпросмотр (перечень оплаченных позиций, сумма платежа, способ оплаты, номер чека, фискальный номер, дата/время, кассир); печать сразу после оплаты; «Печать повторно» из карточки платежа.

**Wireframe:**
```
┌───────────────────────────────┐
│         ЧЕК № 000482           │
│  Дата: 05.07.2026 14:12        │
│  Кассир: Иванов И.И.           │
├───────────────────────────────┤
│ Анализ крови ОАК     170 000  │
│ Консультация          30 000  │
├───────────────────────────────┤
│ Итого:               200 000  │
│ Способ: Наличные               │
│ Фискальный №: FN-88213409       │
└───────────────────────────────┘
        [🖨 Печать повторно]
```

**Примечания для дизайнера:**
- Показывать позиции именно этого платежа (частичной оплаты), а не весь счёт — при повторном заходе на визит с двумя платежами должно быть два разных чека, не один сводный.
- Фискальный номер — всегда виден на чеке, не скрыт в «подробнее» (юридически значимый реквизит).
- Если фискализация не удалась (сбой ККМ) — чек не печатается, показывается блокирующая ошибка с «Повторить»; платёж остаётся оплаченным в системе, но повторная попытка печати обязательна до закрытия экрана.
