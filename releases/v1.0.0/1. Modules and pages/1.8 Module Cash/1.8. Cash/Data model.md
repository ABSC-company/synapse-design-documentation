#### 1.4. Счёт к оплате (Invoice)

> Решение по открытому вопросу «кто считает сумму к оплате»: сущность принадлежит `CASH`, а не `EMR`/`SVC`. `EMR`/`REG` только сообщают событием, что визит готов к оплате (перечень оказанных услуг); `CASH` в этот момент запрашивает у `SVC` актуальные цены по каждой услуге и замораживает их в счёте. Так `CASH` остаётся единственным владельцем всего, что связано с деньгами по визиту (включая частичную оплату и частичный возврат, см. ниже), а `EMR` не хранит финансовых данных, `SVC` — только справочник цен, а не факт задолженности.

| Поле | Тип | Обязательное | Примечание |
|---|---|---|---|
| id | UUID | да | |
| visit_id / эмк_id | FK (внешняя сущность, `EMR`/`REG`) | да | визит, за который выставлен счёт |
| перечень_услуг | JSON, массив (услуга_id, наименование, количество, цена, статус_позиции: активна/возвращена) | да | цены запрошены у `SVC` в момент добавления позиции и заморожены — не пересчитываются, даже если прайс изменится позже. Пока счёт не в статусе `оплачен`/`отменён`, `EMR`/`REG` могут прислать событие о новой оказанной услуге того же визита — она добавляется в этот же перечень (счёт «дорастает»), а не создаёт отдельный документ |
| сумма_итого | decimal | да | сумма по активным позициям перечня услуг; пересчитывается при добавлении новой услуги того же визита |
| сумма_оплачено | decimal | да | нарастающим итогом по всем `Payment` в статусе `оплачен`, привязанным к счёту |
| сумма_возвращено | decimal | да | нарастающим итогом по всем `Refund`, привязанным к счёту (через `Payment`) |
| статус | enum: открыт / частично_оплачен / оплачен / отменён | да | см. §2 — вычисляется из `сумма_итого`, `сумма_оплачено`, `сумма_возвращено` |
| дата_выставления | datetime | да | момент события от `EMR`/`REG` о готовности визита к оплате |

#### 1.5. Платёж (Payment)

| Поле | Тип | Обязательное | Примечание |
|---|---|---|---|
| id | UUID | да | |
| смена_id | FK Shift | да | привязка к смене, в рамках которой принят |
| invoice_id | FK Invoice | да | счёт, который погашается (полностью или частично) |
| visit_id / эмк_id | FK (внешняя сущность, `EMR`/`REG`) | да | дублирует `invoice.visit_id` — для удобства поиска/фильтрации |
| сумма | decimal | да | может быть меньше остатка по счёту — частичная оплата поддерживается |
| способ_оплаты | enum: наличные / терминал / онлайн_payme / онлайн_click / онлайн_uzum | да | |
| статус | enum: черновик / отложен / ожидает_подтверждения / оплачен / отменён / возвращён | да | см. §2 |
| транзакция_id_платёжной_системы | string | нет, кроме онлайн | id операции у Payme/Click/Uzum, для сверки |
| idempotency_key | string | обязательно для онлайн | ключ идемпотентности запроса создания платежа, чтобы повторный клик/повторный вебхук не породил дубль |
| валюта | string (ISO 4217, напр. `UZS`) | да | на случай мультивалютности в будущем — сейчас фиксируется явно, а не подразумевается |
| кассир_id | FK Сотрудник (`HR`) | да | |
| дата_время | datetime | да | |
| номер_чека | string | нет | заполняется после оплаты |
