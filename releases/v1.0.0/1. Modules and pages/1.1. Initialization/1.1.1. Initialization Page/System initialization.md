# 1.1.1. System Initialization Page

## Спецификация страницы интерфейса

| Параметр | Значение |
|----------|----------|
| Название страницы | Инициализация системы |
| Модуль | Management |
| Тип страницы | Мастер первоначальной настройки системы |
| Основные роли | Администратор клиники, Оператор клиники |
| Статус документа | Draft |
| Платформа | Web |
| Доступность | Только до завершения инициализации системы |

---

# Назначение страницы

Страница предназначена для первоначальной настройки системы после её установки и первого запуска.

До завершения процесса инициализации пользователь не может работать с остальными модулями системы. Все действия по подготовке системы к эксплуатации выполняются в рамках единого мастера настройки.

Во время прохождения мастера пользователь:

1. Выполняет подключение клиники.
2. Создаёт первого администратора.
3. Создаёт первый филиал.
4. Выполняет первоначальную настройку системы.
5. Запускает процесс инициализации.

После успешного завершения процесса система становится доступной для дальнейшей работы.

---

# Бизнес-логика страницы

Страница реализует процесс первоначальной инициализации системы.

Каждый этап мастера представляет отдельный шаг подготовки системы к эксплуатации.

Переход между этапами выполняется последовательно.

До успешного завершения текущего этапа переход к следующему этапу невозможен.

Во время прохождения мастера введённые пользователем данные сохраняются в локальном состоянии приложения и используются при формировании итогового запроса на инициализацию системы.

После завершения последнего этапа система выполняет полный процесс первоначальной настройки, который включает:

- подключение клиники;
- создание первого администратора;
- создание административной роли;
- создание первого филиала;
- создание базовой организационной структуры;
- сохранение глобальных настроек;
- завершение процесса инициализации.

После успешного выполнения всех операций пользователь автоматически перенаправляется на страницу авторизации.

Если во время выполнения процесса возникает ошибка, система прекращает выполнение операции, отображает описание проблемы и сохраняет возможность повторного запуска после устранения причины ошибки.

---

# Доступ к странице

Страница автоматически открывается после первого запуска системы.

После завершения процесса инициализации повторный доступ к странице невозможен.

Если система уже находится в состоянии **Initialized**, пользователь автоматически перенаправляется на страницу авторизации.

---

# Layout страницы

Страница представляет собой полноэкранный мастер настройки (**Wizard**).

Общий Layout состоит из четырёх областей.

```
+---------------------------------------------------------------+
| Header                                                        |
+---------------------------------------------------------------+
| Stepper                                                       |
+---------------------------------------------------------------+
|                                                               |
|                      Content Area                             |
|                                                               |
|                                                               |
+---------------------------------------------------------------+
| Footer                                                        |
+---------------------------------------------------------------+
```

Области **Header**, **Stepper** и **Footer** являются фиксированными.

Прокрутка выполняется только внутри рабочей области (**Content Area**).

При переходе между этапами Layout страницы не изменяется.

---

# Header

Header располагается в верхней части страницы.

Основное назначение Header — информирование пользователя о текущем состоянии процесса инициализации.

## Header содержит

- название страницы;
- описание текущего этапа;
- номер текущего этапа;
- общее количество этапов;
- индикатор прогресса прохождения мастера.

### Пример

```
Инициализация системы

Шаг 2 из 5

Создание первого администратора
```

Информация обновляется автоматически при переходе между этапами.

---

# Stepper

Под Header располагается горизонтальный Stepper.

Тип компонента:

**Horizontal Stepper**

Stepper отображает последовательность этапов мастера.

Количество этапов фиксировано.

Этапы:

1. Подключение клиники
2. Администратор
3. Филиал
4. Настройки системы
5. Завершение

---

## Состояния этапов

Каждый этап может находиться в одном из следующих состояний.

### Completed

Этап успешно завершён.

Пользователь может вернуться к нему для просмотра или изменения данных.

---

### Active

Текущий активный этап.

Отображается визуально выделенным.

---

### Disabled

Этап недоступен.

Переход невозможен до завершения предыдущего этапа.

---

### Error

Во время выполнения этапа обнаружены ошибки.

Этап отмечается индикатором ошибки до момента её устранения.

---

## Правила работы Stepper

Stepper отображает:

- текущий этап;
- завершённые этапы;
- оставшиеся этапы;
- общий прогресс прохождения мастера.

Переход между этапами осуществляется только при помощи кнопок управления мастером.

Произвольный переход по Stepper не допускается.

---

# Content Area

Content Area является основной рабочей областью страницы.

На каждом этапе отображается собственный интерфейс, соответствующий текущему шагу мастера.

Рабочая область поддерживает:

- вертикальную прокрутку;
- отображение длинных форм;
- динамическое изменение содержимого;
- отображение состояний загрузки;
- отображение сообщений об ошибках;
- отображение уведомлений об успешном выполнении операций.

Изменение содержимого происходит без перезагрузки страницы.

---

# Footer

Footer располагается в нижней части страницы.

Footer всегда остаётся закреплённым независимо от положения прокрутки.

Footer содержит элементы управления процессом настройки.

## Кнопки

- Назад
- Далее
- Завершить настройку

---

## Правила отображения

### Первый этап

Отображается только кнопка **Далее**.

Кнопка **Назад** отсутствует.

---

### Промежуточные этапы

Отображаются:

- Назад
- Далее

---

### Последний этап

Отображаются:

- Назад
- Завершить настройку

---

## Правила навигации

Кнопка **Далее** становится доступной только после успешного завершения текущего этапа.

Переход к следующему этапу невозможен если:

- не заполнены обязательные поля;
- имеются ошибки валидации;
- выполняется операция сохранения;
- выполняется запрос к серверу.

Во время выполнения длительных операций кнопки Footer блокируются.

---

# User Flows

## Flow 1. Полная инициализация системы

1. Пользователь открывает страницу инициализации.
2. Выполняет подключение клиники.
3. Создаёт первого администратора.
4. Создаёт первый филиал.
5. Настраивает параметры системы.
6. Запускает процесс инициализации.
7. Дожидается завершения выполнения всех операций.
8. Получает уведомление об успешном завершении настройки.
9. Автоматически перенаправляется на страницу авторизации.

---

## Flow 2. Подключение клиники

1. Пользователь выбирает способ подключения.
2. Загружает конфигурационный файл либо выполняет сканирование QR-кода.
3. Система выполняет проверку полученных данных.
4. После успешной проверки отображается информация о подключении.
5. Пользователь переходит к следующему этапу.

---

## Flow 3. Создание администратора

1. Пользователь вводит данные первого администратора.
2. Система выполняет проверку корректности заполнения формы.
3. После успешной проверки данные сохраняются в мастере настройки.
4. Пользователь переходит к следующему этапу.

---

## Flow 4. Создание филиала

1. Пользователь вводит данные первого филиала.
2. Настраивает местоположение.
3. Настраивает региональные параметры.
4. Настраивает график работы.
5. Сохраняет введённые данные.
6. Переходит к следующему этапу.

---

## Flow 5. Настройка системы

1. Пользователь открывает разделы настроек.
2. Последовательно настраивает параметры системы.
3. Сохраняет изменения.
4. Переходит к последнему этапу.

---

## Flow 6. Завершение инициализации

1. Пользователь запускает процесс инициализации.
2. Система выполняет все необходимые операции.
3. После успешного завершения отображается экран завершения.
4. Пользователь автоматически перенаправляется на страницу авторизации.

# Шаг 1. Подключение клиники

## Назначение

Этап предназначен для подключения клиники к системе и проверки возможности дальнейшего выполнения процесса инициализации.

Подключение выполняется на основании конфигурационных данных, предоставленных системой централизованного управления.

До успешного завершения данного этапа переход к следующим этапам мастера невозможен.

После успешной проверки система сохраняет параметры подключения и разрешает продолжить первоначальную настройку.

---

# Бизнес-логика этапа

Подключение клиники является обязательным этапом процесса инициализации.

Пользователь предоставляет конфигурационные данные одним из поддерживаемых способов.

После получения конфигурации система автоматически:

- считывает данные подключения;
- выполняет проверку корректности полученной информации;
- проверяет возможность подключения клиники;
- получает параметры, необходимые для дальнейшей настройки системы;
- сохраняет результат проверки.

Если проверка завершается успешно, пользователь получает возможность перейти к следующему этапу мастера.

Если во время проверки возникает ошибка, переход к следующему этапу блокируется до устранения причины ошибки.

---

# Layout этапа

Страница состоит из нескольких логических блоков.

```
+---------------------------------------------------------------+
| Информация                                                     |
+---------------------------------------------------------------+

+--------------------------+   +-------------------------------+
| Загрузка JSON            |   | Сканирование QR-кода          |
+--------------------------+   +-------------------------------+

+---------------------------------------------------------------+
| Информация о подключении                                      |
+---------------------------------------------------------------+

+---------------------------------------------------------------+
| Результат проверки                                            |
+---------------------------------------------------------------+
```

---

# Способы подключения

Пользователь может выполнить подключение одним из следующих способов.

## Загрузка конфигурационного файла

Позволяет загрузить ранее полученный конфигурационный JSON-файл.

После выбора файла система автоматически начинает его обработку.

---

## Сканирование QR-кода

Позволяет получить конфигурацию путём сканирования QR-кода.

После успешного считывания система автоматически начинает обработку полученных данных.

---

# Блок загрузки JSON

## Назначение

Используется для загрузки конфигурационного файла клиники.

---

## Состав блока

### Drag & Drop

Позволяет перетащить файл в область загрузки.

---

### Кнопка выбора файла

Открывает системный диалог выбора файла.

---

### Поддерживаемые форматы

Допускается загрузка файлов формата:

- JSON

Попытка загрузить файл другого формата приводит к отображению сообщения об ошибке.

---

# Блок сканирования QR-кода

## Назначение

Используется для получения конфигурации через QR-код.

---

## Элементы интерфейса

### Область камеры

Отображает изображение с камеры устройства.

---

### Кнопка запуска

Запускает процесс сканирования.

---

### Индикатор сканирования

Отображается во время поиска QR-кода.

После успешного считывания автоматически начинается обработка данных.

---

# Информация о подключении

После успешной обработки конфигурации отображается карточка подключения.

Карточка содержит сведения о подключаемой организации.

## Отображаемые данные

- идентификатор клиники;
- тип подключения;
- статус подключения;
- состояние проверки конфигурации.

Если часть информации отсутствует, отображаются только доступные данные.

---

# Поведение системы

После получения конфигурации система автоматически запускает процесс проверки.

Во время проверки пользователь не выполняет дополнительных действий.

Переход к следующему этапу становится доступным только после успешного завершения проверки.

Если проверка завершается ошибкой, система отображает описание проблемы и предоставляет возможность повторить попытку подключения.

---

# Валидация

Перед завершением этапа система должна убедиться, что:

- конфигурационный файл успешно прочитан;
- структура конфигурации корректна;
- полученные данные прошли проверку;
- подключение выполнено успешно.

При наличии ошибок кнопка **«Далее»** остаётся недоступной.

---

# Состояния страницы

## Empty

Конфигурационные данные отсутствуют.

Отображаются способы подключения.

---

## Loading

Выполняется обработка конфигурации.

Все элементы управления блокируются.

Отображается индикатор выполнения операции.

---

## Success

Подключение успешно завершено.

Отображается карточка подключения.

Пользователь может перейти к следующему этапу.

---

## Error

Во время проверки обнаружена ошибка.

Отображается сообщение с описанием причины ошибки.

Пользователь может повторить попытку подключения.

---

# Возможные ошибки

## Некорректный формат файла

Причина:

Пользователь выбрал файл неподдерживаемого формата.

---

## Повреждённый файл

Причина:

Не удалось прочитать содержимое файла.

---

## Некорректные конфигурационные данные

Причина:

Структура файла не соответствует ожидаемому формату.

---

## Ошибка подключения

Причина:

Не удалось установить соединение с сервером.

---

## Ошибка проверки данных

Причина:

Полученные параметры подключения не прошли проверку.

---

# Interaction

- Обработка начинается автоматически после выбора файла или успешного сканирования QR-кода.
- Во время проверки элементы управления блокируются.
- После успешного подключения карточка организации отображается автоматически.
- При повторной загрузке файла предыдущие результаты проверки заменяются новыми.
- При возникновении ошибки пользователь может повторить подключение без перезагрузки страницы.

---

# Acceptance Criteria

- Пользователь может выбрать JSON-файл.
- Пользователь может подключить клинику с помощью QR-кода.
- Система автоматически начинает обработку конфигурации.
- После успешной проверки отображается информация о подключении.
- До успешного завершения проверки переход к следующему этапу невозможен.
- При возникновении ошибки отображается понятное сообщение с описанием проблемы.
- Пользователь может повторить попытку подключения.
