# 1.1.1. System Initialization Page

## Спецификация страницы интерфейса

* Название страницы: Инициализация системы
* Модуль: Management
* Тип страницы: Первичная настройка клиники
* Основные роли: Администратор клиники, Оператор клиники
* Статус документа: Draft
* Платформа: веб-приложение для медицинской системы

---

# Назначение страницы и сценарии использования

Страница предназначена для первоначальной настройки системы после установки.

С помощью страницы пользователь может:

1. Интегрировать клинику на основе конфигурационных данных.
2. Создать первого администратора системы.
3. Настроить первый филиал.
4. Настроить системные параметры.
5. Завершить процесс активации системы.

После успешного завершения инициализации пользователь получает возможность авторизоваться и работать в системе.

---

# Layout страницы

Страница представляет собой полноэкранный мастер настройки (Wizard).

Состоит из следующих областей:

## Header

Содержит:

* Название страницы
* Краткое описание текущего этапа
* Индикатор общего прогресса настройки

---

## Stepper

Расположен под Header.

Тип:

* Horizontal Stepper

Этапы:

1. Подключение клиники
2. Администратор
3. Филиал
4. Настройки системы
5. Завершение

Состояния этапов:

* Completed
* Active
* Disabled
* Error

Пользователь всегда видит:

* текущий этап;
* завершённые этапы;
* общее количество этапов.

---

## Content Area

Основная рабочая область страницы.

Отображает контент текущего шага.

Контент может прокручиваться независимо от остальных элементов страницы.

---

## Footer

Закреплён в нижней части страницы.

Содержит кнопки управления мастером:

* Назад
* Далее
* Завершить настройку

Правила:

* На первом шаге кнопка «Назад» скрыта.
* На промежуточных шагах отображаются кнопки «Назад» и «Далее».
* На последнем шаге отображается кнопка «Завершить настройку».
* Переход на следующий шаг невозможен при наличии ошибок валидации.

---

# User Flows

## Flow 1. Полная настройка системы

1. Пользователь открывает страницу.
2. Загружает данные клиники.
3. Создаёт администратора.
4. Создаёт первый филиал.
5. Настраивает параметры системы.
6. Завершает настройку.
7. Получает доступ к авторизации.

---

## Flow 2. Подключение клиники

1. Пользователь загружает JSON-файл или сканирует QR-код.
2. Система считывает конфигурацию.
3. Отображает информацию о найденной клинике.
4. Пользователь подтверждает данные.
5. Переходит к следующему шагу.

---

## Flow 3. Создание администратора

1. Пользователь вводит данные администратора.
2. Создаёт первую административную роль.
3. Переходит к следующему этапу.

---

## Flow 4. Настройка филиала

1. Пользователь указывает данные филиала.
2. Настраивает местоположение.
3. Настраивает расписание работы.
4. Подтверждает данные.

---

## Flow 5. Настройка системы

1. Пользователь открывает разделы настроек.
2. Настраивает необходимые параметры.
3. Сохраняет значения.
4. Завершает инициализацию.

---

# Шаг 1. Подключение клиники

## Назначение

Подключение клиники на основании предоставленных конфигурационных данных.

## Элементы интерфейса

### Блок загрузки

Содержит:

* Drag & Drop область
* Кнопку загрузки JSON-файла
* Поддерживаемые форматы файлов

---

### Блок сканирования

Содержит:

* Область камеры
* Кнопку запуска сканирования
* Индикатор успешного считывания QR-кода

---

### Карточка клиники

После успешного чтения данных отображается карточка:

Поля:

* Название клиники
* Идентификатор клиники
* Тип лицензии
* Страна
* Статус подключения

---

# Шаг 2. Администратор

## Назначение

Создание первого администратора системы.

## Поля

* Логин
* Пароль
* Подтверждение пароля
* Имя
* Фамилия
* Email
* Телефон
* Название роли

## Дополнительные элементы

* Загрузка логотипа клиники

---

# Шаг 3. Филиал

## Назначение

Создание первого филиала организации.

## Поля

### Основная информация

* Название филиала
* Страна
* Город
* Адрес

### Геолокация

* Интерактивная карта
* Координаты

### Региональные настройки

* Язык системы
* Часовой пояс

### Рабочий график

Для каждого дня недели:

* Время открытия
* Время закрытия

### Перерывы

Возможность добавить несколько временных интервалов.

Каждый интервал содержит:

* Время начала
* Время окончания

---

# Шаг 4. Настройки системы

## Назначение

Первичная настройка глобальных параметров системы.

---

## Общий Layout

Шаг состоит из двух областей.

### Левая панель

Список разделов настроек.

Примеры:

* General
* Structure
* HR
* Files
* EDM

Количество разделов заранее неизвестно.

Список должен поддерживать вертикальную прокрутку.

---

### Правая панель

Контент выбранного раздела.

Содержит:

* Название раздела
* Описание раздела
* Список настроек

---

## Карточка настройки

Каждая настройка отображается отдельным блоком.

Состав блока:

### Заголовок

Название настройки.

Пример:

Название клиники

---

### Описание

Краткое описание назначения настройки.

Пример:

Название организации, отображаемое в системе.

---

### Поле ввода

Тип зависит от конфигурации настройки.

---

### Ошибка валидации

Отображается под полем.

---

## Поддерживаемые типы настроек

### Text Input

Однострочное текстовое поле.

---

### Number Input

Поле для ввода чисел.

---

### Text Area

Многострочный ввод текста.

---

### Toggle

Переключатель Да / Нет.

---

### Select

Одиночный выбор значения.

---

### Multi Select

Множественный выбор значений.

---

### Date Picker

Выбор даты.

---

### Time Picker

Выбор времени.

---

### File Upload

Загрузка файлов.

---

### Collection

Редактируемая таблица записей.

Поддерживает:

* Добавление строки
* Удаление строки
* Редактирование строки

---

## Правила отображения

* Интерфейс должен поддерживать неограниченное количество разделов.
* Интерфейс должен поддерживать неограниченное количество настроек.
* Header, Stepper и Footer остаются фиксированными.
* Прокручивается только область контента.
* Изменение значения происходит без перезагрузки страницы.
* Каждая настройка отображает название, описание и текущее значение.

---

# Шаг 5. Завершение

## Экран подтверждения

Отображается итоговая информация:

* Клиника подключена
* Администратор создан
* Филиал создан
* Настройки сохранены

---

## Кнопка

Завершить настройку

---

# Состояния

## Empty

* Данные отсутствуют.
* Следующий шаг недоступен.

---

## Loading

* Выполняется операция.
* Элементы формы заблокированы.
* Отображается индикатор выполнения.

---

## Error

* Ошибка загрузки файла.
* Ошибка проверки данных.
* Ошибка сохранения.
* Ошибка инициализации.

Сообщение должно содержать понятное описание проблемы.

---

## Success

* Данные успешно сохранены.
* Пользователь получает уведомление.

---

## Offline

* Отсутствует соединение с сервером.
* Отображается предупреждение.
* Отправка данных недоступна.

---

# Interaction & Animation

* Переключение между шагами происходит плавно.
* Активный шаг визуально выделяется.
* Ошибки отображаются без смещения интерфейса.
* Загрузка сопровождается индикатором прогресса.
* Завершение настройки сопровождается уведомлением об успехе.
* Все переходы между состояниями должны быть плавными и предсказуемыми.

---

# Acceptance Criteria

* Пользователь может пройти все этапы настройки в рамках одного мастера.
* Поддерживается загрузка JSON-файла.
* Поддерживается сканирование QR-кода.
* Можно создать первого администратора.
* Можно создать первый филиал.
* Можно настроить системные параметры.
* Все обязательные поля проходят валидацию.
* Все состояния отображаются корректно.
* После завершения настройки пользователь перенаправляется на страницу авторизации.
