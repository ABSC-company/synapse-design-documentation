## Состояния экранов

| Состояние | Что видит пользователь |
|------------|--------------------|
| Загрузка | Скелетон таблицы/списка (пользователи, роли, аудит-лог) |
| Пусто | «Ничего не найдено» — для списков и результатов фильтра |
| Ошибка | Сообщение об ошибке с возможностью повторить |
| Нет связи | Баннер отсутствия сети, действия изменения данных недоступны |

## Отдельные случаи, которые стоит предусмотреть в макетах

**Мастер первичной настройки**
- Визард прерван на середине (закрыта вкладка/потеряна сессия) — при следующем входе продолжает с прерванного шага, а не начинается заново.

**Роли и права**
- Попытка удалить или лишить критичного права системную роль — действие недоступно на уровне интерфейса, без диалога подтверждения, который бы намекал на возможность.
- Роль архивирована, но назначена активным пользователям — их доступ сохраняется, роль просто исчезает из списка доступных при назначении новым.

**Пользователи**
- Форма сохранения пользователя без единой назначенной роли — блокируется с понятным объяснением, а не просто подсвечивается как «обязательное поле».
- Отзыв сессии, которую пользователь уже сам закрыл — тост «Сессия уже завершена», а не ошибка.
- Роль, требующая 2FA, назначается пользователю без подключённого второго фактора — интерфейс заранее предупреждает, что 2FA потребуется подключить при следующем входе.

**Откат статуса**
- У сущности нет предоставленной модулем операции отката — действие «Откатить» не отображается вовсе, не как disabled-кнопка.

**Техподдержка**
- Целевой пользователь офлайн в момент подключения — уведомление всё равно отправляется и будет видно, когда пользователь войдёт в систему в следующий раз.

**2FA**
- Пользователь потерял и устройство, и резервные коды одновременно — доступ восстанавливает только другой администратор через сброс 2FA в карточке пользователя (см. [Users Page.md](Users%20Page.md)).
