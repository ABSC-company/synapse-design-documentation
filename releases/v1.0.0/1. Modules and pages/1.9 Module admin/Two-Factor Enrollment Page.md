## Подключение 2FA

Экран для любого пользователя, которому по роли требуется второй фактор (администратор, роли с доступом к техподдержке или сбросу пароля) — не только для администратора, показывается при первом входе такого пользователя, а также в рамках мастера первичной настройки (шаг 3).

**Подключение**
QR-код TOTP-секрета → пользователь сканирует его приложением-аутентификатором и вводит проверочный код → после подтверждения — список одноразовых резервных кодов с явным предупреждением «сохраните их сейчас, второй раз они не покажутся».

**Восстановление доступа**
Вход по одному из резервных кодов → система предлагает подключить 2FA заново: новый секрет и новый набор резервных кодов, старые аннулируются.

---

### Заметки для макетов

- Экран с резервными кодами — единственный момент, когда их можно увидеть; нужны и копирование в буфер, и скачивание файлом, чтобы не полагаться на то, что пользователь успеет их куда-то переписать вручную.
- Закрытие экрана с кодами кликом мимо или по Esc — рискованное поведение (см. [Interaction and animation.md](Interaction%20and%20animation.md)): продолжить дальше должно быть можно только через явное подтверждение «Я сохранил коды».
- Сценарий восстановления (вход по резервному коду) выглядит иначе, чем обычный вход, и должен явно называться «восстановление доступа», а не мимикрировать под обычный ввод кода — пользователь должен понимать, что использует последний оставшийся код.
