## Аудит-лог

Таблица: дата/время, пользователь, модуль, сущность, действие, старое значение → новое значение, IP/сессия. Фильтры по всем колонкам, экспорт.

Панель администратора не пишет эти записи сама — каждая создаётся тем модулем, в котором произошло изменение. Аудит-лог здесь — это фильтруемый просмотр уже существующего журнала, а не отдельная система логирования.

---

### Заметки для макетов

- «Старое значение → новое значение» приходит из совершенно разных модулей с разными типами данных (текст, статус, сумма, список) — нужен один универсальный визуальный паттерн diff'а (например, зачёркнутое старое / выделенное новое), который одинаково хорошо читается для короткого статуса и для длинного текстового поля.
- Значений в колонке «Модуль» может стать много по мере роста системы — фильтр по модулю должен быть удобным при 10+ вариантах (поиск в списке, а не длинный чекбокс-список).
- Строки, относящиеся к откату статуса, стоит визуально не смешивать с обычными изменениями (см. [Reports and Rollback Journal Page.md](Reports%20and%20Rollback%20Journal%20Page.md)) — это более чувствительное действие, чем рядовое редактирование.
