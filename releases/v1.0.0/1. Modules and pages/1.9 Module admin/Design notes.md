## Что важно для макетов

1. Панелью пользуются редко, но люди с максимальным доступом — стоит закладывать больше подтверждающих диалогов на необратимые действия (архивация роли, откат статуса, сброс пароля, отзыв сессии), чем в остальном интерфейсе системы.
2. Статусы (активен/заблокирован, активна/завершена, идёт приём и т.п.) — переиспользовать цветовой язык статусов, уже принятый в остальных модулях (визиты, кассовые смены), не изобретать новый только для админ-панели.
3. «Причина» — обязательное текстовое поле в нескольких не связанных на первый взгляд местах (подключение техподдержки, откат статуса) — стоит использовать один и тот же компонент ввода причины везде, где она нужна.
4. Индикатор активной имперсонации — самый заметный элемент из всей панели: он должен читаться с одного взгляда и администратором, и пользователем, чья сессия сейчас используется.
5. Дерево оргструктуры и матрица прав ролей — оба потенциально длинные и вложенные списки; для обоих нужны сворачивание и поиск с самого начала, а не как доработка «когда данных станет много».
