## Функциональные требования

| ID | Требование | Приоритет |
|---|---|---|
| `PHYS-FR-01` | Приём пациента у врача создаётся по вызову из электронной очереди | must |
| `PHYS-FR-02` | Врач видит на «Рабочем столе» список пациентов, ожидающих приёма именно у него, с актуальным статусом каждого приёма | must |
| `EMR-FR-01` | ЭМК пациента без факта обращения в клинику отображается врачу в ограниченном виде (без содержательной истории) | must |
| `EMR-FR-02` | Поиск пациента доступен по ФИО, номеру ЭМК и ПИНФЛ | must |
| `EMR-FR-03` | Консультация после подписания недоступна для прямой правки — только для создания новой версии или дополнения | must |
| `EMR-FR-04` | Аллергии пациента отображаются на экране консультации без дополнительных действий врача | must |
| `EMR-FR-07` | Врач может открыть ЭМК до начала приёма только если пациент записан к нему, в его очереди или направлен к нему в рамках текущего визита | must |
| `EMR-FR-08` | Экстренный доступ к чужой ЭМК возможен только с обязательным указанием причины и фиксируется в аудите отдельной пометкой | must |
| `PHYS-FR-03` | Врач может создать направление к конкретному целевому врачу из открытой консультации, выбрав один из четырёх типов направления | must |
| `PHYS-FR-04` | Направление типа «внутри текущего визита» и «с возвратом» создаёт приём у целевого врача в рамках того же визита в клинику, с тем же номерком очереди | must |
| `PHYS-FR-10` | Направление типа «с возвратом» приостанавливает исходный приём и возобновляет его после завершения целевого | must |
| `PHYS-FR-11` | Направление типа «плановое на другой день» не создаёт приём/талон в текущей очереди | must |
| `PHYS-FR-05` | Врач может назначить лабораторное и/или инструментальное исследование из консультации | must |
| `PHYS-FR-06` | Результат исследования автоматически появляется в консультации/ЭМК без ручного импорта | must |
| `PHYS-FR-07` | Экстренный приём инициируется регистратурой административно и требует последующего подтверждения уполномоченным медицинским сотрудником, прежде чем получить приоритет вне очереди | must |
| `PHYS-FR-08` | Приём, превысивший настраиваемый порог длительности (если он включён клиникой), требует обязательного указания причины перед подписанием консультации | must |
| `PHYS-FR-09` | Ограничение по длительности приёма — опциональное, каждая клиника включает/выключает и задаёт значение самостоятельно | must |
| `PHYS-FR-12` | Платный приём (по направлению или заказу исследования) не открывается для планового вызова, пока не подтверждена оплата или не получено разрешение ответственного лица | must |
| `PHYS-FR-13` | Врач может повторно вызвать пациента; после исчерпания настраиваемого числа попыток приём переходит в «неявка» | must |
| `PHYS-FR-14` | Регистратура может вручную вернуть пропущенного пациента в очередь врача | must |
| `EMR-FR-05` | Врач видит полную историю болезни пациента (все подписанные консультации и дополнения, документы, направления, результаты) | must |
| `EMR-FR-06` | Каждое открытие и изменение ЭМК фиксируется в аудит-трейле | must |
| `EMR-FR-09` | Врач может подписать консультацию с предварительным диагнозом без окончательного | must |
| `EMR-FR-10` | Уточнение/финализация диагноза после подписания консультации оформляется дополнением, не изменяя исходную запись | must |
| `EMR-FR-11` | Врач может зафиксировать структурированные назначения препаратов, процедур и рекомендаций как часть консультации | must |
