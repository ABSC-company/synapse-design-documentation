## Состояния экранов

| Состояние | Что видит пользователь |
|------------|--------------------|
| Загрузка | Скелетон списка приёмов / формы консультации / ленты ЭМК |
| Пусто | «Нет пациентов в очереди» на «Рабочем столе»; «ЭМК будет доступна после первого визита» на «ЭМК пациента» |
| Ошибка | Сообщение об ошибке с возможностью повторить, без потери уже введённых данных консультации |
| Нет связи | Баннер отсутствия сети; сохранение черновика консультации блокируется до восстановления связи |

## Отдельные случаи, которые стоит предусмотреть в макетах

**Рабочий стол врача**
- Пациент из очереди ушёл до вызова (талон отменён регистратурой) — строка исчезает без действия врача, без сообщения об ошибке.
- Два экстренных приёма подряд — оба видны с приоритетным визуальным маркером, порядок между ними — по времени поступления причины экстренности.
- Врач пытается вызвать пациента, пока предыдущий приём не завершён — «Вызвать» недоступна, пока текущий `Encounter` не в статусе `завершён`/`отменён`/`неявка`.

**Консультация**
- Попытка подписать консультацию без заполненного основного диагноза — блокируется с указанием обязательного поля.
- Приём превысил порог длительности, но клиника не включала это ограничение — предупреждение и поле причины не показываются вовсе.
- Врач правит уже подписанную консультацию — форма открывается в режиме создания новой версии, старая версия остаётся доступна для просмотра рядом.
- Назначенное исследование отменено в `LAB`/`INSTR` (не в этом модуле) — ссылка в консультации отражает статус «отменено» без ручного вмешательства врача.
- Направление отправлено, но целевой врач в этот момент недоступен (нет активной смены/кабинета) — приём у целевого врача всё равно создаётся в статусе `ожидает`, ожидание видно в «Электронной очереди».
- Потеря соединения в момент заполнения формы — черновик сохраняется локально до восстановления связи, повторная отправка не создаёт дубль консультации.

**ЭМК пациента**
- У пациента есть визиты, но нет ни одной подписанной консультации (например, все приёмы завершились неявкой) — лента визитов пуста, хотя `дата_первого_визита` не обязательно заполнена этим фактом (заполняется по факту визита, а не по факту подписанной консультации — уточнить трактовку с бэкендом при реализации).
- Аллергии не заполнены вовсе — блок в шапке не пугает пустотой, показывает нейтральное «Аллергии не указаны», а не пустое место.
- Поиск пациента по ПИНФЛ находит запись без ни одного визита — карта открывается в ограниченном виде, а не как «не найдено».

**Направление к врачу**
- Целевой врач совпадает с направляющим врачом — выбор недоступен/скрыт в списке.
- Пользователь закрывает окно направления с заполненной причиной, не отправив — данные формы не сохраняются, при повторном открытии форма пустая.
- Целевой врач за время заполнения формы стал недоступен (уволен/деактивирован) — исчезает из списка выбора без прерывания уже открытой консультации.

**Назначение исследования**
- Ни одна позиция не выбрана — кнопка «Назначить» неактивна.
- Часть выбранных исследований недоступна в справочнике на момент отправки (обновился прайс/перечень) — недоступные позиции подсвечиваются, отправка возможна по оставшимся.
- Повторное открытие окна в рамках той же консультации — уже назначенные ранее позиции не предлагаются повторно как «не выбранные», видно, что они уже в статусе «назначено».

**Настройки модуля**
- Администратор включает переключатель, но не указывает порог и пытается сохранить — блокируется с подсказкой заполнить значение.
- Порог уменьшен ниже длительности уже идущих приёмов — предупреждение появляется у этих приёмов сразу после сохранения настройки, без ожидания следующего действия врача.

**Открыть экстренный приём**
- Пациент не найден по указанным параметрам — регистратура может продолжить только после регистрации нового пациента (см. `Visits Page.md` / `Page Header.md`), форма экстренного приёма не создаёт пациента сама.
- Указанный целевой врач уже ведёт другой приём — экстренный приём всё равно создаётся сразу в `идёт_приём`, накладка обрабатывается организационно (врач сам решает очерёдность), интерфейс не блокирует создание.
