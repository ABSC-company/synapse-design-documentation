## Состояния экранов

| Состояние | Что видит пользователь |
|------------|--------------------|
| Загрузка | Скелетон списка приёмов / формы консультации / ленты ЭМК |
| Пусто | «Нет пациентов в очереди» на «Рабочем столе»; «ЭМК будет доступна после первого визита» на «ЭМК пациента»; «Нет обращений, ожидающих подтверждения» на «Подтверждении триажа» |
| Ошибка | Сообщение об ошибке с возможностью повторить, без потери уже введённых данных консультации |
| Нет связи | Баннер отсутствия сети; сохранение черновика консультации блокируется до восстановления связи |

## Отдельные случаи, которые стоит предусмотреть в макетах

**Рабочий стол врача**
- Пациент из очереди ушёл до вызова (талон отменён регистратурой) — строка исчезает без действия врача, без сообщения об ошибке.
- Экстренное обращение зафиксировано, но ещё не подтверждено уполномоченным медицинским сотрудником — строка видна с нейтральным индикатором «ожидает подтверждения триажа», не как уже приоритетный экстренный случай.
- Врач пытается вызвать пациента, пока предыдущий приём не завершён — «Вызвать» недоступна, пока текущий `Encounter` не в статусе `завершён`/`отменён`/`неявка`.
- Число попыток повторного вызова исчерпано во время просмотра экрана — строка автоматически переходит в «неявка» без перезагрузки страницы.
- Строка в статусе `приостановлен` долго не возобновляется (пациент не вернулся от целевого врача) — врач всё ещё видит «Возобновить приём», ограничения по времени ожидания на этом экране нет.

**Консультация**
- Попытка подписать консультацию без заполненного диагноза (ни предварительного, ни заключительного) — блокируется с указанием обязательного поля.
- Приём превысил порог длительности, но клиника не включала это ограничение — предупреждение и поле причины не показываются вовсе.
- Врач правит уже подписанную консультацию — форма открывается в режиме создания новой версии; уточнение диагноза после подписания — только через отдельное дополнение, форма консультации при этом не переоткрывается на редактирование.
- Назначенное исследование отменено в `LAB`/`INSTR` — ссылка в консультации отражает статус «отменено» без ручного вмешательства врача.
- Направление отправлено, но целевой врач в этот момент недоступен (нет активной смены/кабинета) — приём у целевого врача всё равно создаётся в статусе `ожидает_вызова` (или `ожидает_оплаты`, если платно), ожидание видно в «Электронной очереди».
- Оформлено направление «с возвратом», но врач не нажал «Приостановить приём» и продолжил заполнять текущую консультацию — кнопка остаётся доступной до явного нажатия; подписание текущей консультации до приостановки возможно и просто завершает приём обычным образом (направление при этом уже создано и живёт своей жизнью).
- Приём находится в `ожидает_оплаты`, но пациент решил отказаться от платной услуги — направление/заказ переводится в «отменено» отдельным действием (в `Referral`/заказе `LAB`/`INSTR`), `Encounter` целевого приёма отменяется, текущая консультация не блокируется этим фактом.
- Потеря соединения в момент заполнения формы — черновик сохраняется локально до восстановления связи, повторная отправка не создаёт дубль консультации.

**ЭМК пациента**
- У пациента есть визиты (`дата_первого_обращения` заполнена), но нет ни одной подписанной консультации (например, все приёмы завершились неявкой) — лента визитов пуста, хотя карта уже не в «ограниченном виде».
- Аллергии не заполнены вовсе — блок в шапке не пугает пустотой, показывает нейтральное «Аллергии не указаны».
- Поиск пациента по ПИНФЛ находит запись без ни одного обращения — карта открывается в ограниченном виде, а не как «не найдено».
- Попытка открыть ЭМК пациента, который не в очереди/не направлен к врачу — кнопка «Посмотреть историю»/«Открыть ЭМК» ведёт на форму «Экстренный доступ», а не на прямое открытие карты.
- Дополнение к консультации привязано к результату исследования, который впоследствии оспорен/переделан — оба дополнения (если их несколько) видны в ленте по порядку дат, не заменяют друг друга.

**Направление к врачу**
- Целевой врач совпадает с направляющим врачом — выбор недоступен/скрыт в списке.
- Пользователь закрывает окно направления с заполненной причиной, не отправив — данные формы не сохраняются, при повторном открытии форма пустая.
- Целевой врач за время заполнения формы стал недоступен (уволен/деактивирован) — исчезает из списка выбора без прерывания уже открытой консультации.
- Выбран тип «Плановое, на другой день», но целевой врач не имеет свободных слотов в ближайшие дни — форма не блокирует отправку, направление создаётся, дальнейшее согласование даты происходит через интеграцию с расписанием `REG` вне этого экрана.
- Выбран тип «С возвратом ко мне», но врач впоследствии заканчивает смену раньше, чем пациент возвращается — приостановленный приём остаётся в `приостановлен` до следующей смены/явного решения регистратуры, автоматической отмены нет.

**Назначение исследования**
- Ни одна позиция не выбрана — кнопка «Назначить» неактивна.
- Часть выбранных исследований недоступна в справочнике на момент отправки (обновился прайс/перечень) — недоступные позиции подсвечиваются, отправка возможна по оставшимся.
- Повторное открытие окна в рамках той же консультации — уже назначенные ранее позиции не предлагаются повторно как «не выбранные», видно, что они уже в статусе «назначено»/«ожидает оплаты».
- Результат по платному исследованию готов, но оплата так и не поступила — результат всё равно передаётся в `EMR`/консультацию (клинические данные не блокируются оплатой), но статус оплаты заказа остаётся видимым отдельно.

**Настройки модуля**
- Администратор включает переключатель длительности, но не указывает порог и пытается сохранить — блокируется с подсказкой заполнить значение.
- Порог уменьшен ниже длительности уже идущих приёмов — предупреждение появляется у этих приёмов сразу после сохранения настройки.
- Число попыток вызова уменьшено ниже уже сделанного количества попыток по идущим сейчас вызовам — при следующей проверке такие приёмы сразу переходят в «неявка».

**Зафиксировать экстренное обращение / Подтверждение триажа**
- Пациент не найден по указанным параметрам — регистратура может продолжить только после регистрации нового пациента, форма экстренного обращения не создаёт пациента сама.
- Указанный целевой врач уже ведёт другой приём — обращение всё равно фиксируется и после подтверждения переходит в `идёт_приём`, накладка обрабатывается организационно.
- Обращение отклонено на этапе подтверждения триажа — приём не удаляется, а становится обычным плановым (`триаж_уровень = плановый`), продолжает жить в общей очереди врача.
- Два уполномоченных сотрудника одновременно открывают одно и то же обращение для подтверждения — второй, кто нажмёт «Подтвердить»/«Отклонить», видит, что решение уже принято, без дублирующего перехода статуса.

**Дополнение к консультации**
- Дополнение открыто из уведомления о результате исследования, но исходная консультация к этому моменту относится к другому визиту, чем текущий приём врача — форма всё равно открывается в контексте именно той исходной консультации, а не текущего приёма.
- Попытка добавить дополнение без заполнения обязательных полей «Текст» и «Причина» — блокируется с указанием, что оба поля обязательны.
