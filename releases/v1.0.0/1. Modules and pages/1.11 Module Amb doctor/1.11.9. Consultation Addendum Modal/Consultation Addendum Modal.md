#### «Дополнение к консультации»

> Решает задачу «результат анализа пришёл после подписания приёма»: подписанная консультация не редактируется, но врач может добавить дополнение отдельной записью (`ConsultationAddendum`), которая становится частью истории ЭМК (`EMR-FR-03`, `EMR-FR-10`).

- **Назначение**: модальное окно добавления дополнения к уже подписанной консультации — из ленты «ЭМК пациента» или по прямой ссылке из уведомления о готовом результате исследования.
- **Состав**:
  - Ссылка на исходную консультацию (дата, диагноз на момент подписания) — только для чтения.
  - Поле «Текст дополнения» — обязательно.
  - Поле «Причина дополнения» — обязательно (например, «получен результат анализа, диагноз уточнён»).
  - Ссылка на результат исследования (`linked_result_id`) — подставляется автоматически, если дополнение открыто из уведомления о готовом результате.
  - Поле «Уточнённый диагноз» (МКБ-10) — опционально.
- **Действия**:
  - «Подписать дополнение» — сохраняет `ConsultationAddendum` сразу в подписанном виде (черновиков дополнений нет); если заполнен «Уточнённый диагноз», `Consultation.статус_диагноза` в целях отображения истории помечается как `уточнённый`, сама исходная запись не изменяется.
  - «Отмена» — закрывает окно без сохранения.

**Wireframe:**
```
┌───────────────────────────────────────────────┐
│ Дополнение к консультации                  [×]│
├───────────────────────────────────────────────┤
│ Консультация от 12.05.2026, Ахмедова Д.        │
│ Диагноз на момент подписания: J06.9 (заключ.)  │
│                                                 │
│ Связанный результат: Биохимический анализ      │
│ крови — готов 14.05.2026                       │
│                                                 │
│ Текст дополнения *                             │
│ [_____________________________________]        │
│                                                 │
│ Причина дополнения *                           │
│ [_____________________________________]        │
│                                                 │
│ Уточнённый диагноз (МКБ-10)                    │
│ [🔍 поиск по МКБ-10________________________]   │
│                                                 │
│                    [Отмена] [Подписать]        │
└───────────────────────────────────────────────┘
```

**Примечания для дизайнера:**
- Окно всегда открывается в контексте конкретной уже подписанной консультации — нет сценария «дополнение без исходной консультации».
- Поле «Уточнённый диагноз» не обязательно: дополнение может быть чисто информационным (например, «пациент уведомлён о результате по телефону»), без изменения диагностического статуса.
- В ленте «ЭМК пациента» дополнение отображается вложенно под исходной консультацией (см. [Patient EMR Page.md](../1.11.3.%20Patient%20EMR%20Page/Patient%20EMR%20Page.md)), а не как отдельная запись того же уровня.
- Дополнение подписывается сразу при сохранении — на экране нет состояния «черновик дополнения», в отличие от самой консультации.
