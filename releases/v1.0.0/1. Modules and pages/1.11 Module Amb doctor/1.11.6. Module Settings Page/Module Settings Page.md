#### «Настройки модуля» *(администратор)*

- **Назначение**: настройка ограничения длительности приёма — опциональная, задаётся отдельно для каждой клиники (`PHYS-FR-08`, `PHYS-FR-09`).
- **Состав**:
  - Переключатель «Ограничивать длительность приёма».
  - Поле «Порог, минут» — активно только при включённом переключателе.
- **Действия**:
  - Включение переключателя — раскрывает поле ввода порога в минутах; без сохранённого значения переключатель нельзя оставить включённым.
  - Выключение переключателя — поле скрывается/дизейблится; `причина_длительной_консультации` в «Консультации» после этого никогда не запрашивается.
  - «Сохранить» — применяет настройку для всех врачей клиники.

**Wireframe:**
```
┌───────────────────────────────────────────────┐
│ Настройки модуля — Амбулаторный врач           │
├───────────────────────────────────────────────┤
│ Ограничение длительности приёма                │
│                                                 │
│ Ограничивать длительность приёма   [●──○ Вкл] │
│                                                 │
│ Порог длительности приёма (мин)                │
│ [  30  ]                                       │
│                                                 │
│ ℹ При превышении порога врач увидит            │
│   предупреждение и обязан указать причину      │
│   перед подписанием консультации               │
│                                                 │
│                                    [Сохранить] │
└───────────────────────────────────────────────┘
```

**Примечания для дизайнера:**
- Нет единого значения по умолчанию для всех клиник — поле не должно быть предзаполнено «правдоподобным» числом, лучше пустое состояние до первого сохранения администратором.
- Выключение переключателя — не деструктивное действие (данные по уже завершённым приёмам с указанной причиной не затрагиваются и не удаляются), поэтому отдельного диалога подтверждения не требуется.
- Экран относится к общей панели администратора (см. `1.9 Module admin`) как модуль-специфичная вкладка настроек, а не отдельностоящая страница вне контекста админки.
