#### «Настройки модуля» *(администратор)*

- **Назначение**: настройка двух опциональных ограничений, которые каждая клиника задаёт самостоятельно — длительность приёма (`PHYS-FR-08`, `PHYS-FR-09`) и количество попыток повторного вызова пациента (`PHYS-FR-13`).
- **Состав**:
  - Переключатель «Ограничивать длительность приёма» + поле «Порог, минут» (активно только при включённом переключателе).
  - Поле «Количество попыток вызова до неявки» — число, применяется ко всем врачам клиники.
- **Действия**:
  - Включение переключателя длительности — раскрывает поле ввода порога в минутах; без сохранённого значения переключатель нельзя оставить включённым.
  - Выключение переключателя — поле скрывается/дизейблится; `причина_длительной_консультации` в «Консультации» после этого никогда не запрашивается.
  - «Сохранить» — применяет обе настройки для всех врачей клиники.

**Wireframe:**
```
┌───────────────────────────────────────────────┐
│ Настройки модуля — Амбулаторный врач           │
├───────────────────────────────────────────────┤
│ Ограничение длительности приёма                │
│                                                 │
│ Ограничивать длительность приёма   [●──○ Вкл] │
│                                                 │
│ Порог длительности приёма (мин)                │
│ [  30  ]                                       │
│                                                 │
│ ℹ При превышении порога врач увидит            │
│   предупреждение и обязан указать причину      │
│   перед подписанием консультации               │
├───────────────────────────────────────────────┤
│ Повторный вызов пациента                       │
│                                                 │
│ Количество попыток вызова до неявки            │
│ [  3  ]                                        │
│                                                 │
│                                    [Сохранить] │
└───────────────────────────────────────────────┘
```

**Примечания для дизайнера:**
- Нет единого значения по умолчанию ни для порога длительности, ни для числа попыток вызова — поля не должны быть предзаполнены «правдоподобными» числами до первого сохранения администратором.
- Выключение переключателя длительности — не деструктивное действие (данные по уже завершённым приёмам с указанной причиной не затрагиваются), отдельного диалога подтверждения не требуется.
- Уменьшение числа попыток вызова ниже уже накопленного счётчика у текущих приёмов в очереди — такие приёмы сразу переходят в «неявка» при следующей проверке, а не откладывается до следующего вызова; это стоит явно предусмотреть в поведении, а не только в статике экрана.
- Экран относится к общей панели администратора (см. `1.9 Module admin`) как модуль-специфичная вкладка настроек, а не отдельностоящая страница вне контекста админки.
