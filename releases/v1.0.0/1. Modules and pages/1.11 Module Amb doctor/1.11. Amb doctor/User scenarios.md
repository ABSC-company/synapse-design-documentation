## Пользовательские сценарии

**1. Приём пациента, записанного через регистратуру**
- *Предусловия:* талон пациента в статусе `ожидает_врача` для этого врача.
- *Основной поток:* врач на «Рабочем столе» видит список ожидающих пациентов (`Encounter.ожидает_вызова`) → нажимает «Вызвать пациента» → `Encounter` переходит в `вызван` → пациент заходит, врач подтверждает начало → `идёт_приём`.

**2. Консультация с созданием медицинского документа**
- *Основной поток:* врач заполняет `Consultation` (причина обращения, жалобы, анамнез, осмотр, диагноз — предварительный или заключительный, план) → добавляет структурированные назначения (`MedicationOrder`/`ProcedureOrder`/`Recommendation`) по необходимости → при необходимости формирует печатный документ (`ClinicalDocument`, шаблон из `DOCT`) → подписывает консультацию → `Encounter` переходит в `завершён`, талон электронной очереди — в `завершён`.

**3. Направление пациента к другому врачу**
- *Основной поток:* в рамках открытой консультации врач создаёт `Referral`, выбирает тип направления и конкретного целевого врача, указывает причину:
  - при `внутри_текущего_визита` — текущий приём завершается, у целевого врача открывается новый приём в рамках того же визита и того же номерка;
  - при `с_возвратом_к_текущему_врачу` — текущий приём приостанавливается, после консультации/исследования у целевого врача пациент возвращается, и исходный приём возобновляется;
  - при `плановое_на_другой_день` — формируется рекомендация/запись на другой день, очередь сегодняшнего дня не затрагивается;
  - при `без_создания_очереди` — направление остаётся рекомендацией в документе, без изменений в очереди.
- *Если целевая услуга платная:* новый `Encounter` создаётся в статусе `ожидает_оплаты` и не попадает в обычный вызов очереди, пока оплата не подтверждена или не разрешена ответственным лицом.

**4. Назначение лабораторного/инструментального исследования**
- *Основной поток:* врач в консультации выбирает «Назначить исследование» → указывает тип (лабораторное/инструментальное) и перечень → система вызывает публичный метод `LAB.СоздатьЗаказ`/`INSTR.СоздатьЗаказ` → в консультации сохраняется ссылка на заказ со статусом «назначено»; если исследование платное — применяется та же логика оплаты, что и для направлений.
- *Продолжение:* когда `LAB`/`INSTR` публикует результат, событие обновляет ссылку в консультации на «результат готов»; если консультация ещё не подписана — результат используется при подписании; если уже подписана — врач оформляет `ConsultationAddendum`, при необходимости уточняя диагноз.

**5. Просмотр истории болезни пациента**
- *Основной поток:* врач открывает ЭМК пациента (при наличии права) → видит ленту всех предыдущих консультаций и дополнений, документов, направлений и результатов исследований → открывает нужную запись для деталей.

**6. Работа с пациентом, впервые пришедшим в клинику**
- *Основной поток:* до `дата_первого_обращения` ЭМК пациента показывается врачу только в общем виде (ФИО, дата рождения), без истории; после первого прихода запись видна административно, но лента истории остаётся пустой, пока не появится первая подписанная консультация (`дата_первой_консультации`).

**7. Экстренный приём по триажу**
- *Основной поток:* пациент с потенциально экстренным случаем обращается в регистратуру → регистратура фиксирует обращение (`инициатор_экстренного_приёма_id`, `причина_экстренности`, `время_инициации`) и вызывает `PHYS.ЗафиксироватьЭкстренноеОбращение` → создаётся `Encounter(ожидает_вызова, триаж_уровень = экстренный)` → уполномоченный медицинский сотрудник подтверждает медицинскую экстренность (`подтвердил_триаж_id`, `время_подтверждения`) → приём переходит в `идёт_приём` напрямую, минуя обычный порядок вызова → далее ведётся как обычный приём.
- *Уточнение:* в стационаре аналогичную роль будет играть приёмный покой — сценарий уточнится при описании стационарного модуля.

**8. Предупреждение об аномально долгой консультации**
- *Основной поток:* приём находится в статусе `идёт_приём` дольше `порог_аномальной_длительности_приёма_минут` (если порог включён клиникой) → врач видит системное предупреждение → при попытке подписать консультацию система требует заполнить `причина_длительной_консультации` → после заполнения подписание становится доступно.

**9. Повторный вызов и неявка пациента**
- *Основной поток:* врач нажимает «Вызвать пациента» → талон переходит в `вызван_к_врачу` → пациент не заходит в течение отведённого времени → врач нажимает «Повторить вызов» → талон переходит в `повторный_вызов`, оповещение на табло/звук повторяется → если после настроенного числа попыток пациент так и не зашёл → талон переходит в `пропущен_врач`, `Encounter` — в `неявка`.
- *Возврат в очередь:* регистратура, связавшись с пациентом, может вручную вернуть его в очередь — талон переходит в `возвращён_в_очередь` и снова становится доступен для вызова.

**10. Приостановка приёма с возвратом**
- *Основной поток:* в рамках приёма врач оформляет `Referral(тип = с_возвратом_к_текущему_врачу)` → текущий `Encounter` переходит в `приостановлен` → пациент проходит консультацию/исследование у другого специалиста → по завершении целевого приёма исходный `Encounter` возвращается в `идёт_приём`, врач продолжает с того места, где остановился.

**11. Экстренный доступ к ЭМК вне обычных прав**
- *Основной поток:* врач, к которому пациент не записан и не направлен, обнаруживает необходимость немедленно посмотреть его ЭМК → открывает карту через «Экстренный доступ» → указывает причину (обязательно) → доступ предоставляется, событие фиксируется в аудит-трейле с пометкой «экстренный доступ».
