## Статусы и переходы

### Консультация (Consultation)

`черновик` (врач заполняет) → `подписана` (врач завершил приём и подписал запись; дальнейшие правки — только новой версией, старая версия остаётся видна в истории).

### Приём (Encounter)

| Из | В | Триггер |
|---|---|---|
| — | `ожидает` | создаётся по событию `REG.ТалонВызванКВрачу` |
| — | `идёт_приём` *(минуя `ожидает`)* | экстренный приём по триажу (`триаж_уровень = экстренный`) — врач/медсестра открывает приём сразу, без ожидания обычного вызова из очереди |
| `ожидает` | `идёт_приём` | врач подтверждает начало приёма (пациент физически зашёл) |
| `идёт_приём` | `завершён` | врач подписал консультацию |
| `ожидает` | `неявка` | пациент не зашёл за отведённое время (стык со статусом талона `пропущен_врач` в компоненте «Электронная очередь») |
| `ожидает` / `идёт_приём` | `отменён` | приём отменён вручную (редкий случай) |

### Направление (Referral)

`создано` → `в_очереди` (текущий `Encounter` переходит в `завершён`, но `visit_id`/`talon_id` не закрываются; у целевого врача создаётся новый `Encounter(ожидает)`, талон переадресуется — его `кабинет_id`/`врач_id` обновляются на нового врача, статус талона возвращается в `ожидает_врача`) → `выполнено` (целевой врач подписал свою консультацию) / `отменено`.
