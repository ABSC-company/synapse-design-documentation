#### «Открыть экстренный приём» *(регистратура)*

- **Назначение**: форма открытия приёма вне очереди по результатам оценки экстренности регистратурой — не доступна врачу или медсестре напрямую (`PHYS-FR-07`).
- **Состав**:
  - Поиск пациента (по ФИО, номеру ЭМК, ПИНФЛ).
  - Целевой врач — опционально, если известен заранее; если не указан, распределяется по доступности (уточняется на уровне «Электронной очереди»).
  - Причина экстренности — обязательное поле.
- **Действия**:
  - «Открыть приём» — вызывает `PHYS.ОткрытьЭкстренныйПриём`, создаёт `Encounter(идёт_приём, триаж_уровень = экстренный)` напрямую, минуя обычный порядок вызова из очереди; талон электронной очереди привязывается к этому приёму.
  - «Отмена» — закрывает форму без создания приёма.

**Wireframe:**
```
┌───────────────────────────────────────────────┐
│ Открыть экстренный приём                   [×]│
├───────────────────────────────────────────────┤
│ Пациент *                                      │
│ [🔍 ФИО / № ЭМК / ПИНФЛ_________________]      │
│                                                 │
│ Целевой врач (если известен)                   │
│ [🔍 Поиск врача________________________]       │
│                                                 │
│ Причина экстренности *                         │
│ [_____________________________________]        │
│                                                 │
│ ⚠ Приём будет открыт немедленно, минуя         │
│   обычный порядок вызова из очереди            │
│                                                 │
│                    [Отмена] [Открыть приём]    │
└───────────────────────────────────────────────┘
```

**Примечания для дизайнера:**
- Форма живёт в интерфейсе регистратуры (модуль амбулаторной регистратуры), а не в интерфейсе врача — врач видит уже готовый приём в статусе «идёт приём» у себя на «Рабочем столе»/сразу в «Консультации».
- Незаполненный целевой врач не блокирует открытие приёма — система должна уметь принять экстренного пациента даже без явного назначения конкретного врача в моменте.
- Индикатор `триаж_уровень = экстренный` должен быть заметен и на «Рабочем столе врача», и в самой «Консультации» — это не служебное поле, а сигнал для врача о характере приёма.
- Сценарий описан для амбулаторного сегмента; аналог для стационара (приёмный покой) будет отдельным модулем — эта форма не должна закладывать стационарные поля заранее.
