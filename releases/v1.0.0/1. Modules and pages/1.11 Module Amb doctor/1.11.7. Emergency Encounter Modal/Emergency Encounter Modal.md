#### «Зафиксировать экстренное обращение» *(регистратура)*

> Первый из двух шагов экстренного приёма: регистратура фиксирует обращение административно, но не принимает медицинское решение — экстренный приоритет в очереди начинает действовать только после подтверждения на экране «Подтверждение триажа» (см. [Triage Confirmation Page.md](../1.11.8.%20Triage%20Confirmation%20Page/Triage%20Confirmation%20Page.md)), которым владеет уполномоченный медицинский сотрудник (`PHYS-FR-07`).

- **Назначение**: форма фиксации потенциально экстренного обращения регистратурой — не доступна врачу или медсестре напрямую в этом шаге.
- **Состав**:
  - Поиск пациента (по ФИО, номеру ЭМК, ПИНФЛ).
  - Целевой врач — опционально, если известен заранее.
  - Причина экстренности — обязательное поле.
- **Действия**:
  - «Зафиксировать обращение» — вызывает `PHYS.ЗафиксироватьЭкстренноеОбращение`, создаёт `Encounter(ожидает_вызова, триаж_уровень = экстренный, инициатор_экстренного_приёма_id, время_инициации)`. Приём остаётся в `ожидает_вызова` — без приоритета — до подтверждения на отдельном экране.
  - «Отмена» — закрывает форму без создания приёма.

**Wireframe:**
```
┌───────────────────────────────────────────────┐
│ Зафиксировать экстренное обращение         [×]│
├───────────────────────────────────────────────┤
│ Пациент *                                      │
│ [🔍 ФИО / № ЭМК / ПИНФЛ_________________]      │
│                                                 │
│ Целевой врач (если известен)                   │
│ [🔍 Поиск врача________________________]       │
│                                                 │
│ Причина экстренности *                         │
│ [_____________________________________]        │
│                                                 │
│ ℹ Обращение будет зафиксировано и направлено   │
│   на подтверждение медицинскому сотруднику —   │
│   приём не получит приоритет автоматически     │
│                                                 │
│                  [Отмена] [Зафиксировать]      │
└───────────────────────────────────────────────┘
```

**Примечания для дизайнера:**
- Форма живёт в интерфейсе регистратуры (модуль амбулаторной регистратуры), а не в интерфейсе врача.
- Незаполненный целевой врач не блокирует фиксацию обращения — система должна уметь принять экстренного пациента даже без явного назначения конкретного врача в моменте.
- После фиксации приём отображается в очереди с нейтральным индикатором «ожидает подтверждения триажа», а не сразу как «экстренный» — визуально разграничить эти два состояния, чтобы врач не воспринял ещё не подтверждённый случай как уже приоритетный.
- Сценарий описан для амбулаторного сегмента; аналог для стационара (приёмный покой) будет отдельным модулем — эта форма не должна закладывать стационарные поля заранее.
