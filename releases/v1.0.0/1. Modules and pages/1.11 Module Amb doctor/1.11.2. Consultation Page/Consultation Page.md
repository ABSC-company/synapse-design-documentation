#### «Консультация»

- **Назначение**: основной рабочий экран приёма — ведение клинической записи (`Consultation`) в рамках открытого `Encounter` (`PHYS-FR-01`, `EMR-FR-03`, `EMR-FR-04`, `EMR-FR-09`, `EMR-FR-11`).
- **Состав**:
  1. Шапка пациента — ФИО, возраст, аллергии (всегда на виду, выделены цветом), хронические заболевания.
  2. Форма клинической записи — причина обращения, жалобы, анамнез заболевания/жизни, объективный осмотр (структурированные поля + свободный текст), диагноз (поиск по МКБ-10, предварительный/заключительный + сопутствующие), план лечения/рекомендации (свободный текст).
  3. Блок структурированных назначений — три секции: «Препараты» (`MedicationOrder`), «Процедуры» (`ProcedureOrder`), «Рекомендации» (`Recommendation`), каждая с построчным добавлением.
  4. Панель действий (кнопки, см. ниже).
  5. Боковая панель — сокращённая ЭМК, доступная не покидая консультацию.
- **Диагноз — предварительный или заключительный**: переключатель статуса диагноза (`статус_диагноза`); при выборе «предварительный» поле `диагноз_заключительный` не обязательно для подписания (`EMR-FR-09`); заключительный диагноз после подписания уточняется только через дополнение (см. [Consultation Addendum Modal.md](../1.11.9.%20Consultation%20Addendum%20Modal/Consultation%20Addendum%20Modal.md)).
- **Кнопки панели действий**:
  - **«Направить к врачу»** — открывает модальное окно «Направление к врачу» с выбором одного из 4 типов (см. [Referral Modal.md](../1.11.4.%20Referral%20Modal/Referral%20Modal.md)); доступна, пока консультация не подписана.
  - **«Назначить исследование»** — открывает модальное окно «Назначение исследования» (см. [Order Study Modal.md](../1.11.5.%20Order%20Study%20Modal/Order%20Study%20Modal.md)); не блокирует и не завершает текущую консультацию.
  - **«Сформировать документ»** — вызывает `DOCT.ПолучитьШаблон`, создаёт `ClinicalDocument`, привязанный к текущей консультации.
  - **«Приостановить приём»** — активна только когда только что оформлено направление типа `с_возвратом_к_текущему_врачу`; переводит `Encounter` в `приостановлен`, форма консультации сохраняется как черновик и возвращается при «Возобновить приём» с «Рабочего стола».
  - **«Подписать и завершить приём»** — переводит `Consultation` в `подписана`, `Encounter` — в `завершён`, талон электронной очереди — в `завершён`. Требует заполненного диагноза (предварительного либо заключительного). Если приём превысил `порог_аномальной_длительности_приёма_минут` (и порог включён клиникой) — не завершает подписание, пока не заполнена «Причина длительной консультации».
- **Функции (не кнопки, но обязательное поведение экрана)**:
  - Автоматическое появление предупреждения об аномально долгой консультации, если приём превысил настроенный порог.
  - Автоматическое обновление ссылки на заказ исследования на «результат готов», как только `LAB`/`INSTR` публикует событие о готовности — без действий врача; если консультация уже подписана, предлагается создать дополнение.
  - Индикатор оплаты: если приём находится в `ожидает_оплаты` (например, при возврате из платного направления), форма показывает баннер статуса без возможности продолжить до `оплачено`/`разрешено_без_оплаты`.
  - История версий консультации: после подписания правка доступна только через создание новой версии или дополнение, старая версия/дополнения видны в ленте ЭМК.

**Примечания для дизайнера:**
- Блок аллергий — не сворачиваемый и не второстепенный элемент: должен быть виден без скролла на любом размере экрана (`EMR-FR-04`).
- Переключатель «Предварительный / Заключительный» диагноза меняет, какое из двух полей диагноза обязательно для подписания — не показывать оба поля как одновременно обязательные.
- «Направить к врачу» и «Назначить исследование» не переводят консультацию в подписанную и не блокируют дальнейшее заполнение формы — они запускают параллельные процессы в других модулях.
- «Приостановить приём» — не самостоятельное решение врача «в любой момент», а прямое следствие только что оформленного направления с типом «с возвратом»; кнопка не должна быть доступна вне этого контекста, чтобы не создавать путаницу с обычной отменой приёма.
- Секции структурированных назначений («Препараты», «Процедуры», «Рекомендации») визуально отделены от свободнотекстового «Плана лечения» — оба поля сосуществуют, одно не заменяет другое.
- Диагноз по МКБ-10 — поиск с автокомплитом по коду или названию; сопутствующих диагнозов может быть несколько, основной (предварительный либо заключительный) — один и обязателен для подписания.
- Баннер «Ожидает оплаты» — некритично блокирующий: форма консультации остаётся заполняемой (черновик сохраняется), но кнопка «Подписать и завершить приём» недоступна, пока статус оплаты не изменится.
