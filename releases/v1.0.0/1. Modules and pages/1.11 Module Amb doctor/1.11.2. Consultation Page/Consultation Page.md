#### «Консультация»

- **Назначение**: основной рабочий экран приёма — ведение клинической записи (`Consultation`) в рамках открытого `Encounter` (`PHYS-FR-01`, `EMR-FR-03`, `EMR-FR-04`).
- **Состав**:
  1. Шапка пациента — ФИО, возраст, аллергии (всегда на виду, выделены цветом), хронические заболевания.
  2. Форма клинической записи — жалобы, анамнез заболевания/жизни, объективный осмотр (структурированные поля + свободный текст), диагноз (поиск по МКБ-10, основной + сопутствующие), план лечения/рекомендации.
  3. Панель действий (кнопки, см. ниже).
  4. Боковая панель — сокращённая ЭМК, доступная не покидая консультацию.
- **Кнопки панели действий**:
  - **«Направить к врачу»** — открывает модальное окно «Направление к врачу» (см. [Referral Modal.md](../1.11.4.%20Referral%20Modal/Referral%20Modal.md)); доступна в любой момент, пока консультация не подписана.
  - **«Назначить исследование»** — открывает модальное окно «Назначение исследования» (см. [Order Study Modal.md](../1.11.5.%20Order%20Study%20Modal/Order%20Study%20Modal.md)); не блокирует и не завершает текущую консультацию.
  - **«Сформировать документ»** — вызывает `DOCT.ПолучитьШаблон`, создаёт `ClinicalDocument`, привязанный к текущей консультации (заключение/справка).
  - **«Подписать и завершить приём»** — переводит `Consultation` в `подписана`, `Encounter` — в `завершён`, талон электронной очереди — в `завершён`. Требует заполненного `диагноз_основной`. Если приём превысил `порог_аномальной_длительности_приёма_минут` (и порог включён клиникой) — кнопка не завершает подписание, пока не заполнено поле «Причина длительной консультации».
- **Функции (не кнопки, но обязательное поведение экрана)**:
  - Автоматическое появление предупреждения об аномально долгой консультации, если приём превысил настроенный порог.
  - Автоматическое обновление ссылки на заказ исследования на «результат готов», как только `LAB`/`INSTR` публикует событие о готовности — без действий врача.
  - История версий консультации: после подписания правка доступна только через создание новой версии, старая версия остаётся видна.

**Примечания для дизайнера:**
- Блок аллергий — не сворачиваемый и не второстепенный элемент: он должен быть виден без скролла на любом размере экрана, это требование безопасности (`EMR-FR-04`), а не просто UI-предпочтение.
- «Направить к врачу» и «Назначить исследование» не переводят консультацию в подписанную и не блокируют дальнейшее заполнение формы — они запускают параллельные процессы в других модулях.
- Предупреждение об аномальной длительности приёма должно быть заметным, но не блокировать работу с формой — блокируется только сама кнопка подписания, пока не указана причина.
- Ссылки на назначенные исследования отображаются прямо в форме консультации (например, отдельным блоком «Исследования») со статусом «назначено» / «результат готов» — открытие результата не требует перехода в другой модуль.
- Диагноз по МКБ-10 — поиск с автокомплитом по коду или названию; сопутствующих диагнозов может быть несколько, основной — один и обязателен для подписания.
