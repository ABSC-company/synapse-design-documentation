## Публичный интерфейс (контракт с другими модулями)

**Синхронные методы, которые вызывает `PHYS`/`EMR`:**

| Метод | Модуль-владелец | Назначение |
|---|---|---|
| `REG.ВызватьПациентаКВрачу` | `REG` | кнопка «Вызвать пациента» / «Повторить вызов» (см. «Электронная очередь») |
| `LAB.СоздатьЗаказ` / `INSTR.СоздатьЗаказ` | `LAB` / `INSTR` | назначение исследования из консультации |
| `DOCT.ПолучитьШаблон` | `DOCT` | формирование печатного документа консультации |
| `CASH.ПроверитьТребуетсяОплата` | `CASH` | проверка, является ли услуга направления/заказа платной |
| `CASH.ПолучитьСтатусОплаты` | `CASH` | получение текущего статуса оплаты по счёту визита |

**Синхронные методы, которые предоставляет `PHYS` (вызывают `REG`/ответственные роли):**

| Метод | Вход | Выход | Назначение |
|---|---|---|---|
| `PHYS.ЗафиксироватьЭкстренноеОбращение` | пациент_id, врач_id (опц.), причина_экстренности, инициатор_id | `Encounter(ожидает_вызова, триаж_уровень=экстренный)` | Регистратура фиксирует потенциально экстренное обращение |
| `PHYS.ПодтвердитьТриаж` | приём_id, подтвердил_id | `Encounter(идёт_приём)` | Уполномоченный медицинский сотрудник подтверждает экстренность и переводит приём в приоритетный |
| `PHYS.РазрешитьБезОплаты` | приём_id/referral_id, ответственный_id, причина | `статус_оплаты = разрешено_без_оплаты` | Ответственное лицо разрешает клиническое продолжение до фактической оплаты |

**Синхронные методы, которые предоставляет `EMR`/`PHYS`:**

| Метод | Вход | Выход | Назначение |
|---|---|---|---|
| `EMR.ПолучитьКраткуюЭМК` | пациент_id | демография, аллергии, хронические заболевания (без полной истории) | Для регистратуры/кассы — административные данные без клинической части |
| `EMR.ПроверитьПервоеОбращение` | пациент_id | `дата_первого_обращения` заполнена/нет | Для реализации правила ограниченной видимости ЭМК |
| `EMR.ОткрытьЭкстренныйДоступ` | пациент_id, врач_id, причина | краткая/полная ЭМК (в зависимости от регламента) | Экстренный доступ вне обычных прав, с обязательным аудитом |

**Публикуемые доменные события:**

| Событие | Когда | Полезная нагрузка |
|---|---|---|
| `EMR.КонсультацияПодписана` | врач завершил и подписал приём | `consultation_id, эмк_id, приём_id, статус_диагноза` |
| `EMR.ДополнениеДобавлено` | врач подписал `ConsultationAddendum` | `addendum_id, consultation_id` |
| `PHYS.ПриёмЗавершён` | приём переведён в `завершён` | `приём_id, visit_id` |
| `PHYS.ПриёмПриостановлен` / `PHYS.ПриёмВозобновлён` | направление с возвратом переводит приём в `приостановлен`/обратно | `приём_id, visit_id` |
| `PHYS.НаправлениеСоздано` | оформлено направление к другому врачу | `referral_id, тип_направления, целевой_врач_id, visit_id` |
| `PHYS.ЭкстренныйПриёмИнициирован` / `PHYS.ТриажПодтверждён` | регистратура зафиксировала обращение / медицинское подтверждение получено | `приём_id, инициатор_id`/`подтвердил_id` |
| `PHYS.ПриёмОжидаетОплаты` | целевой приём/заказ требует оплаты | `приём_id, visit_id, referral_id` |

**Зависимости от других модулей:** `REG` (визит, компонент «Электронная очередь», плановая запись на другой день), `HR` (врач, специализация), `CLINIC` (кабинет), `LAB`/`INSTR` (заказы и результаты исследований — модули ещё не детализированы), `DOCT`/`FILE` (шаблоны и хранение печатных документов), `CASH` (проверка и статус оплаты платных направлений/исследований — без дублирования владения оплатой), `AUTH` (роли и права, включая право подтверждения триажа и разрешения без оплаты), `NOTIF` (уведомление врача о готовом результате исследования, уведомления по очереди).
