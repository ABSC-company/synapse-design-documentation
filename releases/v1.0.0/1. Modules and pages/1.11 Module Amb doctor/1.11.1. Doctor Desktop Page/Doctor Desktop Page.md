#### «Рабочий стол врача»

- **Назначение**: первый экран, который врач видит после входа в модуль; главный рабочий экран на весь день (`PHYS-FR-01`, `PHYS-FR-02`).
- **Состав**: список приёмов на сегодня — талоны электронной очереди, уже поставленные именно к этому врачу; кнопка «Вызвать пациента»; быстрый доступ к истории пациента без открытия приёма.
- **Колонки списка**: *Пациент, Время постановки в очередь, Степень срочности (плановый/экстренный), Статус (ожидает / идёт приём / завершён / неявка)*.
- **Действия**:
  - «Вызвать пациента» — доступна для верхней записи очереди в статусе `ожидает_врача`; создаёт/активирует `Encounter(ожидает)` и переводит талон в вызванный (см. «Электронная очередь»).
  - «Начать приём» — появляется после того, как пациент физически зашёл; переводит `Encounter` в `идёт_приём` и открывает «Консультацию».
  - «Посмотреть историю» на строке пациента — открывает «ЭМК пациента» в режиме просмотра, не открывая приём.

**Wireframe:**
```
┌──────────────────────────────────────────────────────────────────────┐
│  Рабочий стол врача                                    [Электронная  │
│                                                           очередь ▸]  │
├──────────────────────────────────────────────────────────────────────┤
│  Сегодня, 08.07.2026                                                  │
├───────────────────┬───────────┬────────────┬─────────────┬──────────┤
│ Пациент           │ В очереди │ Срочность  │ Статус      │ Действие │
├───────────────────┼───────────┼────────────┼─────────────┼──────────┤
│ Каримов А.А.      │ 09:10     │ Плановый   │ ●Ожидает    │[Вызвать] │
│ Юсупова Н.Б.      │ 09:15     │ Экстренный │ ●Идёт приём │[Открыть] │
│ Тошев Р.К.        │ 09:22     │ Плановый   │ ●Ожидает    │[Вызвать] │
└───────────────────┴───────────┴────────────┴─────────────┴──────────┘
                                              [История пациента]
```

**Примечания для дизайнера:**
- Экстренные приёмы (`триаж_уровень = экстренный`) визуально выделяются в списке (например, цветным маркером срочности) — они могут появиться в очереди врача вне обычного порядка постановки, минуя `ожидает`, сразу в статусе «идёт приём».
- Строка исчезает из активного списка ожидания сразу после того, как приём переходит в `завершён`, `отменён` или `неявка` — реактивно, без ручного обновления.
- «Вызвать пациента» — общий с компонентом «Электронная очередь» контрол, поведение и анимация вызова описаны в `Electronic Queue.md` (`1.3.6 Electronic Queue`), здесь не дублируются.
- Доступ к истории пациента до вызова — намеренно не блокирующее действие: врач должен иметь возможность подготовиться к приёму заранее.
