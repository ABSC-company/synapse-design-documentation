#### «Направление к врачу»

- **Назначение**: модальное окно оформления направления пациента к другому врачу из открытой консультации, с выбором одного из четырёх типов направления (`PHYS-FR-03`, `PHYS-FR-04`, `PHYS-FR-10`, `PHYS-FR-11`).
- **Состав**:
  - Переключатель типа направления: «В рамках этого визита» / «Плановое, на другой день» / «С возвратом ко мне» / «Без создания очереди (рекомендация)».
  - Выбор целевого врача (с индикацией занятости, как в «Booking Page») — обязателен для первых трёх типов; для «Без создания очереди» поле врача опционально (можно указать только специализацию текстом внутри причины).
  - Поле «Дата» — появляется только для типа «Плановое, на другой день».
  - Поле «Причина направления» — обязательно для всех типов.
  - Индикатор «Услуга платная» с текущим статусом оплаты — появляется, если целевая услуга требует оплаты.
  - Информационный блок с пояснением, что произойдёт с текущим приёмом (завершится / приостановится / не изменится) — текст меняется в зависимости от выбранного типа.
- **Действия**:
  - «Отправить направление» — создаёт `Referral(создано)`, переводит в `в_очереди`; дальнейшее поведение зависит от типа (см. таблицу типов направления в [Business rules and validations.md](../1.11.%20Amb%20doctor/Business%20rules%20and%20validations.md)):
    - *В рамках этого визита* — текущий `Encounter` завершается, у целевого врача создаётся новый `Encounter(ожидает_вызова)` с тем же `visit_id`/`talon_id`.
    - *С возвратом ко мне* — текущий `Encounter` приостанавливается, у целевого врача создаётся новый `Encounter`; после его завершения исходный приём возобновляется.
    - *Плановое, на другой день* — создаётся запись/рекомендация на указанную дату, очередь сегодняшнего дня не создаётся.
    - *Без создания очереди* — направление сохраняется как рекомендация в консультации/документе, ничего в очереди не создаётся.
  - «Отмена» — закрывает окно без создания направления, консультация продолжается.

**Примечания для дизайнера:**
- Переключатель типа направления должен быть первым элементом формы — от выбора зависит, какие поля появляются дальше (дата, обязательность врача), а не наоборот.
- Индикатор занятости целевого врача — переиспользовать визуальный паттерн из «Booking Page» (`1.3.2`).
- Врачи без свободных окон в этот день по-прежнему выбираемы для типов, создающих очередь (направление — не запись на конкретное время, а постановка в очередь к врачу), индикатор лишь честно показывает нагрузку.
- Индикатор платности и статуса оплаты — показывается сразу после выбора целевого врача/услуги, до отправки формы, чтобы врач понимал, что пациент не будет вызван немедленно, а попадёт в `ожидает_оплаты`.
- Информационный блок о судьбе текущего приёма — обязательный элемент для каждого типа, формулировка отличается («завершится» / «приостановится» / «останется без изменений»), это не декоративный текст, а способ не дать врачу случайно закрыть свой приём, когда он рассчитывал на паузу.
- После отправки окно закрывается; при типе «с возвратом» на экране консультации сразу появляется активная кнопка «Приостановить приём» (см. [Consultation Page.md](../1.11.2.%20Consultation%20Page/Consultation%20Page.md)).
- Поле «Причина направления» — обычный многострочный текст, без структурированного справочника, в отличие от целевого врача, который выбирается из списка.
