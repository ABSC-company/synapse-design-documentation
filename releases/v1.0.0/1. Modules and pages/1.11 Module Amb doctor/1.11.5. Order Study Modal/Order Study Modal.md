#### «Назначение исследования»

- **Назначение**: модальное окно заказа лабораторного и/или инструментального исследования прямо из консультации, без выхода в отдельный модуль (`PHYS-FR-05`, `PHYS-FR-06`).
- **Состав**:
  - Переключатель «Лабораторное / Инструментальное».
  - Список доступных исследований (из справочника `LAB`/`INSTR`), мультивыбор, с индикатором платности напротив каждой позиции.
  - Итоговый список выбранных позиций перед отправкой заказа.
- **Действия**:
  - «Назначить» — вызывает `LAB.СоздатьЗаказ` и/или `INSTR.СоздатьЗаказ` (в зависимости от типа и состава выбора), в консультации сохраняется ссылка на заказ со статусом «назначено». Если хотя бы одна позиция платная, применяется та же логика оплаты, что и для направлений (`ожидает_оплаты` → `оплачено`/`разрешено_без_оплаты`, см. [Business rules and validations.md](../1.11.%20Amb%20doctor/Business%20rules%20and%20validations.md)).
  - «Отмена» — закрывает окно без создания заказа.

│ Назначение исследования                    [×]│
├───────────────────────────────────────────────┤
│ ( ●Лабораторное )  ( ○Инструментальное )       │
│                                                 │

└───────────────────────────────────────────────┘

**Примечания для дизайнера:**
- Переключатель типа не сбрасывает уже выбранные позиции другого типа — врач может назначить и лабораторное, и инструментальное исследование за один проход (два отдельных вызова `СоздатьЗаказ`, один UI-шаг).
- После назначения в форме консультации сразу появляется строка заказа со статусом «назначено» (или «ожидает оплаты», если платное) — окно не должно быть единственным местом, где виден факт назначения.
- Статус «результат готов» появляется автоматически по событию из `LAB`/`INSTR`; если консультация уже подписана к этому моменту — врачу предлагается оформить дополнение (см. [Consultation Addendum Modal.md](../1.11.9.%20Consultation%20Addendum%20Modal/Consultation%20Addendum%20Modal.md)). На этом экране нет и не должно быть действия «обновить статус вручную» или «импортировать результат».
- Справочник исследований — общий для лабораторных и инструментальных, но выводится раздельно по переключателю; поиск работает в пределах текущего выбранного типа.
- Платность — свойство конкретной позиции справочника, а не всего заказа: заказ может содержать одновременно бесплатные и платные позиции, гейт оплаты применяется к заказу в целом только если в нём есть хотя бы одна платная позиция.
