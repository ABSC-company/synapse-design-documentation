#### «Подтверждение триажа» *(уполномоченный медицинский сотрудник)*

> Второй, обязательный шаг экстренного приёма: регистратура только фиксирует обращение (см. [Emergency Encounter Modal.md](../1.11.7.%20Emergency%20Encounter%20Modal/Emergency%20Encounter%20Modal.md)) — медицинское решение о том, что случай действительно экстренный, принимает дежурный врач, старшая медсестра или иной уполномоченный сотрудник по внутреннему регламенту клиники (`PHYS-FR-07`).

- **Назначение**: рабочий список зафиксированных, но ещё не подтверждённых экстренных обращений, с возможностью подтвердить или отклонить каждое.
- **Состав**: список приёмов в статусе `ожидает_вызова` с `триаж_уровень = экстренный` и незаполненным `подтвердил_триаж_id`; карточка каждого — пациент, причина экстренности, кто и когда зафиксировал обращение (`инициатор_экстренного_приёма_id`, `время_инициации`), целевой врач (если указан).
- **Действия**:
  - «Подтвердить» — вызывает `PHYS.ПодтвердитьТриаж`; приём переходит в `идёт_приём` напрямую, минуя обычный вызов из очереди, получает приоритет; заполняются `подтвердил_триаж_id`, `время_подтверждения`.
  - «Отклонить» — обращение признано не экстренным; приём возвращается в обычный плановый порядок очереди (`триаж_уровень = плановый`), запись не удаляется, а становится обычным плановым приёмом.

**Wireframe:**
```
┌──────────────────────────────────────────────────────────────────────┐
│ Подтверждение триажа                                                   │
├──────────────────────────────────────────────────────────────────────┤
│ Ожидают подтверждения: 2                                               │
├───────────────────┬──────────────────────┬───────────┬────────────────┤
│ Пациент           │ Причина экстренности │ Зафиксировал  │ Действие    │
├───────────────────┼──────────────────────┼───────────┼────────────────┤
│ Юсупова Н.Б.      │ Острая боль в груди  │ Рег. Каримова │[Подтвердить]│
│                   │                      │ в 09:12       │[Отклонить]  │
│ Ниязов Б.Х.       │ Высокая температура, │ Рег. Тошева   │[Подтвердить]│
│                   │ судороги             │ в 09:20       │[Отклонить]  │
└───────────────────┴──────────────────────┴───────────┴────────────────┘
```

**Примечания для дизайнера:**
- Экран не привязан к «Рабочему столу врача» — это отдельная рабочая очередь для роли, которой поручено подтверждение триажа (может быть дежурный врач, а не обязательно тот врач, к которому в итоге попадёт пациент).
- Список должен обновляться в реальном времени — задержка подтверждения напрямую означает задержку помощи пациенту.
- «Отклонить» — не отмена приёма: пациент остаётся в системе и просто переходит в обычный плановый поток, поэтому формулировка кнопки и последующее состояние должны явно избегать слова «отменить», чтобы не путать с полной отменой приёма.
- Причина экстренности, указанная регистратурой, должна быть видна целиком без сокращения — это единственная информация, на основе которой принимается решение, прежде чем откроется сама консультация.
