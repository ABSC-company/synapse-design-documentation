# Global Header

## Описание компонента

**Global Header** — глобальная панель навигации системы, расположенная в верхней части интерфейса и отображаемая на всех страницах приложения.

Компонент предназначен для:

- отображения текущего местоположения пользователя в системе;
- быстрого поиска ЭМК пациентов;
- доступа к уведомлениям;
- перехода в профиль пользователя;
- управления боковым меню приложения.

---

## Структура компонента

Компонент состоит из следующих элементов:

---

## Кнопка управления Sidebar

Расположена в левой части Header рядом с логотипом системы.

### Поведение

1. При нажатии открывает Sidebar, если он скрыт.
2. При повторном нажатии сворачивает Sidebar.
3. Состояние Sidebar сохраняется до изменения пользователем.

---

## Breadcrumb Navigation

Расположена после кнопки Sidebar.

Предназначена для отображения текущего пути пользователя внутри системы.

### Формат отображения

Главная → Раздел → Подраздел → Текущая страница

### Поведение

1. Все промежуточные элементы являются ссылками.
2. Текущая страница отображается как активный элемент.
3. При переходе между страницами Breadcrumb обновляется автоматически.

### Пример

Главная → Регистратура → Визиты

---

## Глобальный поиск ЭМК

Расположен в правой части Header.

Предназначен для быстрого поиска пациентов и перехода в их ЭМК.

### Task Flow

#### Этап 1. Ввод поискового запроса

Пользователь вводит поисковый запрос.

Поддерживаемые типы поиска:

- Ф.И.О пациента
- ID пациента
- PINFL
- номер телефона
- номер документа
- FaceID
- MRZ

---

#### Этап 2. Выполнение поиска

Поиск выполняется автоматически с использованием дебаунсинга.

---

#### Этап 3. Отображение результатов

Отображается выпадающий список найденных пациентов.

Для каждого результата:

- Ф.И.О пациента
- идентификатор пациента
- дата рождения
- фотография пациента (если есть)

---

#### Этап 4. Переход в ЭМК

При выборе пациента открывается его электронная медицинская карта.

---

## Центр уведомлений

Расположен справа от строки поиска.

Предназначен для отображения системных уведомлений пользователя.

---

## Индикатор непрочитанных уведомлений

- Если есть непрочитанные уведомления, в иконке отображается **красный кружок**
- Индикатор исчезает, когда все уведомления прочитаны

---

## Дропдаун уведомлений

Открывается при клике на иконку уведомлений.

### Структура

#### Список уведомлений

Отображает последние уведомления пользователя.

Каждое уведомление содержит:

- тип уведомления
- краткое описание
- дата и время
- статус (прочитано / не прочитано)
- ссылка на связанный объект (если есть)

Непрочитанные уведомления визуально выделяются.

---

### Поведение

- При клике на уведомление выполняется переход к связанному объекту
- После открытия уведомление автоматически помечается как прочитанное
- Изменения статуса происходят без перезагрузки

---

### Нижняя кнопка drop down

Содержит кнопку:

- **Посмотреть все уведомления**

#### Поведение

- Открывает driver-компонент (drawer)
- Закрывает dropdown

---

## Driver-компонент уведомлений

Выезжающая панель со всеми уведомлениями и фильтрацией.

---

### Заголовок

- Уведомления

---

### Tabs (вкладки)

- Все уведомления
- Системные
- Файлы
- Исследования
- Электронная очередь

---

### Поведение tabs

- Фильтрация уведомлений по типу
- «Все уведомления» показывает полный список
- Переключение вкладок не сбрасывает статус прочитанности
- Обновление данных без перезагрузки

---

## Общая логика уведомлений

- Непрочитанные уведомления сохраняют статус до открытия
- Открытие уведомления делает его прочитанным
- После прочтения всех уведомлений индикатор скрывается
- Статус синхронизируется между dropdown и driver-компонентом

---

## Типы уведомлений

- системные уведомления
- уведомления о визитах
- уведомления о документах
- уведомления о задачах
- уведомления о действиях пользователей
- файловые уведомления
- уведомления исследований
- уведомления электронной очереди

---

## Профиль пользователя

Расположен справа в Header.

Отображает:

- аватар пользователя
- статус присутствия

### Поведение

- Открывает меню пользователя по клику

---

## Меню пользователя

Содержит:

- Профиль пользователя
- Настройки
- Переключение языка
- Выход из системы

---

## Сценарии блокировки

### Отсутствие доступа к ЭМК

1. Поиск показывает пациента
2. При попытке открытия отображается ошибка прав доступа
3. Переход в ЭМК не выполняется

---

### Отсутствие соединения

1. Поиск блокируется
2. Показывается ошибка соединения
3. После восстановления доступ возвращается автоматически
