## Логика отображения

### Отображение по ролям

Содержимое Sidebar формируется динамически в зависимости от роли пользователя и назначенных прав доступа.

Пользователь видит только те разделы, подразделы и функциональные возможности, к которым ему предоставлен доступ.

Недоступные разделы не отображаются в интерфейсе Sidebar.

### Примеры отображения

#### Администратор

Пользователь с ролью **Администратор** имеет доступ ко всем модулям системы.

Пример отображаемых разделов:

- Главная
- Регистратура
- Врач
- Лаборатория
- Кассир
- Управление пользователями
- Настройки системы
- Отчеты
- Документация

---

#### Врач

Пользователь с ролью **Врач** имеет доступ только к разделам, связанным с врачебной деятельностью.

Пример отображаемых разделов:

- Главная
- Врач
- Пациенты
- Электронная медицинская карта
- Расписание приемов
- Исследования

---

#### Лаборант

Пользователь с ролью **Лаборант** имеет доступ только к лабораторным модулям.

Пример отображаемых разделов:

- Главная
- Лаборатория
- Исследования
- Результаты исследований

---

#### Кассир

Пользователь с ролью **Кассир** имеет доступ только к финансовым разделам системы.

Пример отображаемых разделов:

- Главная
- Кассир
- Платежи
- Финансовые операции

---

### Поведение

1. Состав Sidebar определяется при авторизации пользователя.
2. Изменение роли или прав доступа обновляет структуру меню.
3. Пользователь не может видеть разделы, на которые у него отсутствуют права доступа.
4. При прямом переходе по URL к недоступному разделу система отображает сообщение об отсутствии прав доступа.
5. Структура меню должна быть одинаковой во всех режимах отображения Sidebar (развернутом и свернутом).
