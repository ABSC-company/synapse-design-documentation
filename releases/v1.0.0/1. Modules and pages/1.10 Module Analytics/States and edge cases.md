

## 9. Состояния и edge cases

### Базовые состояния страницы

| Состояние | Ожидаемое поведение |
|---|---|
| Loading | Skeleton для KPI-карточек, графиков и таблиц; фильтры доступны, но применение блокируется до первой загрузки |
| Empty | Нет данных за выбранный период/фильтры — KPI показывают «—» или «0», графики показывают пустое состояние с подсказкой «Нет данных за выбранный период» |
| Error | Ошибка загрузки агрегатов — сообщение об ошибке и кнопка «Повторить», предыдущие успешно загруженные данные (если были) остаются видимыми с пометкой «Устаревшие данные» |
| No access | У роли нет прав на раздел — вместо дашборда экран с сообщением об отсутствии доступа, без отображения даже агрегированных цифр |
| Partial data | Часть модулей-источников не ответила (например, `WMS` недоступен) — доступные KPI и графики показываются, недоступные помечаются состоянием «Данные временно недоступны» вместо скрытия виджета целиком |
| Data outdated | Агрегаты закэшированы и не обновлялись дольше ожидаемого интервала — рядом с общим временем обновления дашборда показывается метка «Обновлено HH:MM» с предупреждающим цветом, если интервал превышен |
| Export in progress | Кнопка «Экспорт» переходит в состояние загрузки, повторное нажатие блокируется |
| Export failed | Тост/уведомление об ошибке экспорта с возможностью повторить, ранее выбранные параметры экспорта сохраняются |

### Edge cases: KPI и сравнение периодов

1. Для предыдущего периода нет данных для сравнения — вместо % изменения показывается «—» без иконки стрелки.
2. Значение KPI равно нулю, но не является ошибкой (например, «Возвраты» = 0) — отображается как обычное значение, не как empty-состояние.
3. Резкий скачок показателя (например, рост отмен в 5 раз) — карточка визуально попадает и в блок предупреждений, и остаётся в общем ряду KPI.
4. Слишком длинный произвольный период (например, год+) — группировка графиков автоматически укрупняется (по месяцам), чтобы не рисовать сотни точек.

### Edge cases: фильтры

1. Комбинация фильтров не даёт результатов — весь дашборд переходит в состояние Empty с подсказкой изменить фильтры, а не показывает пустые виджеты по отдельности.
2. Выбран врач, но затем изменено подразделение, где этого врача нет — фильтр врача сбрасывается автоматически с уведомлением пользователю.
3. Роль ограничена одним подразделением — фильтр «Филиал/корпус/отделение» показывает только доступную ветку дерева, остальные узлы не отображаются вовсе.
4. Фильтр «Возрастная группа/пол» отключён политикой данных — фильтр отсутствует в интерфейсе, ранее сохранённое значение фильтра (если политику отключили после того, как пользователь его выбрал) игнорируется без ошибки.

### Edge cases: графики и диаграммы

1. В периоде только одна точка данных (например, период «Сегодня») — линейный график вырождается в одну точку/столбец, отображается корректно без «сломанной» линии.
2. У отделения 0 визитов за период — сегмент диаграммы по отделениям не рисуется (не рисуется сегмент нулевой ширины), но отделение доступно для выбора через фильтр.
3. Слишком много отделений для читаемой круговой диаграммы — мелкие сегменты объединяются в категорию «Остальные» с раскрытием по клику.

### Edge cases: таблицы и топ услуг

1. Несколько врачей/услуг имеют одинаковое значение показателя — стабильная вторичная сортировка (например, по алфавиту), чтобы порядок не менялся между обновлениями страницы.
2. У врача нет визитов за период — строка остаётся в таблице загрузки с нулевыми значениями, а не скрывается (иначе руководитель не увидит простаивающего врача).
3. Услуга оказывалась в нескольких отделениях — в таблице топ услуг агрегируется суммарно, отделение может отображаться как «Несколько».

### Edge cases: блок предупреждений

1. Нет активных предупреждений — блок не отображается вовсе, не показывается как «Всё в порядке» пустой карточкой.
2. Одновременно несколько предупреждений одного типа (например, перегружены два врача) — группируются в одну карточку предупреждения со счётчиком, а не дублируются построчно.
3. Предупреждение перестало быть актуальным между обновлениями дашборда (например, очередь рассосалась) — карточка предупреждения исчезает при следующем обновлении без анимации тревоги.

### Edge cases: детализация и права доступа

1. Пользователь без права на персональные данные кликает на KPI, предполагающий список пациентов — открывается агрегированная детализация без ФИО, без сообщения об ошибке доступа (это ожидаемое поведение, не сбой).
2. Финансовый отдел открывает детализацию визита — видит финансовую часть (сумма, статус оплаты), но не клинические данные визита.
3. Экспорт запрошен ролью с ограниченными правами — выгруженный файл соответствует тому же ограничению видимости данных, что и экран (без «утечки» полных данных через экспорт).
