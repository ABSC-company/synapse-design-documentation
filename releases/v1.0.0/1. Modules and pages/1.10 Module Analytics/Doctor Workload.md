## 7. Таблицы и детализация

**Таблица загрузки врачей**

| Колонка | Описание |
|---|---|
| Врач | ФИО и специализация |
| Отделение | Подразделение врача |
| Визиты | Количество визитов за период |
| Загрузка | % коэффициент загрузки от доступного времени приёма |
| Среднее время приёма | В минутах |
| No-show, % | Доля неявок к визитам этого врача |

Сортировка по любому столбцу. Строка с аномально высокой (перегрузка) или низкой загрузкой визуально выделяется (см. §9, §11 предупреждения).

**Таблица топ услуг**

Переключение (Segment Control/Tab) между «По количеству» и «По выручке».

| Колонка | Описание |
|---|---|
| Услуга | Наименование услуги |
| Отделение | Подразделение, оказывающее услугу |
| Количество | Количество оказаний за период |
| Выручка | Суммарная выручка по услуге за период |
| Доля | % от общего количества/выручки клиники |

**Детализация (drill-down)**

Открывается Drawer поверх контента, наследует активные фильтры страницы. Содержимое зависит от источника перехода:

- клик по KPI «Отменённые визиты» — список отменённых визитов за период (дата, врач, отделение, причина отмены);
- клик по сегменту диаграммы отделений — тот же дашборд, но с применённым фильтром по этому отделению;
- клик по строке таблицы врачей — карточка/список визитов конкретного врача за период.

Если у роли нет прав на персональную детализацию — вместо списка визитов с ФИО пациентов показывается только агрегированная разбивка (например, по времени суток или по типу визита) без указания личности пациента.
