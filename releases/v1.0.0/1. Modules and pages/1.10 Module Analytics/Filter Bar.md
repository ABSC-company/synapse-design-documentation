## 8. Фильтры

Верхняя строка фильтров, слева направо:

1. **Период** — сегодня, вчера, 7 дней, 30 дней, месяц, квартал, произвольный период (календарь с диапазоном).
2. **Филиал / корпус / отделение** — выбор из дерева оргструктуры (см. [Organization Structure Page.md](../../1.9%20Module%20admin/Organization%20Structure%20Page.md)), с ограничением по правам роли.
3. **Врач** — список, зависящий от выбранного подразделения.
4. **Специализация** — список, зависящий от выбранного подразделения.
5. **Тип визита** — консультация / инструментальное исследование / лабораторное исследование.
6. **Статус визита** — ожидание, на приёме, завершено, отменено.
7. **Способ оплаты** — наличные, карта, безналичный расчёт, страховая компания и т.д.
8. **Источник пациента** — самостоятельное обращение, направление, повторное обращение и т.д.
9. **Возрастная группа / пол** — отображается только если это разрешено политикой данных клиники; при отключённой политике фильтр в интерфейсе отсутствует, а не просто скрыт disabled.

Поведение фильтров:

- Все фильтры, кроме периода, — многозначный выбор (multi-select).
- Выбранные фильтры отображаются как чипы над контентной областью, каждый можно снять по отдельности.
- Кнопка «Сбросить всё» возвращает фильтры к состоянию по умолчанию.
- Набор фильтров и период сохраняются между визитами пользователя на страницу (в рамках его профиля), чтобы не настраивать дашборд заново каждый раз.
