## 12. Acceptance criteria

1. Дашборд отображает первый ряд операционных KPI и второй ряд финансовых KPI на одном экране без прокрутки на десктопе стандартного разрешения.
2. По умолчанию применяется период «7 дней» и последний сохранённый набор фильтров пользователя.
3. Каждая KPI-карточка показывает крупное значение и сравнение с предыдущим аналогичным периодом.
4. Цвет индикатора изменения KPI соответствует смысловой полярности показателя (рост отмен — предупреждающий цвет, рост выручки — положительный).
5. График динамики визитов и график выручки отображаются рядом на широких экранах и используют единую временную шкалу.
6. Диаграмма по отделениям кликабельна и применяет фильтр отделения при клике на сегмент.
7. Таблица загрузки врачей поддерживает сортировку по каждому столбцу.
8. Таблица топ услуг переключается между режимами «по количеству» и «по выручке».
9. Блок предупреждений отображается только при наличии активных предупреждений и исчезает автоматически, когда предупреждение неактуально.
10. Все обязательные фильтры реализованы: период, филиал/корпус/отделение, врач, специализация, тип визита, статус визита, способ оплаты, источник пациента.
11. Фильтр «Возрастная группа / пол» отображается только при включённой политике данных клиники.
12. Активные фильтры отображаются как чипы с возможностью снять по отдельности и сбросить все разом.
13. Клик по KPI, сегменту диаграммы или строке таблицы открывает детализацию в Drawer с сохранением активных фильтров.
14. Персональная детализация (ФИО, контакты пациентов) доступна только ролям с соответствующим правом; для остальных ролей отображается агрегированная детализация без раскрытия личности пациента.
15. Руководитель подразделения не может через фильтр филиала получить доступ к данным другого подразделения.
16. Кнопка «Экспорт» формирует отчёт в выбранном формате и составе с учётом текущих фильтров и уровня доступа роли.
17. Реализованы состояния Loading, Empty, Error, No access, Partial data, Data outdated, Export in progress, Export failed.
18. При частичной недоступности источника данных (Partial data) доступные показатели отображаются, недоступные помечаются отдельным состоянием, а не приводят к ошибке всей страницы.
19. Показатель «Пациенты в электронной очереди» обновляется в фоне без перезагрузки страницы и визуально помечен как показатель реального времени.
20. Реализованы все перечисленные edge cases для KPI, фильтров, графиков, таблиц, блока предупреждений и детализации.
