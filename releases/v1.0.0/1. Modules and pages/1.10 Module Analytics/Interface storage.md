## 4. Структура интерфейса

Общий layout страницы:

1. Слева: Sidebar в свернутом состоянии.
2. Сверху: **Page Header** — название страницы «Аналитика клиники», кнопка **«Экспорт»**.
3. Под Header: **строка фильтров** на всю ширину.
4. Контентная область (сверху вниз):
   - Блок предупреждений (появляется только при наличии активных предупреждений).
   - Первый ряд KPI-карточек (операционные показатели).
   - Второй ряд KPI-карточек (финансовые показатели).
   - График динамики визитов и график выручки — рядом, в одну строку на широких экранах.
   - Диаграмма по отделениям.
   - Таблица загрузки врачей.
   - Таблица топ услуг.

Правила компоновки:

- Между KPI-рядом и графиками — визуальный разделитель или явный отступ, чтобы дашборд читался слоями: «сначала цифры, потом динамика, потом детализация».
- На узких экранах блоки складываются в одну колонку в том же порядке сверху вниз.
- Блок предупреждений всегда выше KPI, чтобы нештатная ситуация была замечена раньше, чем обычные показатели.
